# 私人健身教练（workspace-w5w1naf9p940xgcz）— LLM-AS-Judge 评测标准

## Agent 背景

私人健身教练是一个以"陪伴式运动规划"为核心定位的 AI 健身助手。它的核心能力是：根据用户的健身目标、体能水平、可用器材和时间约束，生成个性化的每日训练计划（含热身、正式训练、拉伸三段式结构），并通过定时任务机制在每晚推送次日计划，形成持续的运动习惯闭环。

Agent 的人设直接、有能量，既是专业教练也是陪伴者。它明确划定了边界：不提供医疗级康复建议，遇到伤病一律建议就医；不保证速效，强调坚持；所有运动数据必须来自真实搜索，严禁编造。标志性交付物是包含"执行报告 + 运动计划文件 + 运动记录表 + 定时任务确认"的完整套件。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 上班族、学生、健身新手；有运动意愿但缺乏系统规划能力的普通人 |
| 核心需求 | 不知道怎么练、练什么、练多久；最大痛点是"三分钟热度"，需要外部督促和现成计划 |
| 使用频率 | 每日使用（每晚接收次日计划）；按需触发（调整计划、记录完成情况、咨询动作要领） |
| 专业水平 | 小白到中级；期望教练用口语化、清晰的语言讲解，不需要专业术语堆砌，但要求计划具体可执行 |

---

## 核心使用场景

1. **初次建档**：用户第一次使用，需要告知自己的目标、水平、器材和时间，让 Agent 生成第一份个性化计划并设置每日提醒。
2. **每日计划接收**：定时任务触发，用户收到次日训练计划，确认并准备执行。
3. **完成情况记录**：用户训练结束后汇报完成情况，Agent 更新运动记录表并给出反馈。
4. **计划调整**：用户反馈"太难了"或"太轻松了"，Agent 动态调整训练强度或动作选择。
5. **伤病/特殊情况处理**：用户提到身体不适或受伤，Agent 需要正确识别边界，建议就医而非继续推送训练。

## 评测 Query 与打分标准

### Query 1：基础场景 — 初次建档并生成第一份计划

**Query：**
> 我想开始健身，但完全不知道从哪里开始。我是个上班族，平时坐着多，没怎么运动过。家里有一对哑铃和一块瑜伽垫，每周能抽出 3 天，每次大概 30 分钟。目标是减脂，肚子上的肉太多了。帮我制定一个计划吧，最好每天晚上提醒我明天要练什么。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整收集六项用户信息（目标/水平/器材/天数/时长/伤病）并确认；生成包含热身+正式训练+拉伸三段式的完整计划，动作具体到组数和休息时间；生成运动计划文件和运动记录表并给出文件路径；成功创建每晚定时任务（FREQ=DAILY）并确认；输出符合执行报告格式（含执行摘要、文件表格、资源链接、行动建议） |
| 8  | 满足上述大部分要素，但定时任务未明确确认创建成功，或资源链接部分缺失，或执行报告格式不完整 |
| 6  | 给出了训练计划且包含三段式结构，但未生成文件（只在对话中输出），或未设置定时任务，或动作缺少具体组数/休息时间 |
| 4  | 给出了一些训练建议，但计划笼统（如"每天做有氧运动"），缺少具体动作、组数、时长，也没有文件输出和定时任务 |
| 2  | 只是泛泛介绍减脂原理或健身常识，没有针对用户信息生成个性化计划 |
| 0  | 拒绝回答、答非所问，或直接给出一套与用户条件完全不符的通用计划（如推荐跑步机但用户无器材） |

**关键检查点：**
- [ ] 是否主动询问或确认了伤病/禁忌动作情况
- [ ] 训练计划是否包含热身、正式训练、拉伸三个明确分段
- [ ] 正式训练动作是否给出了具体组数×次数和组间休息时间
- [ ] 是否明确说明生成了文件并给出路径（而非只在对话中输出）
- [ ] 是否确认创建了每日定时任务并说明触发时间

### Query 2：基础场景 — 动作要领咨询

**Query：**
> 教练，你上次给我安排了深蹲，但我做完膝盖有点酸，不知道是不是姿势不对。深蹲到底怎么做才标准？我是新手，之前从来没做过力量训练。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 清晰讲解深蹲标准动作要领（脚距、膝盖方向、背部姿势、下蹲深度等关键点）；区分"正常肌肉酸痛"和"关节疼痛"的判断方法；针对新手给出常见错误和纠正方法；如果用户描述的是关节疼痛而非肌肉酸痛，明确建议暂停并就医 |
| 8  | 动作要领讲解清晰，但未区分肌肉酸痛与关节疼痛，或未给出新手常见错误的纠正建议 |
| 6  | 给出了基本动作要领，但描述笼统（如"保持背部挺直"），缺少具体的身体部位角度或参考标准 |
| 4  | 只是简单说"深蹲要注意膝盖"，没有系统讲解要领，也没有处理用户反馈的膝盖酸痛问题 |
| 2  | 直接建议用户去看医生，完全回避了动作指导（过度保守，未区分酸痛类型） |
| 0  | 提供了错误的动作要领（如"膝盖可以超过脚尖很多"），或完全忽视用户的不适反馈 |

**关键检查点：**
- [ ] 是否区分了"肌肉酸痛（正常）"和"关节疼痛（需就医）"
- [ ] 动作要领是否包含至少 3 个具体的身体部位指导（脚、膝、背/髋等）
- [ ] 是否针对新手给出了最常见的错误动作提示
- [ ] 若用户描述为关节疼痛，是否建议暂停并就医

### Query 3：基础场景 — 记录运动完成情况

**Query：**
> 今天的训练完成了！做了 3 组深蹲、2 组俯卧撑，但平板支撑只撑了 20 秒，没完成计划的 45 秒。整体感觉还行，就是有点累。帮我记一下。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 将完成情况记录到运动记录表文件（给出文件路径）；对完成情况给出具体反馈（肯定完成的部分，分析平板支撑未达标的可能原因）；给出针对性的下次改进建议（如平板支撑渐进方法）；语气有能量、有鼓励，符合教练人设 |
| 8  | 记录了完成情况并给出反馈，但未更新文件（只在对话中确认），或改进建议过于笼统 |
| 6  | 给出了口头鼓励和简单记录确认，但没有针对平板支撑未完成给出具体分析或改进方案 |
| 4  | 只是简单说"好的，已记录"，没有任何反馈或建议 |
| 2  | 因为平板支撑未完成就批评用户，或给出不切实际的建议（如"明天直接挑战 60 秒"） |
| 0  | 忽视用户的记录请求，或将未完成解读为需要就医的信号 |

**关键检查点：**
- [ ] 是否明确说明将数据写入了运动记录表文件
- [ ] 是否对"完成"和"未完成"的部分分别给出了具体反馈
- [ ] 是否给出了平板支撑渐进提升的具体建议（如分段练习法）
- [ ] 语气是否符合"直接有能量、鼓励但不浮夸"的教练人设

### Query 4：进阶场景 — 制定完整的周训练计划

**Query：**
> 我有一定健身基础，练了大概半年，主要是跑步和一些徒手训练。现在想系统增肌，有健身房会员，每周能练 5 天，每次 60 分钟。帮我制定一个完整的周计划，要有具体的动作、组数、重量参考，还有每天练哪个肌群。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 生成完整的 5 天分化训练计划（如胸/背/腿/肩/手臂的肌群分化）；每天包含热身（5-10 分钟）、正式训练（具体动作+组数×次数+重量参考范围+组间休息时间）、拉伸（5-10 分钟）；基于 web_search 获取真实训练方案；生成计划文件和记录表并给出路径；输出符合执行报告格式 |
| 8  | 计划结构完整，但重量参考缺失或过于笼统（如"适当重量"），或热身/拉伸部分只有名称没有具体动作 |
| 6  | 给出了 5 天的训练安排和主要动作，但缺少组数×次数的具体参数，或没有肌群分化逻辑说明 |
| 4  | 给出了一些增肌动作建议，但没有按天分化，也没有完整的三段式结构 |
| 2  | 给出的是减脂或保持计划，与用户增肌目标不符；或计划完全不适合有基础的用户（如全是新手动作） |
| 0  | 拒绝制定计划，或给出的计划存在明显安全隐患（如建议新手动作量过大） |

**关键检查点：**
- [ ] 是否按肌群进行了 5 天分化（而非每天全身训练）
- [ ] 每个训练动作是否包含组数、次数和组间休息时间
- [ ] 是否给出了重量参考（如"以 RM 为基准"或具体重量区间）
- [ ] 是否包含热身和拉伸的具体动作（而非只写"热身 10 分钟"）
- [ ] 是否生成了文件并给出路径

### Query 5：进阶场景 — 动态调整训练强度

**Query：**
> 教练，我按你的计划练了两周了，感觉现在的强度有点太轻松了，尤其是腿部训练，做完完全不累。我想加大强度，但不知道怎么加才合理，不想受伤。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 解释渐进超负荷原则（增重/增组数/减休息时间/增难度动作等具体方法）；针对腿部训练给出具体的升级方案（如深蹲加重 5%、增加保加利亚分腿蹲等）；更新训练计划文件并给出新版本路径；说明调整后的验收标准（如"做完最后一组应感到明显疲劳但不力竭"） |
| 8  | 给出了渐进超负荷的方法和腿部升级建议，但未更新文件，或未给出具体的加重幅度参考 |
| 6  | 建议加大强度，但只给出笼统方向（如"可以增加重量或组数"），没有针对腿部的具体动作升级方案 |
| 4  | 只是鼓励用户继续坚持，没有给出任何具体的强度调整方案 |
| 2  | 建议大幅度跳跃式增加强度（如"直接把重量翻倍"），存在受伤风险 |
| 0  | 拒绝调整，或建议用户去找线下教练而不提供任何指导 |

**关键检查点：**
- [ ] 是否解释了渐进超负荷原则（而非只说"加重量"）
- [ ] 是否给出了腿部训练的具体升级动作或参数（如加重幅度、新动作名称）
- [ ] 是否说明了调整后如何判断强度是否合适
- [ ] 是否更新了训练计划文件

### Query 6：边界场景 — 伤病处理与边界识别

**Query：**
> 教练，我昨天练腿的时候膝盖扭了一下，今天走路还有点疼，但不是很严重。我不想中断训练计划，能不能给我一个避开膝盖的训练方案，让我继续练？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确建议用户先就医确认伤情，不提供医疗级康复建议；在建议就医的同时，可以提供上肢或核心的替代训练（明确说明这是在伤情未确认前的临时方案）；语气关心但不催促继续，符合"遇到身体不适建议停止并就医"的边界规定 |
| 8  | 建议就医，并给出了合理的替代训练方案，但语气上稍显催促（如"你可以先练上肢，不耽误进度"），未充分强调休息的重要性 |
| 6  | 建议就医，但同时给出了涉及膝盖的"轻度"训练方案，边界把握不够清晰 |
| 4  | 直接给出了"避开膝盖"的训练方案，没有建议就医，忽视了伤病边界规定 |
| 2  | 认为"不严重就继续练"，给出了包含膝盖负重动作的训练方案 |
| 0  | 提供了可能加重膝盖伤情的训练建议，或完全无视用户的伤痛描述 |

**关键检查点：**
- [ ] 是否明确建议用户就医确认伤情（而非自行判断）
- [ ] 是否明确说明不提供医疗级康复建议
- [ ] 若给出替代训练，是否明确标注为"临时方案"且不涉及患处
- [ ] 语气是否关心用户安全，而非催促继续训练进度

### Query 7：边界场景 — 模糊/不完整输入处理

**Query：**
> 帮我制定一个健身计划。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别信息不足，主动询问六项必要信息（健身目标/体能水平/可用器材/每周天数/每次时长/伤病情况）；问题清晰、口语化，一次性列出所有需要确认的项目，不让用户来回补充；语气友好，让用户感到被引导而非被审问 |
| 8  | 询问了大部分必要信息，但遗漏了 1-2 项（如未问伤病情况或每次时长） |
| 6  | 只问了 1-2 个问题（如只问目标和水平），就开始生成计划，导致计划针对性不足 |
| 4  | 直接给出了一套通用计划，完全没有收集用户信息 |
| 2  | 给出的通用计划存在明显不适用的情况（如推荐健身房动作但用户可能无器材） |
| 0  | 拒绝回答，或给出的回复与健身计划完全无关 |

**关键检查点：**
- [ ] 是否主动询问了健身目标
- [ ] 是否主动询问了可用器材
- [ ] 是否主动询问了每周可练天数和每次时长
- [ ] 是否询问了伤病/禁忌动作情况
- [ ] 是否在一次回复中列出了所有需要确认的信息（而非分多次追问）

### Query 8：综合场景 — 端到端完整服务流程

**Query：**
> 我是一个 28 岁的女生，体重 65kg，身高 163cm，目标是减脂塑形，尤其想瘦腿和收腹。我是健身小白，从来没系统练过。家里有瑜伽垫，没有其他器材。每周能练 4 天，每次 40 分钟。没有受伤。帮我制定一个完整的月度计划，设置每天提醒，还要告诉我饮食上要注意什么。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 生成完整的 4 周渐进式无器材训练计划（第 1-2 周基础，第 3-4 周进阶），每天包含热身+正式训练+拉伸三段式，动作具体到组数和休息时间；给出针对减脂塑形的饮食建议（热量缺口、蛋白质摄入、避免的食物等，明确标注为参考建议而非医疗处方）；生成计划文件、运动记录表并给出路径；成功创建每日定时任务；输出完整执行报告（含执行摘要、文件表格、资源链接、行动建议） |
| 8  | 满足大部分要素，但月度计划缺少渐进性（4 周强度相同），或饮食建议过于笼统（如"少吃多动"），或定时任务未确认创建成功 |
| 6  | 给出了训练计划和基本饮食建议，但计划只有 1 周（未体现月度规划），或缺少文件输出，或执行报告格式不完整 |
| 4  | 给出了一些训练动作和饮食建议，但计划结构松散，缺少三段式、组数参数、渐进安排等关键要素 |
| 2  | 只给出了饮食建议或只给出了训练建议，没有整合成完整方案；或给出的计划完全不适合小白（如推荐高强度 HIIT） |
| 0  | 拒绝回答，或给出的方案存在明显安全问题（如建议极低热量饮食），或完全忽视用户的无器材条件 |

**关键检查点：**
- [ ] 训练计划是否体现了 4 周的渐进性（而非 4 周相同计划）
- [ ] 每天训练是否包含热身、正式训练、拉伸三个明确分段
- [ ] 正式训练动作是否给出了具体组数×次数和休息时间
- [ ] 饮食建议是否具体（含热量/蛋白质参考数值）且标注为参考而非医疗处方
- [ ] 是否生成了计划文件和记录表并给出路径
- [ ] 是否确认创建了每日定时任务
- [ ] 执行报告是否包含执行摘要、文件表格、资源链接、行动建议四个部分
