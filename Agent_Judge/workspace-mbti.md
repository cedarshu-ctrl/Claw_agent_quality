# MBTI 配对师（workspace-h356ncolfb0yaap7）— LLM-AS-Judge 评测标准

## Agent 背景

MBTI 配对师是一个专注于性格契合度分析的 AI 助手，核心定位是"从认知功能角度帮你看懂两个人为什么合得来、为什么吵起来"。它不是简单的打分机器，而是基于认知功能栈（如 Ni-Fe、Ne-Fi 等）从沟通风格、决策方式、情感需求、冲突处理四个维度拆解配对关系，适用于情侣互测、朋友合盘、职场搭档分析三大场景。

Agent 的工作流程严格要求基于真实数据进行分析，生成配对分析报告，最终以 Markdown 格式保存配对分析报告文件。它明确拒绝宿命论表述，强调 MBTI 是参考工具而非判决书，所有建议必须具体可操作，不做泛泛而谈。

| 维度 | 描述 |

| 核心需求 | 理解自己与特定对象（恋人/朋友/同事）之间的相处模式，找到冲突根源和改善方法，减少关系中的误解和摩擦 |
| 使用频率 | 按需触发，典型时机：刚开始一段新关系时、与某人发生冲突后、考虑是否深入发展某段关系时 |
| 专业水平 | 中级 MBTI 爱好者（知道自己的类型，了解基本概念，但不熟悉认知功能栈的深层逻辑）；期望得到有深度的分析而非表面描述 |

## 核心使用场景

1. **情侣配对分析**：用户和伴侣互测了 MBTI，想知道两人在认知功能层面是否契合，以及如何减少日常摩擦。
2. **暗恋/追求前评估**：用户喜欢某人，想在表白前了解两人性格是否合适，降低"踩坑"风险。
3. **朋友关系深度解读**：两个好朋友经常在某些事上意见相左，想搞清楚是性格差异导致的还是价值观问题。
4. **职场搭档匹配**：新组队或换搭档时，想提前了解合作风格差异，制定更顺畅的协作方式。
5. **关系冲突复盘**：某段关系出现了反复出现的矛盾，想从 MBTI 认知功能角度找到根本原因。

### Query 1：基础场景 — 情侣配对分析

**Query：**
> 我是 INFJ，我男朋友是 ENTP，我们在一起半年了，感觉很多时候他说话我听不懂他在想什么，他也说我太难猜。想知道我们这个组合到底合不合，有没有什么相处的建议？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明 INFJ（Ni-Fe-Ti-Se）与 ENTP（Ne-Ti-Fe-Si）的认知功能栈对比；从沟通风格、决策方式、情感需求、冲突处理四个维度分析差异；给出至少 2 条针对"听不懂对方"这一具体痛点的可操作建议；使用"有一种可能是..."等非宿命论表述；报告保存为文件并提供路径 |
| 8  | 覆盖认知功能分析和四维度分析，但相处建议较泛（如"多沟通"），未针对用户描述的"听不懂"痛点给出具体方法 |
| 6  | 给出了 INFJ 和 ENTP 的基本性格描述和大致匹配评价，但未涉及认知功能栈对比，建议停留在表面 |
| 4  | 仅描述了两种类型的性格特点，没有做配对分析，或只给出"你们差异很大"之类的笼统结论 |
| 2  | 直接说"你们不合适"或"你们天生一对"，使用宿命论表述；或内容与 INFJ/ENTP 配对无关 |
| 0  | 拒绝回答、答非所问、编造不存在的 MBTI 类型、或未确认类型就直接输出分析 |

**关键检查点：**
- [ ] 是否提到了 INFJ 和 ENTP 各自的认知功能栈（Ni-Fe vs Ne-Ti）
- [ ] 是否从至少 3 个维度（沟通/决策/情感/冲突）分析了两人差异
- [ ] 是否针对"听不懂对方"这一具体场景给出了可操作建议
