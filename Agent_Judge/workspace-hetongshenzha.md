# 合同审查专家（workspace-mvb0xpici5dga4u7）— LLM-AS-Judge 评测标准

## Agent 背景

合同审查专家（📋）是一位精通中国《民法典》合同编及商业合同实务的资深法务 AI，覆盖买卖合同、服务合同、技术开发合同、劳动合同、租赁合同、股权投资协议等七大合同类型。其核心能力是识别合同风险点并分级（高/中/低），出具结构化的书面审查意见书，并为每个风险点提供具体的修改建议和替代方案。

Agent 的人设特点是"风险导向但不杀死交易"——每个红旗都附带修复方案，区分必须改的红线与可接受的商业风险，以促成交易为目标而非阻碍签约。其能力边界明确：不提供具有法律约束力的正式法律意见，不替代持牌律师，仅基于中国法律框架进行审查。

## 核心使用场景

1. **采购/销售合同审查**：业务部门收到对方提供的标准合同模板，需要在签署前快速确认是否有对己方不利的条款
2. **技术合作合同知识产权保护**：与外部开发商合作，担心项目完成后代码归属和知识产权权属不清晰
3. **违约金条款谈判支持**：对方坚持高额违约金条款，需要法律依据来支撑谈判立场
4. **股权投资/融资协议审查**：初创公司创始人面对投资方提供的 TS/投资协议，需要识别对赌条款、回购条款等高风险条款
5. **合同缺失条款补全**：自己起草的合同草稿需要检查是否遗漏了保密、知识产权、不可抗力等关键条款

## 评测 Query 与打分标准

### Query 1：基础场景 — 采购合同违约金条款审查

**Query：**
> 我是一家制造业公司的采购经理，刚收到供应商发来的采购合同。里面第8条写的是："乙方（我们公司）如未按时付款，须按合同总金额的30%支付违约金，且不影响甲方要求赔偿实际损失的权利。"我感觉这个违约金有点高，但不知道有没有法律问题，也不知道怎么跟供应商谈。帮我看看这个条款有没有问题，应该怎么改？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确引用《民法典》第585条（违约金过高可请求调减），指出30%违约金叠加实际损失赔偿构成双重惩罚的法律风险（高风险），提供具体修改建议（如"违约金上限为实际损失的130%"或"违约金与损失赔偿二选一"），并给出谈判话术或替代方案 |
| 8  | 引用了《民法典》第585条，识别出双重惩罚问题，提供了修改建议，但未给出具体的替代条款文本或谈判策略 |
| 6  | 指出违约金偏高有法律依据可调减，但未识别"不影响赔偿实际损失"叠加的双重惩罚问题，或未提供具体修改方向 |
| 4  | 仅笼统说"违约金太高建议降低"，没有引用法律条文，没有分析双重惩罚问题，没有修改建议 |
| 2  | 回答偏离核心问题（如只讲违约金的一般知识），或建议"直接拒绝签署"而不提供谈判方案 |
| 0  | 拒绝回答、答非所问，或给出"这是合法的没有问题"的错误结论 |

**关键检查点：**
- [ ] 是否引用《民法典》第585条作为法律依据
- [ ] 是否识别出"违约金+实际损失双重赔偿"的叠加风险
- [ ] 是否将该条款标注为高风险（必须修改）
- [ ] 是否提供具体的替代条款文本或修改方向
- [ ] 是否给出谈判策略或替代方案（体现"促成交易"而非"杀死交易"的原则）

### Query 3：进阶场景 — 技术开发合同知识产权条款深度审查

**Query：**
> 我们公司要外包一个 SaaS 系统的开发，找了一家软件公司，对方发来的合同里关于知识产权是这样写的："本合同项下开发的软件系统，其著作权归乙方（开发商）所有，甲方（我们公司）获得永久使用授权。"我们出钱出需求，凭什么著作权归他们？但对方说这是他们的标准条款。请帮我出一份针对这个知识产权条款的详细审查意见，包括风险分析和我们应该坚持的修改方案。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出结构化审查意见，包含：（1）将该条款标注为🔴高风险，引用《著作权法》委托作品归属规定；（2）区分背景IP（开发商已有技术）和前景IP（本项目新开发成果）；（3）提出具体修改方案（前景IP归甲方所有，或共有，或约定分配）；（4）提示源代码交付和开源代码GPL感染性风险；（5）给出谈判策略（如接受背景IP归乙方但前景IP必须归甲方） |
| 8  | 识别高风险并提出前景IP归甲方的修改方案，区分了背景IP和前景IP，但未提及源代码交付条款或GPL感染性风险 |
| 6  | 指出著作权归属不合理，建议修改为归甲方所有，但未区分背景IP/前景IP，未提供谈判折中方案 |
| 4  | 仅说"著作权应该归你们"，没有法律依据，没有区分IP类型，没有具体修改条款文本 |
| 2  | 建议接受对方条款（"永久使用授权也够用了"），未识别核心风险 |
| 0  | 拒绝回答，或给出"著作权归开发商是合法的没问题"的错误结论 |

**关键检查点：**
- [ ] 是否将该条款标注为高风险（🔴）
- [ ] 是否区分背景IP（开发商已有技术）和前景IP（本项目新开发成果）
- [ ] 是否提供具体的替代条款文本（前景IP归甲方或约定分配）
- [ ] 是否提示源代码交付条款的必要性
- [ ] 是否提及开源代码GPL感染性风险

### Query 5：进阶场景 — 股权投资协议对赌条款审查

**Query：**
> 我是一家 A 轮融资的创业公司创始人，投资方发来的投资协议里有这样一条对赌条款："若公司未能在本协议签署后36个月内完成B轮融资，或公司2026年度经审计净利润未达到人民币500万元，投资方有权要求创始人以原始投资额加年化8%利息回购全部股权。"这个条款正常吗？有什么风险？我应该怎么谈？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别该条款为VAM（对赌协议/估值调整机制），标注为🔴高风险；分析双重触发条件（融资里程碑+利润指标）的叠加风险；指出创始人个人承担回购义务（而非公司）的法律风险（参考《九民纪要》对赌协议效力规则）；提供谈判策略（如：单一触发条件、设置业绩豁免条款、回购主体改为公司、降低利率或设置上限） |
| 8  | 识别VAM高风险，分析了双重触发条件，提供了谈判方向，但未提及《九民纪要》或创始人个人vs公司回购的法律效力差异 |
| 6  | 指出对赌条款风险较高，建议谈判降低要求，但未分析双重触发条件的叠加效应，未提供具体谈判方案 |
| 4  | 仅说"对赌条款很常见，这个条款比较严苛"，没有具体风险分析，没有谈判建议 |
| 2  | 建议直接接受（"这是行业惯例"），或建议直接拒绝融资，未提供平衡方案 |
| 0  | 拒绝回答，或给出"这个条款完全合法没有风险"的错误结论 |

**关键检查点：**
- [ ] 是否识别该条款为VAM（对赌协议）并标注为高风险
- [ ] 是否分析双重触发条件（融资里程碑+利润指标）的叠加风险
- [ ] 是否指出创始人个人承担回购义务的特殊法律风险
- [ ] 是否提供至少2种具体的谈判修改方向
- [ ] 是否体现"促成交易，控制风险"的商业平衡原则

### Query 6：边界场景 — 要求提供正式法律意见书

**Query：**
> 我们公司正在进行一笔重大并购，需要一份正式的法律意见书，要有律师事务所的抬头和律师签字，用于向银行申请并购贷款。你能帮我出具这份正式法律意见书吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 清晰说明自己不能出具具有法律约束力的正式法律意见书（需要持牌律师事务所），解释原因（银行要求的法律意见书需要律师执业资格和律所资质背书）；同时提供实质性帮助（如：说明正式法律意见书通常包含哪些内容、推荐寻找专业并购律师的渠道、可以帮助梳理并购合同的法律风险点供律师参考） |

- [ ] 是否避免出具任何可能被误认为正式法律意见书的文件

### Query 8：综合场景 — 端到端合同风险管理（从审查到签署）

**Query：**
> 我是一家互联网公司的 BD 总监，我们要和一家数据供应商签一份数据采购合同，合同金额 200 万/年。对方发来的合同草稿里有几个我不确定的地方：第一，数据使用范围写的是"用于甲方内部业务"，但我们可能会把数据用于给客户提供的分析报告；第二，合同里没有写数据质量标准和出错时的赔偿；第三，争议解决写的是"双方协商解决，协商不成提交仲裁"，但没写是哪个仲裁委。我需要你帮我：1）分析这三个问题的风险等级；2）给出每个问题的修改建议；3）告诉我这份合同还缺什么重要条款；4）建议我用什么方式签署（纸质还是电子签章）。

**打分标准（10 分制）：**
