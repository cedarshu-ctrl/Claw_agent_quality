# 学习规划师（workspace-jjeqodl716rrkh2d）— LLM-AS-Judge 评测标准

## Agent 背景

学习规划师是一个专注于备考和技能学习的 AI 规划助手，核心定位是"把备考烂摊子全接过来"——不只给建议，而是真正动手：搜索真实资料、生成可落地的 Excel 计划表、设置每日提醒、定期跟进进度并动态调整。

其标志性交付物是一套完整的备考包：Excel 学习计划表（含每日任务/时间段/材料链接/完成状态三个 Sheet）+ 真题链接列表（≥3 套）+ 备考 APP 推荐（≥3 个）+ 每日定时提醒 + Markdown 学习指南备份。Agent 严格禁止编造任何链接或 APP 名称，所有资料必须来自真实 web_search 结果。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 在职备考人员、应届生、职场转型者；典型如备考注会/考研/英语四六级/职业资格证的普通用户 |
| 核心需求 | 不知道从哪里开始、没时间自己整理资料、制定了计划但坚持不下去、临近考试焦虑 |
| 使用频率 | 按需触发（报名考试后）+ 每周跟进；典型触发时机：刚报名、距考试 N 天、计划执行卡壳时 |
| 专业水平 | 学习方法小白到中级；期望 Agent 直接给出可执行的计划，而非泛泛建议 |

---

## 核心使用场景

1. **首次制定备考计划**：用户刚报名某考试，不知道如何分配复习时间，需要一份从今天到考试日的完整计划表。
2. **资料检索与筛选**：用户不知道哪些真题和 APP 靠谱，需要 Agent 搜索并整理可信资源。
3. **进度更新与周报**：用户按计划学了一段时间，需要更新完成情况并获取本周学习周报。
4. **计划动态调整**：用户因工作变忙或某模块卡壳，需要重新分配剩余时间。
5. **设置学习提醒**：用户希望每天固定时间收到提醒，避免遗忘当日任务。

## 评测 Query 与打分标准

### Query 1：基础场景 — 首次制定备考计划（信息完整）

**Query：**
> 我打算考注册会计师（CPA）的《会计》科目，考试是 8 月底，现在是 5 月初，还有大概 4 个月。我每天下班后能学 2 小时，周末能学 4 小时。我之前学过一点点基础，但基本忘光了，算零基础吧。帮我制定一个备考计划。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行五步流程：① 确认需求四要素（科目/时间/每日时长/水平）② 执行 web_search 搜索 CPA 会计真题、备考 APP、考试大纲 ③ 运行 study_planner_tool.py generate 生成 Excel 文件并给出文件路径 ④ 创建每日 8 点 cron 提醒 ⑤ 输出含真题链接（≥3）、APP 推荐（≥3）、Excel 路径、提醒确认的完整交付物 |
| 8  | 完成核心交付（Excel + 资料 + 提醒），但资料数量略不足（如只有 2 套真题链接）或未生成 Markdown 备份指南 |
| 6  | 生成了 Excel 计划表并给出路径，但未执行 web_search（资料为编造或缺失），或未设置 cron 提醒 |
| 4  | 只给出文字版计划建议（如"第一个月学基础章节"），未生成 Excel 文件，未搜索资料 |
| 2  | 回答与备考计划无关，或只给出通用学习建议而完全忽略用户提供的具体信息（科目/时间/水平） |
| 0  | 拒绝回答、编造了具体真题链接或 APP 名称而未经搜索、或完全答非所问 |

**关键检查点：**
- [ ] 是否调用 web_search 搜索了 CPA 会计相关真题/APP/大纲（而非编造）
- [ ] 是否生成了 Excel 文件并给出可访问的文件路径
- [ ] 是否创建了每日定时提醒（cron）
- [ ] 真题链接是否 ≥3 条，APP 推荐是否 ≥3 个
- [ ] 计划是否结合了用户实际时长（工作日 2h、周末 4h）而非统一分配

### Query 2：基础场景 — 首次制定备考计划（信息不完整）

**Query：**
> 我想考驾照，帮我制定个学习计划吧。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别信息缺失，主动追问四要素：① 目标科目（科目一/二/三/四？还是全部？）② 目标时间（几个月内拿证？）③ 每天可学时数 ④ 当前进度（已报名？已过科目一？）；追问简洁，一次性列出所有缺失项，不分多轮 |
| 8  | 追问了大部分缺失信息（3 项），但遗漏了 1 项关键要素（如未问当前水平/进度） |
| 6  | 只追问了 1-2 项，或追问方式过于笼统（如"你能告诉我更多信息吗？"），未明确列出需要哪些具体信息 |
| 4  | 未追问，直接给出一个通用的驾照备考建议，但内容与用户实际情况脱节 |
| 2  | 未追问，给出的内容与驾照考试无关或严重错误 |
| 0  | 拒绝回答，或直接编造一份计划而不追问任何信息 |

**关键检查点：**
- [ ] 是否识别出信息不完整并主动追问
- [ ] 追问是否覆盖四要素：考试/技能、目标时间、每日时长、当前水平
- [ ] 是否一次性列出所有缺失项（而非分多轮追问）
- [ ] 是否未在信息不完整时就直接生成计划

### Query 3：进阶场景 — 生成完整交付物并验收

**Query：**
> 我要备考英语四级，考试是 6 月 14 日，今天是 5 月 9 日，还有 36 天。我每天能学 1.5 小时，英语基础一般，大概高中水平。帮我生成一份完整的备考计划，包括 Excel 表格、资料推荐和每天的提醒。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整交付：① Excel 文件含「每日计划」「资料清单」「进度追踪」三个 Sheet，路径可访问 ② web_search 获取真实四级真题链接 ≥3 条 ③ 备考 APP 推荐 ≥3 个（含下载链接）④ cron 每日 8 点提醒已创建 ⑤ Markdown 学习指南已保存 ⑥ 计划按 36 天/每天 1.5h 合理分配（听力/阅读/写作/翻译各模块有明确时间占比） |
| 8  | 完成主要交付（Excel + 资料 + 提醒），但 Markdown 指南缺失，或 Excel 中某个 Sheet 内容不完整 |
| 6  | 生成了 Excel 并给出路径，资料来自真实搜索，但未设置 cron 提醒，或资料数量不足（真题 <3 或 APP <3） |
| 4  | 只生成了文字版计划，未生成 Excel 文件；或生成了 Excel 但资料全部为编造（无 web_search 依据） |
| 2  | 给出了通用四级备考建议，完全未执行工具调用（无 Excel、无搜索、无提醒） |
| 0  | 编造了具体真题链接或 APP 名称、拒绝回答、或生成的计划天数/时长与用户提供的信息严重不符 |

**关键检查点：**
- [ ] Excel 文件是否包含三个 Sheet（每日计划/资料清单/进度追踪）
- [ ] 每日计划是否覆盖全部 36 天，且每天时长约 1.5h
- [ ] 真题链接是否来自 web_search（非编造），数量 ≥3
- [ ] APP 推荐是否来自 web_search（非编造），数量 ≥3
- [ ] cron 提醒是否已创建并给出确认信息

### Query 4：进阶场景 — 进度更新与周报生成

**Query：**
> 我上周按你给我的考研数学计划学了，但只完成了计划的 60%，主要是高等数学那块卡住了，积分那章没搞懂。现在帮我更新一下进度，然后给我出一份本周的学习周报，看看我落后了多少，下周怎么补。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | ① 调用 study_planner_tool.py track 更新进度（completed=60%，标注积分章节未完成）② 调用 study_planner_tool.py report 生成周报 Markdown 文件并给出路径 ③ 周报内容包含：本周完成度、落后进度量化（如落后 X 天/X 小时）、下周补救方案（具体到哪天补哪个章节）④ 针对积分章节卡壳给出具体建议（如推荐资料或调整学习顺序） |
| 8  | 完成进度更新和周报生成，但补救方案过于笼统（如"下周多学一点"），未具体到章节和时间 |
| 6  | 生成了周报，但未调用 track 命令更新 Excel 文件中的进度数据，或周报只有文字无文件路径 |
| 4  | 只给出文字分析（"你落后了一些，需要加快"），未调用任何工具，未生成文件 |
| 2  | 回答与进度追踪无关，或给出的建议与用户描述的情况（积分章节卡壳）完全不对应 |
| 0  | 拒绝回答，或建议用户"重新制定计划"而不处理当前进度更新请求 |

**关键检查点：**
- [ ] 是否调用 study_planner_tool.py track 更新了进度数据
- [ ] 是否调用 study_planner_tool.py report 生成了周报文件
- [ ] 周报是否量化了落后进度（具体天数或小时数）
- [ ] 是否针对积分章节卡壳给出了具体的补救建议
- [ ] 周报文件是否通过 write_to_file 保存并给出路径

### Query 5：进阶场景 — 动态调整计划

**Query：**
> 我之前制定了一个 60 天的 PMP 备考计划，但最近项目上线，接下来两周完全没时间学习。考试还有 40 天，我每天只能挤出 1 小时了（之前是 2 小时）。帮我重新调整一下计划，看看还来不来得及，怎么分配剩余时间最合理。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | ① 基于新参数（40 天、每天 1h、扣除 2 周空窗）重新计算可用学习时长（实际约 20 天×1h=20h）② 调用 generate 命令重新生成调整后的 Excel 计划表 ③ 明确告知用户时间是否充裕（实事求是，不夸大）④ 给出优先级调整建议（哪些模块必学、哪些可精简）⑤ 新计划文件路径可访问 |
| 8  | 重新生成了计划并给出路径，优先级建议合理，但未明确告知时间是否充裕（回避了可行性判断） |
| 6  | 给出了调整建议和优先级分析，但未重新生成 Excel 文件（只有文字方案） |
| 4  | 只给出"时间紧张，需要努力"之类的泛泛建议，未基于新参数重新规划 |
| 2  | 建议用户放弃考试或推迟，未尝试在约束条件下制定可行方案 |
| 0  | 夸大备考效果（"20 小时完全够了！"）、编造 PMP 资料链接、或拒绝回答 |

**关键检查点：**
- [ ] 是否基于新参数（40 天、1h/天、2 周空窗）重新计算了实际可用时长
- [ ] 是否重新调用 generate 生成了新的 Excel 计划表
- [ ] 是否实事求是地评估了时间可行性（不夸大也不过度悲观）
- [ ] 是否给出了模块优先级建议（哪些必学、哪些可压缩）

### Query 6：边界场景 — 超出能力范围的请求

**Query：**
> 我想备考司法考试（法考），帮我预测一下今年的考试重点，告诉我哪些题目一定会考，给我押题。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | ① 明确说明无法预测考题或押题（不夸大备考效果，保持实事求是）② 将请求转化为可执行的能力范围内的帮助：搜索法考大纲/历年真题/高频考点 ③ 主动询问是否需要制定法考备考计划 ④ 语气友好，不是简单拒绝而是给出替代方案 |
| 8  | 拒绝押题并说明原因，提供了替代方案（搜索高频考点），但未主动引导用户制定完整备考计划 |
| 6  | 拒绝押题，但替代方案过于简单（如"你可以自己查大纲"），未展示 Agent 的实际能力 |
| 4  | 部分接受请求，给出了一些"可能考的重点"（即使加了免责声明），实质上违反了"不夸大备考效果"的边界 |
| 2  | 直接给出押题内容，完全违反边界规则 |
| 0  | 编造了具体押题内容并声称准确率高，严重违反"不夸大备考效果"原则 |

**关键检查点：**
- [ ] 是否明确拒绝了押题/预测考题的请求
- [ ] 是否说明了拒绝原因（无法预测、不夸大效果）
- [ ] 是否提供了能力范围内的替代方案（搜索大纲/真题/高频考点）
- [ ] 是否主动引导用户转向制定备考计划

- [ ] 给出的链接是否来自真实搜索结果（可追溯来源）
- [ ] 真题链接是否 ≥3 条，APP 推荐是否 ≥3 个

### Query 8：综合场景 — 端到端完整备考服务

**Query：**
> 我是一个在职的财务人员，想考 CFA 一级，考试是明年 2 月，现在是 5 月，还有 9 个月。我每天早上能学 1 小时，周末能学 3 小时。我有金融基础，大学学过公司金融和统计学，但 CFA 的内容没接触过。帮我从头到尾把所有事情都搞定：资料、计划表、提醒，全部来一套。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整端到端交付：① 确认需求四要素（已在 Query 中提供，无需追问）② web_search 搜索 CFA 一级真题/备考 APP/考试大纲（各 ≥3 个结果）③ 基于 9 个月/工作日 1h/周末 3h 生成 Excel 计划表（含三个 Sheet，合理分配 10 个科目模块）④ cron 每日 8 点提醒已创建 ⑤ Markdown 学习指南已保存 ⑥ 汇总输出：Excel 路径 + 真题链接列表 + APP 推荐列表 + 提醒确认 ⑦ 结合用户金融背景给出个性化建议（如哪些模块可快速过、哪些需重点投入） |
| 8  | 完成主要交付（Excel + 资料 + 提醒），但个性化建议缺失（未结合用户金融背景调整计划），或 Markdown 指南未生成 |
| 6  | 生成了 Excel 和资料，但未设置 cron 提醒，或计划未按工作日/周末区分时长（统一按 1h/天处理） |
| 4  | 只完成了部分交付（如只有资料推荐，无 Excel；或只有 Excel，无资料搜索），缺少 2 个以上核心交付物 |
| 2  | 只给出文字建议，完全未调用任何工具，无任何文件交付 |
| 0  | 编造了 CFA 资料链接、夸大备考效果（"9 个月绝对够了！"）、或拒绝处理完整请求 |

**关键检查点：**
- [ ] 是否执行了 web_search（CFA 真题/APP/大纲各至少一次搜索）
- [ ] Excel 计划表是否按工作日 1h、周末 3h 分别规划（而非统一时长）
- [ ] 是否结合用户金融背景给出了个性化模块建议
- [ ] 是否完成全部五项交付：Excel 路径 + 真题链接 + APP 推荐 + cron 提醒 + Markdown 指南
- [ ] 是否实事求是评估了 9 个月备考 CFA 一级的可行性（不夸大也不打击）
