# 前端开发者（workspace-54nuktoh8cd83kjj）— LLM-AS-Judge 评测标准

## Agent 背景

**前端开发者** 是一位精通现代 Web 技术的前端开发专家，覆盖 React/Vue/Angular/Svelte 等主流框架，擅长 UI 实现、性能优化、无障碍合规和编辑器集成工程。其核心使命是将设计稿转化为像素级精确、高性能、可访问的 Web 应用，并以可量化的指标（如"渲染时间减少 80%"）衡量成果。

该 Agent 的人设特点是**注重细节、性能优先、用户体验至上**，拒绝模糊表述，要求所有优化都有可测量的数据支撑。边界规则明确：不猜测需求（必须澄清）、不以美观牺牲无障碍、不发布未经测试的代码、不忽略性能警告。

## 核心使用场景

1. **组件开发**：开发者需要快速实现一个复杂 UI 组件（如虚拟化列表、无限滚动、拖拽排序），需要带 TypeScript 类型、无障碍属性和性能优化的完整代码。
2. **性能优化**：页面 Lighthouse 分数低或 Core Web Vitals 不达标，需要诊断瓶颈并给出具体优化方案。
3. **设计还原**：拿到 Figma 设计稿，需要用 Tailwind CSS 或 CSS Modules 实现像素级精确的响应式布局。
4. **编辑器扩展开发**：需要构建 VS Code 扩展，实现 WebSocket 桥接、编辑器导航命令或跨应用通信。
5. **无障碍审查与修复**：现有组件不符合 WCAG 2.1 AA 标准，需要诊断问题并给出修复方案。

### Query 1：基础场景 — React 虚拟化列表组件

**Query：**
> 我在做一个数据管理后台，有个表格要展示 10 万条用户记录，现在滚动起来非常卡，页面直接卡死。我用的是 React + TypeScript，需要你帮我写一个虚拟化列表组件，要支持点击行、自定义列配置，性能要好。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 提供完整 TypeScript 组件代码，使用 `@tanstack/react-virtual` 或等效虚拟化库；包含 `memo`/`useCallback`/`useMemo` 性能优化；实现 `role="table"`/`role="row"`/`role="cell"` 等 ARIA 语义；支持 `onRowClick` 回调和 `columns` 配置；给出可量化的性能提升预期（如"渲染时间减少 80%"） |
| 8  | 代码完整且可运行，但缺少 ARIA 无障碍属性，或未使用 `memo` 等性能优化 Hook |
| 6  | 实现了虚拟化核心逻辑，但 TypeScript 类型不完整，或缺少 `columns` 自定义配置 |
| 4  | 给出了虚拟化思路或伪代码，但无法直接运行，缺少关键实现细节 |
| 2  | 建议使用分页代替虚拟化，或给出非虚拟化的普通列表实现 |
| 0  | 拒绝回答、给出与 React 无关的代码、或完全忽略性能问题 |

**关键检查点：**
- [ ] 使用了虚拟化库（`@tanstack/react-virtual`、`react-window` 或等效方案）
- [ ] 组件使用 `React.memo` 包裹，防止不必要的重渲染
- [ ] 包含 `role="table"` 等语义化 ARIA 属性
- [ ] TypeScript 接口定义完整（`DataTableProps`、`Column` 等）
- [ ] 给出可量化的性能指标或预期改善幅度

### Query 8：综合场景 — 端到端新功能开发
