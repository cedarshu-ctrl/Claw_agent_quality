# API 测试专家（workspace-awqog7xmcsz256v8）— LLM-AS-Judge 评测标准

## Agent 背景

API 测试专家是一个面向后端开发和测试工程师的 API 批量测试全链路助手，内置 6 大工作流：依赖搭建、需求解析、用例生成、批量测试、报告输出、验收适配。其核心价值在于将手动逐条测试接口的繁琐工作自动化——从读取 Swagger/OpenAPI 文档，到生成覆盖正常/异常/边界场景的测试用例，再到批量执行请求、验证响应（状态码/字段/性能），最终 write 保存可追踪的 Markdown 测试报告。

Agent 的人设强调"实操导向、数据真实"：接到需求直接开干，严禁编造任何测试结果，报告必须以文件形式保存而非仅在对话中输出。支持 REST/GraphQL/gRPC 三种 API 类型的场景化适配。

## 核心使用场景

1. **上线前批量回归**：版本发布前，测试工程师需要快速跑一遍所有接口，确认没有因代码改动导致的回归问题。
2. **新接口冒烟测试**：后端开发刚写完一批接口，需要验证基础功能是否正常，包括正常请求、缺参数、错误认证等场景。
3. **Swagger 文档驱动测试**：团队维护了 OpenAPI/Swagger 文档，希望直接从文档自动生成并执行测试用例，减少手工维护测试脚本的成本。
4. **性能基线验证**：关注接口响应时间，需要在测试报告中体现各接口的耗时，判断是否满足 SLA 要求。
5. **异常场景覆盖**：确保接口对非法参数、越权请求、超大 payload 等边界情况有正确的错误响应（4xx 系列），而不是直接 500。

### Query 1：基础场景 — 快速上手批量测试

**Query：**
> 我是后端开发，刚写完一个用户模块，有 3 个接口：GET /api/users（获取用户列表）、POST /api/users（创建用户）、DELETE /api/users/{id}（删除用户），服务跑在本地 http://localhost:8080，不需要认证。帮我批量测一下这几个接口，看看基本功能有没有问题，给我一份测试报告。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 先确认需求（URL/接口列表/认证/报告格式），生成覆盖正常+异常+边界场景的测试用例集，用 curl 或 python3 脚本实际执行请求，验证状态码（200/201/404/400 等）和响应字段，最终 write 保存 Markdown 报告，报告包含执行摘要、详细结果表格（接口名/状态码/耗时/是否通过）、行动建议 |
| 8  | 完成了测试执行和报告保存，但测试用例仅覆盖正常场景，缺少异常场景（如 DELETE 不存在的 id、POST 缺少必填字段）|
| 6  | 生成了测试用例并尝试执行，但报告只在对话中输出文字，未用 write 保存为文件；或执行了请求但未验证响应字段，只看了状态码 |
| 4  | 给出了 curl 命令示例让用户自己跑，没有实际执行；或只生成了用例集但没有执行测试 |
| 2  | 仅给出通用的 API 测试建议或工具推荐（如"可以用 Postman"），没有针对用户提供的具体接口做任何操作 |
| 0  | 拒绝回答、答非所问，或声称"无法访问本地服务"后直接放弃而不给出任何可执行方案 |

**关键检查点：**
- [ ] 是否实际执行了请求（curl 或 python3 脚本），而非仅生成用例
- [ ] 是否覆盖了至少一个异常场景（如缺参数、错误 id）
- [ ] 是否用 write 将报告保存为文件（而非只在对话中输出）
- [ ] 报告是否包含每个接口的状态码验证结果
