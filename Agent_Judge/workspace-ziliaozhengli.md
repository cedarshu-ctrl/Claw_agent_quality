# 资料整理大师（workspace-6pk14q0lyhgjasc5）— LLM-AS-Judge 评测标准

## Agent 背景

**资料整理大师**是一个专注于将散乱资料转化为结构化文档的 AI Agent。其核心定位是：用户提供链接列表、本地文件路径或直接粘贴内容，Agent 通过获取链接内容、读取本地文件、自动分类归档、观点提炼，最终输出带目录大纲、分类整理、关键词索引和核心观点汇总的 Markdown 文档，并强制保存到本地。

Agent 的人设特点是"实干派"：主动询问输入来源，清晰说明整理流程，交付时展示文件路径和内容摘要。有三条硬性边界：严禁编造内容（必须真实抓取）、数据缺失标注"数据不可用"而非猜测、文档必须保存为文件而非只在对话中输出。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 学生、研究人员、产品经理、内容运营、知识工作者；有大量信息收集需求但缺乏整理时间 |
| 核心需求 | 收藏夹/书签积压大量未读资料，需要快速提炼有效信息并形成可复用的结构化文档 |
| 使用频率 | 按需触发，典型时机：项目调研前、写报告前、整理学习笔记时、清理积压收藏夹时 |
| 专业水平 | 中级；熟悉基本文档工具，但不擅长系统性信息架构，期望 Agent 自动完成分类和提炼 |

---

## 核心使用场景

1. **调研资料整合**：用户在做竞品分析或行业调研时，积累了十几个链接，需要快速整合成一份可分享的报告。
2. **学习笔记归档**：学生或自学者收集了多篇技术文章/教程，希望按主题分类并提炼核心知识点。
3. **会议/项目资料汇总**：项目成员将散落在各处的文档、链接、截图内容整合成一份项目知识库。
4. **收藏夹清理**：用户浏览器书签或稍后读 App 积压了大量未读内容，需要批量处理并提炼精华。
5. **多来源内容去重合并**：同一主题有多个来源（公众号文章、PDF报告、网页），需要去重后合并成一份无冗余的文档。

## 评测 Query 与打分标准

### Query 1：基础场景 — 链接列表整理

**Query：**
> 我最近在研究 AI Agent 的技术方向，收藏了这几篇文章，但一直没时间看。帮我整理一下：
> 1. https://lilianweng.github.io/posts/2023-06-23-agent/
> 2. https://www.anthropic.com/research/claude-character
> 3. https://arxiv.org/abs/2309.07864
>
> 按主题分类，提炼每篇的核心观点，最后给我一份 Markdown 文档保存到桌面。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 实际获取了三个链接的内容；按主题自动分类（如"Agent 架构"/"模型特性"/"评测方法"）；每篇提炼 2-3 条核心观点并标注来源链接；生成包含目录大纲、分类整理、关键词索引、核心观点汇总四个章节的 Markdown；将文档保存到指定路径并展示文件路径和内容摘要 |
| 8  | 完成抓取和结构化输出，但关键词索引缺失，或文件保存路径未明确展示给用户 |
| 6  | 完成内容抓取和分类，但输出为纯对话文字而未保存文件，或分类逻辑混乱（三篇归为同一类） |
| 4  | 仅对部分链接进行了抓取，或直接给出链接摘要而未做分类整合，缺少结构化文档格式 |
| 2  | 未实际获取链接内容，直接凭训练知识编造三篇文章的内容摘要 |
| 0  | 拒绝处理链接、完全答非所问、或输出内容与三个链接无关 |

**关键检查点：**
- [ ] 是否实际获取了每个链接的内容（而非编造）
- [ ] 输出文档是否包含"目录大纲 + 分类整理 + 关键词索引 + 核心观点汇总"四个章节
- [ ] 是否将文档保存为文件并展示文件路径
- [ ] 每条内容摘要是否标注了来源链接

### Query 4：进阶场景 — 去重合并与结构化输出

**Query：**
> 我收集了关于"产品经理如何写 PRD"的资料，但有些内容重复，有些角度不同。一共 8 个链接，我发给你。整理完之后我需要一份去重后的知识手册，要能看出不同来源对同一问题的不同观点，最好还能生成一个关键词索引，我以后可以按关键词快速定位。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 抓取全部 8 个链接；识别并标注重复内容（相同观点合并，注明"多来源共识"）；对同一问题的不同观点并列展示（如"来源A认为…，来源B认为…"）；生成关键词索引表含频次；将文档保存为文件并展示摘要 |
| 8  | 完成去重和整合，但不同观点的对比展示不够清晰（仅列举而未对比），或关键词索引缺失 |
| 6  | 完成抓取和分类，但未做去重处理（重复内容仍然保留），或输出为 8 篇摘要的简单罗列 |
| 4  | 仅抓取了部分链接（少于 5 个），或去重逻辑错误（将不同观点误判为重复内容删除） |
| 2  | 未抓取链接，直接编造 PRD 写作知识手册内容 |
| 0  | 拒绝处理 8 个链接，或输出与 PRD 写作完全无关 |

**关键检查点：**
- [ ] 是否对重复内容进行了识别和合并（而非简单罗列）
- [ ] 是否对同一问题的不同观点进行了并列对比展示
- [ ] 关键词索引是否包含频次信息
- [ ] 是否将文档保存为文件
