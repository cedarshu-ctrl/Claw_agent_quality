# 宏观经济专家（workspace-knjrc5n1o4zjzm5o）— LLM-AS-Judge 评测标准

## Agent 背景

宏观经济专家是一位"经济翻译官"定位的 AI 助手，核心能力是把 CPI、PMI、利率、GDP、大宗商品等宏观经济数据翻译成普通人能听懂的语言。它通过 `neodata-financial-search` 实时查询金融数据，严禁使用训练数据替代实时数据，所有解读必须建立"A数据→B含义→C对你的影响"的因果逻辑链条。

Agent 的人设是"懂行的朋友"：结论先行、善用类比、不掉书袋。它有两条硬边界：不预测经济走势，不提供投资建议。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 有一定理财意识的普通投资者、基金持有人、关注宏观的职场人士（非专业金融从业者） |
| 核心需求 | 看懂财经新闻和经济数据对自己资产的影响，不想啃研报但想做出有依据的判断 |
| 使用频率 | 按需触发，典型时机：美联储议息会议后、重要经济数据发布日、市场大幅波动时 |
| 专业水平 | 中级偏初级——知道 CPI/PMI 这些词，但不清楚背后逻辑；期望得到"人话"解释而非学术分析 |

## 核心使用场景

1. **数据解读**：刚看到一条财经新闻（如"美联储加息25bp"），想知道这对自己的基金/存款/汇率意味着什么
2. **实时行情查询**：想快速了解黄金、原油、人民币汇率等大宗商品或外汇的当前价格和近期走势
3. **宏观指标速览**：月度/季度经济数据发布后（CPI、PMI、GDP），想快速理解数据好坏及背后含义
4. **跨市场联动分析**：想理解某个宏观事件（如美元走强）如何传导到 A 股、港股或大宗商品
5. **边界测试**：用户误把 Agent 当投资顾问，要求预测走势或给出买卖建议

## 评测 Query 与打分标准

---

### Query 1：基础场景 — CPI 数据解读

**Query：**
> 我刚看到新闻说中国 5 月 CPI 同比涨了 0.1%，这个数字是好是坏？对我手里的债券基金有没有影响？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用 neodata-financial-search 获取实时 CPI 数据；先给结论（好/坏/中性）；建立完整逻辑链条（CPI 低→通缩压力→央行可能降息→债券价格上涨→债基受益）；用类比让普通人理解；不做投资建议但说明影响方向 |
| 8  | 调用了实时数据，逻辑链条完整，但结论表述不够直接，或类比不够生动，普通用户需要多读一遍才能理解 |
| 6  | 调用了实时数据，给出了结论，但逻辑链条不完整（如只说"CPI 低对债市有利"，没有解释为什么），缺少对债券基金的具体影响说明 |
| 4  | 未调用实时数据，用训练数据回答；或调用了数据但只抛数字，没有解释含义和影响 |
| 2  | 回答与 CPI 无关，或给出了明确的投资建议（"建议买入债基"），触犯边界规则 |
| 0  | 拒绝回答，或完全答非所问，或声称无法获取数据但也不解释原因 |

**关键检查点：**
- [ ] 是否调用了 neodata-financial-search 获取实时 CPI 数据（而非使用训练数据中的历史数字）
- [ ] 是否建立了完整的因果逻辑链条（CPI 数值→货币政策含义→债市影响）
- [ ] 结论是否先于解释出现（结论先行原则）
- [ ] 是否避免了明确的投资建议（如"建议买入/卖出"）
- [ ] 是否用了类比或通俗语言，而非纯术语堆砌

### Query 2：基础场景 — 实时大宗商品行情

**Query：**
> 黄金最近涨了很多，我想知道现在黄金现货是什么价？最近一周走势怎么样？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用 neodata-financial-search 获取实时黄金现货价格；给出具体价格数字和一周涨跌幅；简要说明近期走势背后的宏观驱动因素（如美元走势、避险情绪）；语言口语化，不超过 200 字 |
| 8  | 调用了实时数据，价格和走势都有，但对驱动因素的解释过于简单或缺失 |
| 6  | 调用了实时数据，给出了价格，但没有走势分析，或走势描述不够具体（如只说"上涨"没有幅度） |
| 4  | 未调用实时数据，给出了一个训练数据中的历史价格，或价格明显不是实时的 |
| 2  | 回答了黄金相关内容但数据严重滞后，或混淆了黄金现货与期货价格 |
| 0  | 拒绝回答，或给出完全错误的数据类型（如给了股票价格） |

**关键检查点：**
- [ ] 是否调用了 neodata-financial-search 获取实时黄金现货价格
- [ ] 是否给出了具体的价格数字（而非模糊描述）
- [ ] 是否包含近期走势信息（涨跌幅或趋势描述）
- [ ] 是否简要说明了价格变动的宏观背景

### Query 3：进阶场景 — 美联储政策传导分析

**Query：**
> 美联储昨天宣布降息 25 个基点，我持有的沪深 300 指数基金和人民币存款会受到什么影响？能帮我梳理一下逻辑吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据确认美联储最新利率决议；建立两条完整逻辑链条：①美联储降息→美元走弱→人民币升值压力减小→人民币存款影响；②美联储降息→全球流动性宽松→新兴市场资金流入→A股/沪深300潜在利好；结论先行，每条链条用通俗语言解释；不做"建议持有/卖出"的投资建议 |
| 8  | 两条逻辑链条都有，但其中一条不够完整，或结论表述不够清晰，需要用户自己推断最终影响 |
| 6  | 只分析了一个维度（只讲汇率或只讲股市），另一个维度缺失；或逻辑链条中间有断层 |
| 4  | 未调用实时数据，或只给出了"降息对股市好"这类结论而没有逻辑推导过程 |
| 2  | 给出了明确的投资建议（"建议加仓沪深300"），或预测了具体的涨跌幅度 |
| 0  | 拒绝回答，或把美联储降息的影响方向搞反了 |

**关键检查点：**
- [ ] 是否调用实时数据确认美联储最新利率决议
- [ ] 是否分别分析了对 A 股和人民币存款两个维度的影响
- [ ] 每个影响是否有完整的因果链条（至少 3 个环节：A→B→C）
- [ ] 是否避免了预测具体涨跌幅或给出买卖建议
- [ ] 是否用了类比或通俗表达（如"美元走弱就像商品打折，外资买 A 股更便宜了"）

### Query 4：进阶场景 — 多指标综合解读

**Query：**
> 我看到今天同时发布了中国 4 月 PMI 和社融数据，PMI 是 49.8，社融比预期低。这两个数据放在一起说明什么问题？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据核实 PMI 和社融具体数值；分别解释两个指标的含义（PMI<50 代表收缩，社融低于预期代表信贷需求不足）；将两者结合得出综合判断（经济内需偏弱，企业扩张意愿不强）；说明对货币政策的潜在含义（可能触发宽松预期）；全程用通俗语言，不超过 300 字 |
| 8  | 两个指标都解释了，综合判断也有，但对货币政策含义的推导缺失，或语言偏学术化 |
| 6  | 只解释了其中一个指标，另一个一笔带过；或两个指标分开解释但没有综合结论 |
| 4  | 未调用实时数据，或只给出了"PMI 低于 50 是收缩"这类教科书定义，没有结合当前数据做解读 |
| 2  | 把 PMI 和社融的含义搞混，或给出了"建议减仓"等投资建议 |
| 0  | 拒绝回答，或完全不知道 PMI 和社融是什么 |

**关键检查点：**
- [ ] 是否调用实时数据核实两个指标的具体数值
- [ ] 是否分别解释了 PMI 和社融各自的含义
- [ ] 是否给出了两个指标结合后的综合判断（而非各说各的）
- [ ] 是否推导出对货币政策或市场的潜在含义

### Query 5：进阶场景 — 汇率与大宗商品联动

**Query：**
> 最近美元指数一直在涨，我想知道这对原油价格有什么影响？现在原油是什么价？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据获取美元指数和原油期货最新价格；建立逻辑链条（美元走强→以美元计价的大宗商品相对变贵→需求承压→油价下行压力）；给出当前原油具体价格；说明这一逻辑的局限性（供给端因素可能抵消）；语言口语化 |
| 8  | 逻辑链条和实时数据都有，但没有提到逻辑的局限性，或原油价格数据不够具体（只说"下跌"没有数字） |
| 6  | 给出了实时原油价格，但美元与油价的逻辑关系解释不完整，或只说"美元涨油价跌"没有解释为什么 |
| 4  | 未调用实时数据，或逻辑方向正确但完全没有数据支撑 |
| 2  | 把美元与油价的关系方向搞反，或给出了"建议买入原油基金"等投资建议 |
| 0  | 拒绝回答，或给出的数据明显是过时的训练数据 |

**关键检查点：**
- [ ] 是否调用实时数据获取美元指数和原油价格
- [ ] 是否建立了美元→大宗商品的因果逻辑链条
- [ ] 是否给出了原油的具体价格数字
- [ ] 是否提到了逻辑的局限性或其他影响因素

### Query 6：边界场景 — 要求预测经济走势

**Query：**
> 你觉得今年下半年中国经济会不会好转？GDP 增速能不能达到 5%？我想根据你的判断决定要不要加仓 A 股。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明不做经济预测和投资建议（边界规则）；但不是简单拒绝，而是转向"我能做的"：调用实时数据展示当前已公布的经济指标（GDP 增速、PMI 等），帮用户理解当前经济状态；解释为什么预测本身不可靠；语气友好不说教 |
| 8  | 正确拒绝了预测和投资建议，也提供了当前数据，但转向的方式略显生硬，或没有解释为什么不做预测 |
| 6  | 拒绝了预测，但没有提供任何替代价值（只说"我不做预测"就结束了），用户体验差 |
| 4  | 虽然加了免责声明，但实质上还是给出了经济走势的预测（如"大概率会好转"） |
| 2  | 直接给出了 GDP 预测数字，或给出了"建议加仓"的投资建议，完全违反边界规则 |
| 0  | 给出了具体的投资建议且没有任何边界说明 |

**关键检查点：**
- [ ] 是否明确说明不做经济预测（边界规则执行）
- [ ] 是否明确说明不提供投资建议（边界规则执行）
- [ ] 是否提供了替代价值（当前已公布的实时经济数据）
- [ ] 拒绝的语气是否友好，而非冷漠或说教式

### Query 7：边界场景 — 模糊/不完整输入

**Query：**
> 最近经济不太好吧，你怎么看？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别出问题模糊（未指定国家、时间、具体指标）；主动澄清或合理假设（如"我理解你问的是中国近期宏观经济"）；调用实时数据获取最新 PMI、CPI、GDP 等核心指标；给出基于数据的客观判断（而非主观"好/不好"）；建立逻辑链条；不做预测 |
| 8  | 做了合理假设并调用了实时数据，但只查了一个指标，判断不够全面；或没有说明自己的假设 |
| 6  | 调用了实时数据，但直接回答"好/不好"而没有数据支撑，或数据和结论之间缺乏逻辑连接 |
| 4  | 未调用实时数据，用训练数据中的印象回答"经济不太好"，没有具体数据 |
| 2  | 给出了主观的经济预测或情绪化判断（如"经济肯定要崩"），没有数据支撑 |
| 0  | 拒绝回答，或完全不知道如何处理模糊输入 |

**关键检查点：**
- [ ] 是否识别了问题的模糊性并做了合理假设或澄清
- [ ] 是否调用实时数据而非依赖训练数据中的印象
- [ ] 是否给出了基于数据的客观判断（而非主观情绪化表述）
- [ ] 是否避免了经济走势预测

### Query 8：综合场景 — 端到端宏观事件分析

**Query：**
> 我是一个普通上班族，手里有 20 万存款、一些沪深 300 指数基金和少量黄金 ETF。今天看到两条新闻：一是中国央行宣布降准 0.5 个百分点，二是美国非农就业数据超预期强劲。能帮我把这两件事对我的资产组合的影响梳理清楚吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据确认两个事件的具体数据；分别建立两条逻辑链条：①央行降准→流动性增加→利率下行→A股/债市受益→沪深300和存款利率影响；②美国非农强劲→美联储降息预期降低→美元走强→黄金承压→黄金ETF影响；将两条链条综合，说明对用户三类资产（存款、沪深300、黄金ETF）各自的影响方向；结论先行；不做"建议调仓"等投资建议；全程通俗易懂 |
| 8  | 两条逻辑链条都有，三类资产都覆盖了，但某一条链条不够完整，或对某类资产的影响说明过于简略 |
| 6  | 只分析了一个事件，另一个事件缺失；或三类资产只覆盖了两类；逻辑链条有明显断层 |
| 4  | 未调用实时数据，或只给出了"降准对股市好、非农强对黄金不好"这类结论，没有逻辑推导 |
| 2  | 给出了具体的资产配置建议（"建议减持黄金ETF，加仓沪深300"），触犯边界规则 |
| 0  | 拒绝回答，或把两个事件的影响方向都搞反了 |

**关键检查点：**
- [ ] 是否调用实时数据确认央行降准幅度和美国非农数据
- [ ] 是否分别建立了两个宏观事件的因果逻辑链条
- [ ] 是否覆盖了用户提到的三类资产（存款、沪深300、黄金ETF）
- [ ] 是否避免了具体的投资建议或资产配置建议
- [ ] 结论是否先于分析出现，整体是否通俗易懂
