# 腾讯问卷设计专家（workspace-atzf3s5lkk875riy）— LLM-AS-Judge 评测标准

## Agent 背景

腾讯问卷设计专家是一个专注于 wj.qq.com 平台的问卷全生命周期助手，核心定位是"从 0 到 1 帮用户搞定用户调研"。它能根据用户的调研目的自动规划问卷结构、编写题目、配置跳转逻辑，并通过 tencent-survey MCP 工具直接在平台上创建、查看、修改问卷，以及拉取和汇总回答数据。

Agent 的人设是一位高效、直接的用研专家：不讲方法论，不废话，先理解目的，再给出完整方案并立即执行。它只操作腾讯问卷平台，不涉及其他问卷服务，不提供统计分析方法论，不生成涉及隐私或歧视性内容。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 产品经理、运营、HR、市场调研人员、高校学生，以及需要快速收集意见的团队负责人 |
| 核心需求 | 快速设计出结构合理、回收率高的问卷，并能直接在腾讯问卷平台上发布，省去手动逐题录入的时间 |
| 使用频率 | 按需触发，典型时机：新功能上线前做用户调研、活动结束后收集反馈、HR 做员工满意度调查、课题组做问卷数据收集 |
| 专业水平 | 中低水平——知道自己想问什么，但不熟悉问卷设计方法论；期望 Agent 直接给出可用的完整问卷，而非教他们怎么设计 |

---

## 核心使用场景

1. **新功能用户调研**：产品经理在功能灰度后需要快速收集用户体验反馈，告诉 Agent 功能名称和关注点，期望直接拿到可发布的问卷链接。
2. **活动满意度收集**：运营在活动结束后需要评估用户满意度，提供活动背景，期望 Agent 生成包含 NPS 或满意度量表的问卷。
3. **已有问卷的题目修改**：用户发现某道题表述不清或选项缺失，提供问卷链接，期望 Agent 直接帮改好。
4. **问卷回答数据汇总**：问卷回收一段时间后，用户想了解整体情况，期望 Agent 拉取全部回答并给出简明汇总。
5. **问卷诊断与优化**：用户已有一份问卷但回收率低，期望 Agent 指出题目设计问题并给出改进建议。

## 评测 Query 与打分标准

### Query 1：基础场景 — 从零创建用户调研问卷

**Query：**
> 我是一个产品经理，我们的 App 刚上线了一个 AI 写作助手功能，想收集用户的使用体验反馈。主要想了解：用户用了多久、觉得好不好用、最喜欢哪个功能点、有没有遇到问题。帮我设计一份问卷，直接帮我创建好，给我发布链接。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行创建流程：(1) 先进行鉴权检查（setup.sh）；(2) 按调研目的规划 4 个核心模块（使用时长、满意度、功能偏好、问题反馈），题目类型合理（单选/多选/量表/开放题混用）；(3) 使用 DSL 纯文本格式调用 create_survey 创建问卷（scene=1）；(4) 返回完整投放链接 `wj.qq.com/s2/{survey_id}/{hash}` |
| 8  | 完成创建并给出链接，但题目设计有小瑕疵（如全部用单选题、缺少开放性问题，或选项未覆盖"遇到问题"模块） |
| 6  | 给出了完整的题目设计方案（DSL 文本或题目列表），但未实际调用工具创建问卷，只是"建议"用户自己去创建 |
| 4  | 给出了部分题目，但结构混乱、题目类型单一，或遗漏了用户明确提出的调研维度（如没有"遇到问题"相关题目） |
| 2  | 仅给出泛泛的问卷设计建议或方法论介绍，没有具体题目，也没有调用工具 |
| 0  | 拒绝回答、答非所问，或声称无法操作腾讯问卷平台 |

**关键检查点：**
- [ ] 是否执行了鉴权检查（提及 setup.sh 或 TENCENT_SURVEY_TOKEN）
- [ ] 是否调用了 create_survey 工具（而非仅输出题目文本）
- [ ] 是否返回了 `wj.qq.com/s2/{survey_id}/{hash}` 格式的投放链接
- [ ] 题目是否覆盖了用户提出的 4 个维度（使用时长、好不好用、功能偏好、遇到问题）
- [ ] 是否使用了多种题目类型（单选、多选、量表或开放题至少 2 种）

### Query 4：进阶场景 — 员工满意度调查（结构化交付物）

**Query：**
> 我是 HR，公司有 200 名员工，年底要做一次全员满意度调查。需要覆盖：薪酬福利、工作环境、团队氛围、职业发展、管理层沟通这五个维度。每个维度要有量表题，最后要有一道开放题让大家写建议。帮我设计并创建这份问卷。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 执行鉴权检查；按五个维度分模块设计问卷，每个维度至少 1 道量表题（如 5 分制李克特量表）；末尾包含 1 道开放题；使用 DSL 格式调用 create_survey（scene=1）创建；返回投放链接；题目总数合理（10-20 题） |
| 8  | 完成创建并给出链接，但部分维度缺失量表题（改用单选题替代），或开放题缺失，或题目数量过少（少于 8 题） |
| 6  | 给出了完整的五维度题目设计文本，但未调用工具实际创建问卷 |
| 4  | 只覆盖了部分维度（少于 3 个），或题目类型全部为单选，没有量表题 |
| 2  | 给出了通用的满意度调查模板介绍，没有针对用户指定的五个维度定制题目 |
| 0  | 拒绝操作，或生成了与满意度调查完全无关的内容 |

**关键检查点：**
- [ ] 是否覆盖了全部五个维度（薪酬福利、工作环境、团队氛围、职业发展、管理层沟通）
- [ ] 每个维度是否包含量表题（而非全部单选题）
- [ ] 是否包含末尾开放题
- [ ] 是否实际调用 create_survey 并返回链接
- [ ] 题目总数是否在合理范围（8-25 题）

### Query 5：进阶场景 — 修改已有问卷题目

**Query：**
> 我有一份课程满意度问卷，链接是 https://wj.qq.com/s2/29876543/xyz789。第 3 题是问"课程内容是否满足你的需求"，选项只有"满足"和"不满足"，太简单了。帮我改成 5 个选项的量表题，从"完全不满足"到"完全满足"。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 执行鉴权检查；先调用 get_survey 获取问卷详情，从 pages→questions 中找到第 3 题的 question_id；按 DSL 语法编写新的量表题文本（5 个选项：完全不满足、不太满足、一般、比较满足、完全满足）；调用 update_question 更新；可选再次调用 get_survey 确认结果 |
| 8  | 完成更新，但量表选项文字与用户描述略有出入（如只写了 4 个选项，或选项顺序颠倒），或未做最终确认 |
| 6  | 正确获取了 question_id 并给出了 DSL 文本，但未实际调用 update_question 执行更新 |
| 4  | 直接尝试调用 update_question 但未先调用 get_survey 获取 question_id，导致参数错误 |
| 2  | 给出了如何手动在腾讯问卷后台修改题目的操作步骤，没有调用工具 |
| 0  | 拒绝操作，或修改了错误的题目（非第 3 题） |

**关键检查点：**
- [ ] 是否先调用 get_survey 获取 question_id（而非自行构造）
- [ ] 是否调用了 update_question 工具
- [ ] 新题目是否包含 5 个选项且覆盖"完全不满足"到"完全满足"的完整量表
- [ ] DSL 文本中选项是否不含字母前缀（写"完全不满足"而非"A. 完全不满足"）

### Query 6：进阶场景 — 问卷回答数据汇总分析

**Query：**
> 我的产品反馈问卷已经收了一段时间了，问卷 ID 是 12345678。帮我把所有回答都拉出来，告诉我一共收了多少份，每道题的回答情况大概是什么样的。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 执行鉴权检查；调用 list_answers 获取首页数据；如果 list.length == per_page 则循环翻页直到获取全部数据；汇报总回答数（total 字段）；对每道题的回答做简明汇总（选择题给出各选项频次，开放题列举代表性回答） |
| 8  | 正确获取了全部数据并给出总数，但题目汇总不完整（如只汇总了部分题目，或开放题未列举任何回答内容） |
| 6  | 只获取了第一页数据（未翻页），基于不完整数据给出了汇总，但未说明数据可能不完整 |
| 4  | 调用了 list_answers 但未解析回答内容，只返回了原始数据结构或仅报告了总数 |
| 2  | 未调用工具，仅描述了如何在腾讯问卷后台查看数据 |
| 0  | 拒绝操作，或声称无法获取回答数据 |

**关键检查点：**
- [ ] 是否执行了翻页逻辑（当 list.length == per_page 时继续请求）
- [ ] 是否报告了总回答数（total 字段）
- [ ] 是否对每道题的回答做了汇总（而非仅返回原始数据）
- [ ] 翻页时是否正确使用了 last_answer_id 作为游标参数

### Query 8：边界场景 — 模糊/不完整的输入

| 分数 | 标准描述 |

### Query 9：综合场景 — 端到端问卷调研全流程

**Query：**
> 我是一个电商平台的运营，双十一活动刚结束，我想做一个购物体验调研。需要了解：用户对活动力度是否满意、物流速度评价、客服体验、以及有没有遇到什么问题。帮我设计并创建问卷，等我发给你回答数据后，再帮我分析一下哪些方面需要改进。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 第一阶段：执行鉴权检查，按四个维度（活动力度、物流速度、客服体验、问题反馈）设计问卷，使用量表题+开放题组合，调用 create_survey（scene=1）创建，返回投放链接；同时明确告知用户"收到回答数据后可以帮你分析"，为第二阶段做好铺垫 |
| 8  | 完成了问卷创建并给出链接，但未主动说明后续分析流程，或四个维度中有 1 个遗漏 |
| 6  | 给出了完整的题目设计方案，但未实际创建问卷；或创建了问卷但未覆盖全部四个维度 |
| 4  | 只完成了问卷设计部分，对"分析改进建议"的后续需求完全没有回应或规划 |
| 2  | 给出了通用的电商调研建议，没有针对双十一场景定制，也没有调用工具 |
| 0  | 拒绝操作，或只回答了问题的一半（如只讲分析不做问卷） |

**关键检查点：**
- [ ] 是否覆盖了全部四个维度（活动力度、物流速度、客服体验、问题反馈）
- [ ] 是否使用了量表题（而非全部单选题）
- [ ] 是否实际调用 create_survey 并返回投放链接
- [ ] 是否对后续数据分析阶段有明确的说明或承诺
- [ ] 是否执行了鉴权检查
