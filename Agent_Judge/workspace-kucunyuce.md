# 库存预测专家（workspace-6cy3zeqov5e2g7zy）— LLM-AS-Judge 评测标准

## Agent 背景

库存预测专家是一位深耕中国电商供应链的库存管理老兵，核心定位是"数据驱动库存决策，精准预测不缺不压"。其核心能力覆盖需求预测建模（移动平均/指数平滑/ARIMA）、安全库存计算（基于服务水平目标）、补货策略优化（ROP+EOQ）、大促备货倒排计划（618/双11/年货节）、SKU生命周期管理，以及多渠道库存协同分配。

Agent 的人设特点是严谨务实、数据驱动、风险意识强：不说"我觉得"，只说"根据近三个月数据"；预测结果必须附带置信区间；大促备货量不超过预测值的1.3倍；库存策略必须同步给采购、仓储、财务三个部门。

## 核心使用场景

1. **大促备货规划**：双11/618前6-8周，运营团队需要确定各SKU的备货量和入仓时间节点，防止爆单断货或备货过多积压。
2. **日常补货决策**：某SKU库存告急或即将触发再订货点，需要快速判断补多少、什么时候补、找哪个供应商。
3. **库存健康度诊断**：周期性盘点库存状态，识别呆滞库存、高周转风险SKU，制定清仓或调仓策略。
4. **新品首批备货**：新品上市前无历史数据，需要基于类比品类推算首批备货量，控制试销风险。
5. **多渠道库存分配**：同一批货需要在天猫、京东、抖音、拼多多等多渠道之间合理分配，避免某渠道断货而另一渠道积压。

## 评测 Query 与打分标准

### Query 1：基础场景 — 日常补货量计算

**Query：**
> 我们有个爆款洗面奶，SKU编号是C-088，最近三个月日均销量是150件，供应商交货周期通常是12天，但上个月有一次延迟到了18天。我们的目标服务水平是98%。现在仓库里还剩1,800件，请帮我算一下安全库存应该设多少，再订货点是多少，这次应该补多少货？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确给出安全库存计算过程（基于98%服务水平对应的Z值≈2.05，结合交货周期波动）；给出具体的再订货点数值（ROP = 日均销量 × 平均交货周期 + 安全库存）；给出补货量建议并说明依据；附带置信区间或误差说明；提示供应商交货延迟风险，建议按最长交货周期18天计算缓冲 |
| 8  | 计算逻辑正确，给出了安全库存和ROP的具体数值，但未说明Z值来源或置信区间，或未提及供应商延迟风险 |
| 6  | 给出了安全库存和ROP的数值，但计算方法简化（如直接用最大交货周期而非统计方法），缺少对98%服务水平的专业处理 |
| 4  | 只给出了部分计算结果（如只算了安全库存，没给ROP或补货量），或数值明显有误 |
| 2  | 给出了模糊建议（如"多备一些"、"按两周销量备货"），没有基于数据的具体计算 |
| 0  | 拒绝回答、答非所问，或给出"精确到个位数"的虚假精度而不附任何置信区间 |

**关键检查点：**
- [ ] 是否明确引用了98%服务水平对应的Z值（约2.05）或等效统计方法
- [ ] 是否给出了具体的安全库存数值（而非仅描述公式）
- [ ] 是否给出了再订货点（ROP）的具体数值
- [ ] 是否提示了供应商交货延迟风险，并建议按最长周期18天计算缓冲
- [ ] 预测结果是否附带了置信区间或误差范围说明

### Query 2：基础场景 — 库存健康度诊断

**Query：**
> 帮我看看我们店的库存状况。现在有326个在售SKU，总库存金额280万，上周销售额42万。其中A类SKU（52个）平均周转天数18天，B类SKU（128个）平均周转天数38天，C类SKU（146个）平均周转天数65天，呆滞库存（90天无动销）占总库存的8.5%。我想知道现在库存健康不健康，有哪些需要马上处理的问题？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出结构化的库存健康度看板，包含各指标与目标值的对比（周转天数目标<30天、呆滞占比目标<5%）；明确标注各指标状态（达标/偏高/超标）；给出优先级排序的行动项（至少3条具体措施，如C类呆滞清仓、B类周转优化）；计算出整体库存周转天数（280万/42万×7≈46.7天）并与行业标准对比 |
| 8  | 给出了健康度评估和主要问题识别，行动建议具体，但缺少结构化表格或指标状态标注，或未计算整体周转天数 |
| 6  | 识别出了呆滞库存超标的问题，给出了清仓建议，但对B类周转偏高的问题未深入分析，行动项不够具体 |
| 4  | 只做了简单描述（"呆滞库存有点高"），没有与目标值对比，没有具体行动建议 |
| 2  | 回答流于表面，没有利用提供的数据做任何计算或对比分析 |
| 0  | 拒绝回答，或给出与库存管理无关的内容 |

**关键检查点：**
- [ ] 是否计算了整体库存周转天数并与行业目标（<30天）对比
- [ ] 是否对A/B/C三类SKU的周转状态分别给出评价
- [ ] 是否明确指出呆滞库存8.5%超标（目标<5%），并给出具体清仓建议
- [ ] 行动项是否按优先级排序，且每条都有具体可执行的措施

### Query 3：进阶场景 — 双11大促备货倒排计划

**Query：**
> 现在是9月中旬，我们要备战双11。主推SKU是一款爆款面膜，去年双11卖了28,000件，今年我们的天猫店铺权重提升了，预计增长30%。供应商在广州，生产周期10天，发货到菜鸟无锡仓需要3天，入仓操作需要2天。我们在华东、华南、华北三个仓都有货。请帮我制定一个完整的双11备货倒排计划，包括备货量、各仓分配比例和关键时间节点。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 给出预测备货量（约36,400件，并附置信区间，不超过预测值1.3倍即47,320件上限）；输出完整的时间线表格，包含T-45天到D-Day的关键节点、具体日期、关键动作和负责方；给出三仓分配比例建议（参考华东60%/华南25%/华北15%或类似合理分配）；提示风险点（如供应商产能、入仓排期拥堵）；说明预售期/尾款期/返场期的分阶段入仓策略 |
| 8  | 备货量计算合理，时间线完整，但三仓分配比例缺少依据说明，或未提及分阶段入仓策略（预售/尾款/返场） |
| 6  | 给出了备货量和主要时间节点，但时间线不完整（缺少T-40天供应商下单或T-20天入仓完成等关键节点），或未给出三仓分配方案 |

**关键检查点：**
- [ ] 备货量是否附带置信区间，且不超过预测值的1.3倍（约47,320件）
- [ ] 时间线是否包含至少5个关键节点（T-45天到D-Day），并标注具体日期
- [ ] 是否给出三仓分配比例及依据
- [ ] 是否提示供应商产能或入仓排期的风险点
- [ ] 是否区分了预售期/尾款期/返场期的分阶段备货策略

### Query 4：进阶场景 — 新品首批备货量估算

**Query：**
> 我们下个月要上一款新品精华液，完全没有历史销售数据。但我们有一款类似的老款精华液，上市第一个月卖了1,200件，第二个月2,800件，第三个月稳定在3,500件左右。新品的定价比老款高20%，功效宣传更强，我们计划在天猫做一波开屏广告投放。请帮我估算新品第一个月的备货量，以及试销期的库存策略。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明类比预测方法的局限性和置信度；基于老款数据给出新品首月预测区间（而非单点值）；考虑定价溢价（高20%可能抑制转化）和广告投放（提升流量）两个相反因素的综合影响；给出试销期小批量快速补货策略（首批建议量+补货触发条件）；建议设置销售里程碑（如首周/首月达到X件则追加补货） |
| 8  | 给出了合理的预测区间和试销期策略，但对定价溢价和广告投放的影响分析不够深入，或未给出具体的补货触发条件 |
| 6  | 给出了首批备货量建议，但只是简单类比老款数据（如"参考老款首月1,200件备1,500件"），缺少对影响因素的分析和试销期动态补货策略 |
| 4  | 只给出了一个数字，没有区间，没有分析，没有试销期策略 |
| 2  | 建议首批大量备货（如直接按老款成熟期3,500件备货），忽视新品试销风险 |
| 0  | 拒绝回答（以"没有历史数据"为由），或给出精确到个位数的虚假精度 |

**关键检查点：**
- [ ] 是否明确说明类比预测的置信度限制（数据不足时必须说明）
- [ ] 预测结果是否给出区间而非单点值
- [ ] 是否分析了定价溢价（抑制转化）和广告投放（提升流量）两个相反因素
- [ ] 是否给出试销期小批量快速补货策略，而非一次性大量备货
- [ ] 是否建议设置销售里程碑作为追加补货的触发条件

### Query 5：进阶场景 — 多渠道库存分配策略

| 分数 | 标准描述 |
|------|---------|

**关键检查点：**
- [ ] 是否输出了包含优先级、数量/比例、分配依据的结构化表格
- [ ] 是否针对抖音直播爆单波动预留了弹性应急库存
- [ ] 是否给出了各渠道独立安全库存设置建议
- [ ] 是否说明了共享库存池的使用规则（超出独立安全库存部分进入共享池）
- [ ] 是否建议临期/尾货优先分配拼多多或折扣渠道

### Query 6：边界场景 — 数据不足时的处理

**Query：**
> 我们是一家刚起步的小店，上个月才开始卖货，总共卖了300件。我想让你帮我预测一下今年双11能卖多少，然后告诉我应该备多少货。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明仅1个月数据（远低于最低6个月要求）导致预测置信度极低，无法给出可靠的双11预测；不给出虚假精确的预测数字；提供替代方案（如参考同品类行业数据、竞品表现、平台大促倍率均值）；给出低风险的试探性备货建议（小批量+快速补货机制）；建议先积累3-6个月数据再做系统性预测 |
| 8  | 明确说明了数据不足的问题和置信度限制，给出了替代方案，但备货建议过于保守或激进，缺少具体的小批量+快速补货策略 |
| 6  | 说明了数据不足，但仍然给出了一个"参考性"的预测数字，且没有充分强调置信度极低的风险 |
| 4  | 忽视数据不足的问题，直接基于1个月数据推算双11预测量，给出看似精确的数字 |
| 2  | 给出了精确到百位甚至十位的双11预测量，完全没有提及数据不足的问题 |
| 0  | 给出"精确到个位数"的虚假精度预测，或建议按预测值1.3倍以上大量备货（违反边界规则） |

**关键检查点：**
- [ ] 是否明确说明1个月数据不满足最低6个月的数据要求
- [ ] 是否明确标注了预测置信度极低
- [ ] 是否拒绝给出虚假精确的预测数字（如"双11能卖X件"）
- [ ] 是否提供了可行的替代方案（行业数据参考、小批量试探性备货）
- [ ] 是否建议先积累足够数据再做系统性预测

### Query 8：综合场景 — 端到端大促复盘与下一轮备货优化

**Query：**
> 我们刚刚结束了618大促，来复盘一下。备货情况：主推SKU A001面膜备了25,000件，实际卖了18,500件，还剩6,500件库存；SKU B003精华液备了8,000件，卖了9,200件，大促第三天就断货了，损失了至少2,000件的销售机会；SKU C012洁面乳备了5,000件，只卖了1,800件，现在有3,200件积压。供应商方面，A001的供应商这次交货延迟了5天。请帮我做一个完整的618复盘报告，并给出双11备货策略的优化建议。

**打分标准（10 分制）：**
