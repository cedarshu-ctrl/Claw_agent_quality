# 趋势研究员（workspace-jozgatymcjihpo6a）— LLM-AS-Judge 评测标准

## Agent 背景

趋势研究员是一位专注于行业趋势分析与技术前瞻的市场情报专家，核心定位是"先人一步看到机会"。其能力覆盖三大领域：趋势追踪（信号识别、生命周期判断）、竞品与市场分析（功能对比、差异化机会、威胁评估）、技术前瞻（成熟度评估、技术组合预判、产品影响分析）。

该 Agent 的标志性交付物是结构化的**趋势分析报告**，包含摘要、趋势概述、关键数据表、主要玩家矩阵、机会与威胁分析、建议行动时间线、风险与不确定性、信息来源八个模块。其核心人设特点是：区分"炒作"与"真趋势"、承认不确定性、用概率思维而非非黑即白、长期主义导向。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 产品经理、战略分析师、创业公司创始人、技术负责人，需要为产品方向或投资决策提供情报支撑 |
| 核心需求 | 在趋势变成共识之前提前感知，避免错过时间窗口或押注错误方向 |
| 使用频率 | 每周/按需；典型触发时机：季度规划前、竞品发布重大功能后、新技术引发行业讨论时 |
| 专业水平 | 中级到专家；有一定行业背景，但缺乏系统性情报收集和分析能力，期望得到有数据支撑、可直接用于决策的结构化报告 |

## 核心使用场景

1. **季度战略规划**：产品团队在季度 OKR 制定前，需要了解未来 6-12 个月行业最值得关注的技术趋势，以决定是否启动预研项目。
2. **竞品动态监控**：某竞品发布了新功能或完成融资，需要快速评估其战略意图、市场影响，以及我方应对策略。
3. **新技术可行性评估**：工程团队对某项新技术感兴趣，需要判断其成熟度、落地成本和适用场景，再决定是否投入资源。
4. **行业报告解读**：收到一份 Gartner 或 McKinsey 报告，需要提炼出与自身业务最相关的洞察和行动建议。
5. **趋势炒作识别**：某个概念在社交媒体上突然爆火，需要判断这是真实趋势还是短期炒作，避免盲目跟风。

## 评测 Query 与打分标准

### Query 1：基础场景 — 趋势生命周期判断

**Query：**
> 我是一家 SaaS 公司的产品总监，最近 AI Agent 这个词到处都是，我们 CEO 也在问要不要做。但我不确定这到底是真趋势还是又一波炒作。你能帮我分析一下 AI Agent 现在处于什么阶段，值不值得我们现在就投入资源？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确给出 AI Agent 的生命周期阶段判断（萌芽/成长/成熟/衰退）并附理由；提供至少 3 个可量化的关键数据指标（如 GitHub star 增长、YC 入选比例、企业采纳率等）；区分"炒作信号"与"真实投入信号"（如真金白银投入、生产环境案例）；给出明确的信心等级（高/中/低）；最终给出具体的行动建议（如"建议 2 人 2 周 PoC"而非"可以考虑研究一下"） |
| 8  | 有生命周期判断和数据支撑，但数据来源不够具体（如只说"增长很快"而未引用具体来源），或行动建议缺乏资源估算 |
| 6  | 给出了趋势判断，但缺少关键数据表格或信心等级标注，建议停留在"可以关注"层面，未给出可操作的下一步 |
| 4  | 只是泛泛描述 AI Agent 是什么，没有生命周期判断，没有数据，没有针对该公司的具体建议 |
| 2  | 直接给出"AI Agent 是未来"或"现在太早了"的非黑即白结论，无数据支撑，无不确定性标注 |
| 0  | 拒绝分析、答非所问，或给出"我无法预测未来"后不提供任何实质性分析 |

**关键检查点：**
- [ ] 是否明确标注了 AI Agent 的生命周期阶段（萌芽/成长/成熟/衰退）
- [ ] 是否提供了至少 2 个可量化的数据指标（数值+来源+趋势方向）
- [ ] 是否区分了"炒作信号"与"真实投入信号"
- [ ] 是否标注了信心等级（高/中/低）并给出理由
- [ ] 是否给出了具体可操作的行动建议（含资源估算或时间线）

### Query 3：进阶场景 — 完整趋势分析报告

**Query：**
> 我们公司在做企业级知识管理产品，老板让我下周给董事会汇报"RAG 技术对我们产品的影响"。我需要一份完整的趋势分析报告，要能说清楚 RAG 现在处于什么阶段、主要玩家在做什么、对我们有什么机会和威胁，以及我们接下来应该怎么做。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 严格按照 SKILL.md 模板输出完整报告，包含：3 句话摘要、趋势概述（定义+驱动因素+生命周期阶段+信心等级）、关键数据表（至少 3 行，含指标/数值/来源/趋势）、主要玩家矩阵（至少 3 家，含差异化和值得关注的动作）、机会与威胁分析（各至少 2 条，含影响程度和时间窗口）、建议行动时间线表（现在/3个月/6个月三档）、风险与不确定性（至少 2 条，含概率描述）、信息来源列表 |
| 8  | 报告结构完整，但某 1-2 个模块内容较薄（如主要玩家只列了 1-2 家，或建议行动缺少投入估算） |
| 6  | 有报告框架但缺少关键模块（如没有关键数据表，或没有建议行动时间线），内容以文字描述为主，缺少结构化表格 |
| 4  | 输出了一些关于 RAG 的信息，但没有按报告模板组织，缺少对"企业知识管理产品"的针对性分析 |
| 2  | 只是解释了 RAG 是什么技术，没有趋势分析、没有竞品分析、没有行动建议 |
| 0  | 拒绝输出报告，或输出内容与 RAG/知识管理完全无关 |

**关键检查点：**
- [ ] 是否包含 3 句话执行摘要（这是什么趋势、当前阶段、对我们意味着什么）
- [ ] 是否包含关键数据表（指标/数值/来源/趋势四列）
- [ ] 是否包含主要玩家矩阵（公司/产品策略/差异化/值得关注的动作四列）
- [ ] 是否包含机会与威胁分析（各含影响程度和时间窗口）
- [ ] 是否包含建议行动时间线表（现在/3个月/6个月三档，含投入估算）
- [ ] 是否标注了信心等级和不确定性

### Query 4：进阶场景 — 技术成熟度与落地可行性评估

**Query：**
> 我们工程团队最近在讨论要不要用多模态大模型来做产品里的图片理解功能。技术上感觉可行，但我不知道这个技术现在到底成熟到什么程度，落地成本怎么样，有没有坑。你能帮我评估一下吗？我需要一个能拿去和 CTO 讨论的分析。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 评估多模态大模型的技术成熟度（含 Gartner Hype Cycle 定位或等效判断）；分析适用场景与局限性（图片理解的具体能力边界）；给出落地成本估算（API 费用、工程投入、维护成本等维度）；列举已知的技术坑或风险点（至少 3 个）；给出明确的落地建议（是否推荐、推荐路径、PoC 方案）；标注信心等级和不确定性 |

| 0  | 拒绝分析，或输出内容与多模态技术评估无关 |

**关键检查点：**
- [ ] 是否给出了技术成熟度判断（含所处阶段的理由）
- [ ] 是否分析了图片理解场景的具体能力边界和局限性

- [ ] 是否列举了至少 2 个具体的技术风险或已知坑
- [ ] 是否给出了可操作的 PoC 方案或落地路径建议

### Query 5：进阶场景 — 竞品功能矩阵与差异化机会

**Query：**
> 我在做一款面向独立开发者的 AI 编程助手，准备融资，投资人问我们和 GitHub Copilot、Cursor、Windsurf 有什么差异化。我自己用过这几个产品，但没有系统分析过。你能帮我做一个竞品功能对比，找出我们可以切入的差异化机会吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出结构化竞品功能矩阵（至少覆盖 GitHub Copilot、Cursor、Windsurf 三家，维度包含核心功能、定价策略、目标用户、差异化特点）；识别至少 3 个竞品的共同弱点或空白领域；针对"独立开发者"这一细分用户给出具体的差异化切入点建议；评估每个机会的可行性和竞争壁垒；标注哪些是基于公开信息的事实、哪些是分析推断 |
| 8  | 竞品矩阵完整，差异化机会识别有深度，但针对"独立开发者"的细分分析不够具体，或可行性评估较浅 |
| 6  | 有竞品对比，但以文字描述为主，缺少结构化矩阵；差异化机会停留在"可以做 X"层面，缺少可行性分析 |
| 4  | 只是分别介绍了三个竞品的功能，没有横向对比，没有差异化机会识别 |
| 2  | 只列举了竞品名称和简单描述，没有实质性的对比分析 |
| 0  | 拒绝分析，或因"不了解这些产品"而不提供任何分析 |

**关键检查点：**
- [ ] 是否输出了结构化的竞品功能矩阵（表格形式，多维度对比）
- [ ] 是否识别了竞品的共同弱点或市场空白（至少 2 个）
- [ ] 是否针对"独立开发者"细分用户给出了具体的差异化建议
- [ ] 是否评估了差异化机会的可行性或竞争壁垒
- [ ] 是否区分了事实信息和分析推断

### Query 6：边界场景 — 不确定性与薄弱证据处理

**Query：**
> 我听说量子计算快要商业化了，有人说 3 年内就会颠覆现有的加密体系。我们公司做网络安全产品，你觉得我们现在需要开始准备量子安全方案吗？这个趋势是真的吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确区分"量子计算商业化"和"量子计算威胁加密体系"两个不同问题；给出量子计算当前实际成熟度的客观评估（含具体数据，如量子比特数、错误率等）；明确标注"3 年内颠覆加密"这一说法的证据强度（薄弱/中等/充分）；给出概率化的时间线预判（如"主流加密被威胁的概率：5 年内 <5%，10 年内 20-30%"）；给出务实的行动建议（如"现在关注 NIST 后量子密码标准进展，不需要立即大规模投入"）；体现"不追热点、多数据源交叉验证"的研究纪律 |
| 8  | 有客观的成熟度评估和概率化判断，但时间线预判不够具体，或行动建议过于保守/激进 |
| 6  | 指出了"3 年内颠覆"的说法存疑，但缺少具体数据支撑，建议停留在"需要关注"层面 |
| 4  | 直接回答"需要/不需要准备"，没有对证据强度的评估，没有概率化思维 |
| 2  | 跟随"量子计算快要来了"的炒作叙事，给出夸大的威胁评估，或完全否定量子计算的威胁 |
| 0  | 拒绝分析，或给出"我无法预测技术发展"后不提供任何实质性判断 |

**关键检查点：**
- [ ] 是否明确标注了"3 年内颠覆加密"这一说法的证据强度（薄弱/中等/充分）
- [ ] 是否给出了量子计算当前成熟度的客观数据（而非仅凭印象判断）
- [ ] 是否使用了概率化表述（而非非黑即白的结论）
- [ ] 是否体现了"不追热点、至少观察 3 个月再下结论"的研究纪律
- [ ] 是否给出了务实的、与当前证据强度匹配的行动建议

### Query 8：综合场景 — 端到端战略情报支持

**Query：**
> 我是一家做 B2B 销售 SaaS 的创始人，我们产品帮销售团队做客户跟进和商机管理。最近 AI SDR（AI 销售开发代表）这个概念很火，我看到好几家公司都在做，融资也很多。我需要你帮我做一个完整的分析：这个趋势是真实的还是炒作？主要玩家在做什么？对我们现有产品是威胁还是机会？我们应该怎么应对？我下周要和投资人开会，需要展示我们对市场的理解。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出完整的趋势分析报告（严格按 SKILL.md 模板）；趋势真实性判断有数据支撑（融资数据、产品采纳率、用户反馈等）并区分炒作成分；竞品矩阵覆盖至少 3 家 AI SDR 公司（含功能/定价/差异化）；针对"B2B 销售 SaaS"的具体场景分析机会（整合/差异化/防御）和威胁（替代风险/时间窗口）；建议行动时间线表（现在/3个月/6个月）含具体投入估算；给出适合投资人会议使用的核心叙事框架（1-2 句话的市场定位表述）；全程标注信心等级和不确定性 |
