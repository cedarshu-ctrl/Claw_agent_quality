# AI论文速读导师（workspace-rjtygtk52wdjdqjn）— LLM-AS-Judge 评测标准

## Agent 背景

AI论文速读导师是一个专注于学术论文快速消化的智能助手，核心定位是"5分钟掌握一篇论文精华"。它能通过 arXiv ID 或关键词搜索论文，自动抓取页面提取核心贡献、方法论、实验结果和局限性，并以通俗语言输出结构化阅读笔记（Markdown 文件）。支持多篇论文的横向对比分析，以及 CVPR/ICML/NeurIPS 等顶会动态追踪。

该 Agent 的人设特点是"结构清晰 + 语言通俗"：复杂公式用大白话讲解，对比分析用表格呈现，所有输出必须保存为文件而非仅在对话中展示。严禁编造数据，缺失信息一律标注"数据不可用"。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 研究生、高校科研人员、AI 从业者（算法工程师、研究员）、技术博主 |
| 核心需求 | 快速了解一篇陌生论文的核心贡献，不想花 1-2 小时精读全文；或需要横向对比多篇论文找出方法差异 |
| 使用频率 | 高频（每周 3-5 次），典型触发时机：看到推文/微信群分享的论文链接、准备组会汇报、做文献综述前的快速筛选 |
| 专业水平 | 中级偏上（有一定领域背景，但不一定精通所有子方向）；期望输出既有深度又不晦涩，能直接用于汇报或笔记 |

---

## 核心使用场景

1. **单篇速读**：看到一篇 arXiv 新论文，想在 5 分钟内判断是否值得精读，需要核心贡献 + 方法摘要。
2. **多篇对比**：做文献综述时，需要对比 3-5 篇同主题论文的方法差异，生成对比表格。
3. **顶会追踪**：CVPR/NeurIPS 放榜后，想快速了解某个方向（如 diffusion model）的最新进展。
4. **研究方向梳理**：刚进入一个新领域，需要梳理该方向的论文脉络和发展历程。
5. **组会汇报准备**：明天要汇报一篇论文，需要结构化笔记 + 方法论通俗解读，方便做 PPT。

## 评测 Query 与打分标准

### Query 1：基础场景 — 单篇论文速读（提供 arXiv 链接）

**Query：**
> 帮我读一下这篇论文：https://arxiv.org/abs/2305.10601 。我是做 NLP 的研究生，明天组会要汇报，想快速搞清楚这篇的核心贡献是什么、方法上有什么创新、实验结果怎么样，最好能给我一份可以直接参考的阅读笔记。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 实际获取了真实论文页面内容（而非凭训练数据）；输出包含 📄核心贡献、🔬方法论（通俗解读）、📊实验结果（含具体数值）、⚠️局限性四个模块；产出了可交付的阅读笔记并给出文件路径；语言通俗，非专业读者也能理解方法创新点 |
| 8  | 四个模块齐全，阅读笔记已产出，但方法论解读仍偏学术化（未做大白话转换），或实验结果缺少具体数值对比 |
| 6  | 有核心贡献和方法论，但缺少局限性分析，或未产出可交付的阅读笔记（只在对话中输出文字） |
| 4  | 仅给出论文摘要的简单翻译，没有结构化拆解，也没有方法论通俗解读 |
| 2  | 未实际获取论文页面内容，凭已有知识编造或猜测论文内容，或输出与该论文无关 |
| 0  | 拒绝处理、答非所问，或明确编造数据却未标注"数据不可用" |

**关键检查点：**
- [ ] 是否基于实际获取的 arxiv.org/abs/2305.10601 内容回答（而非凭训练数据）
- [ ] 输出是否包含四个标准模块：📄核心贡献 / 🔬方法论 / 📊实验结果 / ⚠️局限性
- [ ] 是否产出了可交付的阅读笔记并给出路径（而非仅在对话中输出）
- [ ] 方法论解读是否使用了通俗语言（避免直接堆砌公式或术语）

### Query 2：基础场景 — 通过关键词搜索最新论文

**Query：**
> 我最近在研究 RAG（检索增强生成），想了解一下 2024 年有哪些比较重要的新进展，能帮我找几篇相关的 arXiv 论文，给我一个简单的概览吗？不需要太深入，主要想知道现在大家在解决什么问题。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 实际搜索了"RAG paper arXiv 2024"等关键词；找到 3 篇以上真实论文（含 arXiv ID）；每篇给出核心问题 + 解决思路的简短概述；输出包含资源链接表格 |
| 8  | 找到 2-3 篇真实论文并给出概述，但资源链接表格缺失 |
| 6  | 给出了 RAG 方向的概述，但论文来源不明确（无 arXiv ID 或链接），或只列标题没有内容摘要 |
| 4  | 仅给出 RAG 技术的通用介绍，没有检索具体论文，没有 2024 年的新进展 |
| 2  | 编造了论文标题或 arXiv ID，或给出的链接无法验证真实性 |
| 0  | 拒绝搜索、完全答非所问，或声称无法访问网络但未说明替代方案 |

**关键检查点：**
- [ ] 是否基于实际搜索结果回答（而非凭训练数据）
- [ ] 给出的论文是否有可验证的 arXiv ID 或链接
- [ ] 是否覆盖了 2024 年的新进展（而非只讲 2020-2022 年的经典工作）
- [ ] 是否给出了论文的可验证链接（arXiv ID 或链接）

### Query 3：进阶场景 — 多篇论文对比分析

**Query：**
> 我在做视觉语言模型（VLM）方向的综述，想对比一下 LLaVA、InstructBLIP 和 MiniGPT-4 这三篇论文的方法差异。主要想搞清楚：它们在视觉编码器的选择、指令微调策略、训练数据上有什么不同？能给我一个对比表格吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 分别获取三篇论文的真实内容；生成对比表格，列维度包含：视觉编码器、指令微调策略、训练数据、核心贡献、优缺点；表格数据来自真实获取而非编造；附行动建议（如"综述写作时可重点关注 X 差异"） |
| 8  | 对比表格结构完整，数据基本准确，但某一篇论文的信息获取不完整（标注了"数据不可用"） |
| 6  | 给出了对比表格，但维度不完整（缺少用户指定的某个对比维度），或三篇论文中有一篇信息明显错误 |
| 4  | 只给出了三篇论文各自的简介，没有横向对比表格，无法直接用于综述写作 |
| 2  | 对比表格中存在编造数据（如虚构的训练数据集名称或参数量），且未标注"数据不可用" |
| 0  | 拒绝处理多篇对比、只处理一篇、或完全混淆三篇论文的内容 |

**关键检查点：**
- [ ] 是否基于实际获取的三篇论文内容回答（而非凭训练数据）
- [ ] 对比表格是否包含用户指定的三个维度：视觉编码器、指令微调策略、训练数据
- [ ] 数据缺失时是否标注"数据不可用"而非猜测填充
- [ ] 是否附有行动建议（综述写作方向）

### Query 4：进阶场景 — 顶会方向追踪与研究脉络梳理

**Query：**
> NeurIPS 2024 刚结束，我做强化学习方向的，想了解一下今年 NeurIPS 在 offline RL 这个子方向上有哪些值得关注的论文？帮我梳理一下这个方向今年的主要进展和研究趋势，最好能给我一份可以存档的报告。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 实际搜索了"NeurIPS 2024 offline RL"等关键词；找到 3 篇以上真实论文；梳理出该方向的 2-3 个研究趋势（如"保守性约束的新方法"、"与 LLM 结合"等）；产出了包含论文列表 + 趋势分析 + 资源链接的完整可存档报告 |
| 8  | 找到真实论文并给出趋势分析，但报告结构不完整（缺少资源链接表格或行动建议），或未产出可存档报告 |
| 6  | 给出了 offline RL 方向的通用介绍，但未能聚焦 NeurIPS 2024 的新论文，或论文来源不明确 |
| 4  | 仅列出几篇论文标题，没有趋势分析，没有研究脉络梳理，无法帮助用户把握方向 |
| 2  | 编造了 NeurIPS 2024 的论文（虚构标题/作者），或将其他年份/会议的论文混入 |
| 0  | 声称无法获取 NeurIPS 2024 信息但未尝试搜索，或完全偏离 offline RL 主题 |

**关键检查点：**
- [ ] 是否基于实际搜索 NeurIPS 2024 + offline RL 的结果回答（而非凭训练数据）
- [ ] 给出的论文是否有可验证来源（arXiv ID 或会议页面链接）
- [ ] 是否提炼出研究趋势（而非只列论文清单）
- [ ] 是否产出了可存档的报告（含执行摘要、论文列表、资源链接、行动建议）

### Query 5：进阶场景 — 通俗解读复杂方法论

**Query：**
> 我是做 CV 的，但对 Transformer 架构不太熟。有人推荐我看 ViT（Vision Transformer）的原始论文，arXiv ID 是 2010.11929。能帮我把这篇论文的核心方法用大白话解释一下吗？特别是 patch embedding 和 self-attention 在图像上怎么工作的，我不需要公式，就想搞懂直觉。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 实际获取了 arxiv.org/abs/2010.11929 真实页面内容（而非凭训练数据）；用类比或生活化比喻解释 patch embedding（如"把图片切成小块，每块变成一个词"）和 self-attention（如"每个小块关注其他小块的相关性"）；避免公式；给出实验结果中的关键数字 |
| 8  | 解释通俗易懂，但类比不够生动（仍有较多术语），或缺少实验结果数字 |
| 6  | 解释了 patch embedding 和 self-attention，但语言仍偏学术，非 CV 背景用户难以理解 |
| 4  | 给出了 ViT 的通用介绍，但没有针对用户"不懂 Transformer"的背景做通俗化处理 |
| 2  | 未实际获取论文页面内容，凭训练数据回答，且解释中存在技术性错误 |
| 0  | 拒绝处理、给出与 ViT 无关的内容，或直接粘贴论文摘要原文而不做任何解读 |

**关键检查点：**
- [ ] 是否基于实际获取的 arxiv.org/abs/2010.11929 内容回答（而非凭训练数据）
- [ ] patch embedding 的解释是否使用了非公式的直觉类比
- [ ] self-attention 的解释是否避免了矩阵运算公式

### Query 6：边界场景 — 输入模糊/信息不完整

**Query：**
> 帮我读一下那篇关于 diffusion model 的论文，就是最近很火的那个，你知道的。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别出输入模糊（未提供具体论文 ID 或标题）；主动向用户确认：研究领域偏好（生成图像/视频/音频）、是否有具体论文链接或 arXiv ID、输出偏好；同时可以提供 2-3 篇候选论文供用户选择（来自真实搜索结果） |
| 8  | 主动澄清了模糊输入，但候选论文列表不够（只给 1 篇），或未询问输出偏好 |
| 6  | 直接选择了一篇"最火"的 diffusion 论文（如 DDPM）开始处理，未澄清用户意图，但处理结果本身质量尚可 |
| 4  | 直接处理了某篇论文，但选择依据不透明，且未告知用户"我选了哪篇" |
| 2  | 编造了一篇"最近很火的 diffusion 论文"，给出虚假的 arXiv ID 或标题 |
| 0  | 拒绝处理模糊输入，或给出完全无关的回答 |

**关键检查点：**
- [ ] 是否识别出输入缺少具体论文标识符（ID/链接/完整标题）
- [ ] 是否主动向用户澄清（而非直接猜测处理）
- [ ] 如果提供候选论文，是否来自真实搜索结果（有可验证链接）
- [ ] 是否避免了编造论文信息

### Query 7：边界场景 — 超出能力范围的请求

**Query：**
> 我有一篇论文的 PDF 文件在我电脑上，路径是 C:/Users/me/papers/my_draft.pdf，这是我自己写的还没发表的草稿，能帮我分析一下这篇论文的逻辑是否严密，实验设计有没有问题吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明该 Agent 的能力边界：专注于 arXiv 已发表论文的速读，无法读取本地 PDF 文件，也不提供论文审稿/逻辑审查服务；建议用户将论文上传到 arXiv 预印本后再使用，或推荐其他适合论文审稿的工具；态度友好，不生硬拒绝 |
| 8  | 说明了无法读取本地文件，但未解释能力边界的原因，或未给出替代建议 |
| 6  | 尝试通过其他方式处理（如让用户粘贴论文内容），但未说明这超出了标准工作流程 |
| 4  | 假装可以处理，给出了一些通用的"论文逻辑审查建议"，但实际上没有读取任何内容 |
| 2  | 声称已读取了本地 PDF 并给出了具体分析（明显编造） |
| 0  | 完全拒绝回应，或给出与请求完全无关的内容 |

**关键检查点：**
- [ ] 是否明确说明无法访问本地文件系统中的 PDF
- [ ] 是否说明该 Agent 专注于 arXiv 已发表论文（而非草稿审阅）
- [ ] 是否给出了有用的替代建议（如上传 arXiv、使用其他工具）
- [ ] 是否避免了编造"已读取本地文件"的虚假分析

### Query 8：综合场景 — 端到端研究综述辅助

**Query：**
> 我是一名博士生，正在写关于"大语言模型推理能力"的综述论文。我需要你帮我：1）找出 2023-2024 年在 arXiv 上关于 LLM reasoning 的 5 篇重要论文；2）对每篇做核心提炼；3）生成一个对比表格，对比它们在推理方法（如 CoT、ToT、RL 等）上的差异；4）最后给我一份完整的综述辅助报告，我可以直接用来整理文献。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 执行完整的 5 步流程：实际搜索找到 5 篇真实论文（含 arXiv ID）→ 逐篇获取真实内容（而非凭训练数据）→ 每篇输出四模块摘要（核心贡献/方法/实验/局限）→ 生成方法对比表格（含推理方法类型、数据集、性能指标）→ 产出完整综述辅助报告（含执行摘要、论文列表、对比表格、资源链接、行动建议）；全程无编造数据 |
| 8  | 完成了 4 个子任务中的 3 个，综述辅助报告已产出，但对比表格维度不完整，或某篇论文信息标注了"数据不可用" |
| 6  | 完成了论文搜索和单篇摘要，但对比表格缺失或质量差，报告结构不完整 |
| 4  | 只完成了论文搜索和列表，没有逐篇摘要，没有对比表格，无法直接用于综述写作 |
| 2  | 论文列表中存在编造的论文（虚假 arXiv ID 或标题），或对比表格数据明显错误 |
| 0  | 只完成了部分子任务（如只找了 2 篇论文），或拒绝处理综合性请求，或未产出可交付的综述辅助报告 |

**关键检查点：**
- [ ] 是否找到了 5 篇有可验证 arXiv ID 的真实论文
- [ ] 每篇论文是否都有四模块结构（核心贡献/方法论/实验结果/局限性）
- [ ] 对比表格是否包含推理方法类型（CoT/ToT/RL 等）这一核心维度
- [ ] 是否产出了可交付的综述辅助报告（含执行摘要、资源链接表格、行动建议）
- [ ] 全程是否无编造数据（缺失信息标注"数据不可用"）
