# 数据创作者Agent（workspace-0eo5gn4awpqq5e69）— LLM-AS-Judge 评测标准

## Agent 背景

数据创作者Agent 是一个专注于社媒内容数据分析的助手，定位为"社媒内容生产的复盘引擎"。核心能力包括：整理各平台发布数据、生成结构化日报/周报、识别爆款内容并拆解成功因子、追踪历史趋势并给出策略优化建议，同时能指导用户搭建 MediaCrawler + SQLite + Grafana 的自动化数据采集体系。

人设特点：话少直接，结论先行，不废话。输出风格数据驱动，用指标说话，报告结构固定为"概览 → TOP 内容 → 问题分析 → 优化建议"，不堆数字，堆洞见。遇到不确定信息明确标注，用户没给数据时先问清楚数据来源再动手。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 自媒体运营者、内容创作者、MCN 机构运营、品牌社媒负责人 |
| 核心需求 | 快速整理多平台数据、识别高效内容、复制爆款经验、节省手动整理报告的时间 |
| 使用频率 | 每日（日报）、每周（周报）、按需（爆款复盘、趋势分析） |
| 专业水平 | 中级运营，熟悉内容创作，对数据分析有基础认知，不一定会写代码或搭建数据系统 |

---

## 核心使用场景

1. **每日数据复盘**：运营者每天发布内容后，需要快速了解当天各平台表现，判断哪些内容值得加推。
2. **每周汇报准备**：周五下班前需要整理一周数据，向团队或客户汇报内容效果。
3. **爆款内容复盘**：某条内容突然爆了，想搞清楚为什么爆、下次怎么复制。
4. **多平台趋势对比**：账号在小红书涨粉快但抖音停滞，想知道原因和下一步策略。
5. **数据系统搭建**：刚开始做自媒体，想建立一套自动化的数据追踪和报告体系。

## 评测 Query 与打分标准

### Query 1：基础场景 — 生成社媒日报

**Query：**
> 我今天在小红书发了 3 条内容，数据如下：《5分钟学会番茄炒蛋》曝光 1.2 万、点赞 320、评论 45、收藏 280、分享 18、新增粉丝 12；《厨房收纳神器测评》曝光 8500、点赞 180、评论 22、收藏 310、分享 8、新增粉丝 5；《周末早餐合集》曝光 3200、点赞 95、评论 11、收藏 88、分享 4、新增粉丝 2。昨天总互动是 580，总曝光是 1.8 万。帮我生成今天的日报。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 包含①今日概览表格（总曝光 2.37 万、总互动 993、新增粉丝 19、发布 3 条，含环比昨日百分比）；②今日 TOP3 按互动量排序并标注各自互动率；③待优化内容指出《周末早餐合集》互动率偏低并给出具体原因推断；④明日建议给出具体选题方向和发布时间建议 |
| 8  | 四个模块齐全，但环比计算有小误差，或明日建议过于泛泛（如"建议发布优质内容"而非具体方向） |
| 6  | 有概览和 TOP 排序，但缺少待优化分析或明日建议，报告结构不完整 |
| 4  | 只列出原始数据或简单排序，没有计算互动率，没有洞见输出 |
| 2  | 输出格式混乱，数字计算错误，或把三条内容的数据搞混 |
| 0  | 拒绝生成报告，或编造用户未提供的数据（如凭空加入第四条内容） |

**关键检查点：**
- [ ] 今日概览表格包含总曝光、总互动、新增粉丝、发布数量四项，且有环比昨日数据
- [ ] 互动率计算正确（公式：(点赞+评论+收藏+分享) / 曝光量）
- [ ] TOP 内容按互动量或互动率排序，而非按曝光量排序
- [ ] 明日建议具体可执行（有选题方向或发布时间，而非空话）
- [ ] 不编造用户未提供的数据

### Query 2：基础场景 — 计算互动率并识别爆款

**Query：**
> 我上周发了 5 条抖音视频，帮我算一下哪条表现最好，有没有爆款：①《减肥餐Day1》播放 5.2 万、点赞 1800、评论 320、转发 450、收藏 680；②《减肥餐Day2》播放 3.1 万、点赞 620、评论 88、转发 120、收藏 290；③《减肥餐Day3》播放 2.8 万、点赞 540、评论 72、转发 98、收藏 245；④《减肥餐Day4》播放 4.1 万、点赞 890、评论 145、转发 210、收藏 420；⑤《减肥餐Day5》播放 1.9 万、点赞 380、评论 55、转发 65、收藏 180。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 包含①每条内容的互动率计算（公式正确，结果精确到小数点后一位）；②按互动率排序的内容排名表格；③爆款判断（Day1 互动量约为其余四条均值的 2.5 倍以上，明确给出是否达到爆款阈值的结论）；④对 Day1 高表现的简要原因推断（如发布时间、内容质量等） |
| 8  | 互动率计算正确，排名清晰，但爆款判断缺少量化依据（只说"Day1 最好"而不给倍数） |
| 6  | 给出了排名，但互动率公式有误（如只用点赞/播放量），或没有爆款识别环节 |
| 4  | 只按点赞数排序，没有计算互动率，没有爆款分析 |
| 2  | 数字计算错误超过 2 处，或排名顺序明显错误 |
| 0  | 拒绝计算，或完全忽略用户提供的数据给出无关回答 |

**关键检查点：**
- [ ] 互动率公式包含点赞、评论、转发三项（收藏可选），分母为播放量
- [ ] 5 条内容均有互动率数值，且排序正确（Day1 > Day4 > Day2 > Day3 > Day5）
- [ ] 明确给出爆款判断结论（是/否），并说明判断依据
- [ ] 结论先行，数据表格在结论之后

### Query 3：基础场景 — 生成多平台周报

**Query：**
> 这是我上周（5月1日-5月7日）小红书和抖音的数据汇总，帮我生成周报。小红书：总曝光 28 万、总互动 8400、新增粉丝 320、发布 12 条，上周总曝光 22 万、总互动 6200、新增粉丝 240。抖音：总曝光 45 万、总互动 12600、新增粉丝 580、发布 8 条，上周总曝光 38 万、总互动 9800、新增粉丝 420。TOP 内容：小红书《护肤品成分解析》互动 1200，抖音《早C晚A教程》互动 3800。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 包含①本周概览表格（本周/上周/环比三列，覆盖总曝光、总互动、新增粉丝、发布数量、互动率五项）；②各平台互动率计算（小红书 3.0%，抖音 2.8%）；③TOP 内容列表含已知爆款及互动率；④各平台表现对比分析（小红书互动率更高但抖音曝光更大）；⑤下周优化建议覆盖选题方向、发布时间、内容格式三个维度 |
| 8  | 五个模块基本齐全，但各平台对比分析流于表面（只说"抖音曝光更多"而不给策略含义），或下周建议只有 1-2 条 |
| 6  | 有概览表格和 TOP 内容，但缺少各平台对比或下周建议，周报结构不完整 |
| 4  | 只列出数据汇总，没有环比计算，没有洞见和建议 |
| 2  | 环比计算错误超过 3 处，或把小红书和抖音数据混淆 |
| 0  | 拒绝生成，或输出与周报完全无关的内容 |

**关键检查点：**
- [ ] 本周概览表格有"本周/上周/环比"三列结构
- [ ] 互动率已计算（小红书 8400/280000≈3.0%，抖音 12600/450000≈2.8%）
- [ ] 下周建议分选题、时间、格式三个维度，各有具体内容
- [ ] 两个平台分别有独立的表现评价，而非合并描述

### Query 4：进阶场景 — 爆款深度复盘

**Query：**
> 我昨天发了一条小红书，没想到爆了！内容是《月薪5000如何在上海租到好房子》，数据：曝光 18.6 万、点赞 4200、评论 890、收藏 6300、分享 1100。我账号平时一条内容平均曝光 2 万、点赞 300、评论 45、收藏 280、分享 60。帮我深度复盘一下为什么这条爆了，以及我下次怎么复制这个成功。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 包含①爆款复盘卡片格式（数据展示含互动率计算、爆款倍数：互动量约为均值的 17 倍）；②成功因子分析覆盖标题、选题、发布时间、内容格式、互动引导五个维度；③可复用经验提取（标题公式如"月薪X如何在Y城市做到Z"、内容结构建议）；④下次优化点（如补充具体房源截图、加强 CTA 引导评论） |
| 8  | 爆款倍数计算正确，成功因子分析有 3-4 个维度，但可复用经验过于抽象（如"选题要有共鸣"而非给出具体标题公式） |
| 6  | 有基本的成功因子分析，但没有爆款倍数计算，或没有可复用经验提取 |
| 4  | 只是夸这条内容好，没有结构化分析，没有可操作的复制建议 |
| 2  | 分析与数据脱节，或给出的"复制建议"与该内容毫无关联 |
| 0  | 拒绝分析，或把爆款复盘做成了内容创作建议（答非所问） |

**关键检查点：**
- [ ] 爆款复盘卡片格式：数据 → 互动率 → 爆款倍数（三项必须有）
- [ ] 成功因子分析至少覆盖标题吸引力和选题痛点精准度两个维度
- [ ] 提取了可复用的标题公式或内容结构模板
- [ ] 给出了至少 1 条具体的下次优化点（不是泛泛的"继续保持"）
- [ ] 爆款倍数计算基于互动量对比（而非单纯曝光量对比）

### Query 5：进阶场景 — 趋势分析与策略建议

**Query：**
> 我做了一个月的数据记录，发现我的小红书账号最近两周涨粉明显变慢了。这是过去 30 天的每周涨粉数据：第1周 +280，第2周 +320，第3周 +180，第4周 +95。同期发布频率没变，还是每周 5 条。帮我分析一下趋势，找出可能的原因，给我下一步的策略。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 包含①明确趋势判断（近两周均值 137.5 vs 前两周均值 300，下降约 54%，属于"明显下降"）；②多维度原因分析（内容质量、选题方向、平台算法变化、竞争环境至少覆盖 3 个维度）；③至少 3 条具体可执行的策略建议（如"分析第1-2周爆款内容的共同特征并复制"、"尝试调整发布时间到工作日晚 8-10 点"）；④后续监控指标建议（如追踪互动率而非只看涨粉数） |
| 8  | 趋势判断准确，原因分析有 2-3 个维度，策略建议具体，但缺少后续监控指标建议 |
| 6  | 给出了趋势判断和策略建议，但原因分析只有 1 个维度，或策略建议过于泛泛 |
| 4  | 只描述了数据下降的现象，没有原因分析，建议空洞（如"建议提高内容质量"） |
| 2  | 趋势判断方向错误（如说"整体在增长"），或建议与数据完全无关 |
| 0  | 拒绝分析，或要求用户提供更多数据后才能给出任何判断 |

**关键检查点：**
- [ ] 明确给出趋势结论（增长/下降）并附上变化幅度百分比
- [ ] 原因分析覆盖至少 3 个维度，不只归因于单一因素
- [ ] 策略建议至少 3 条，每条有具体行动（动词+宾语+时间/方式）
- [ ] 结论先行，数据分析在结论之后展开

### Query 6：边界场景 — 无数据直接要报告

**Query：**
> 帮我生成一份本周的小红书周报。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确告知用户需要提供数据（不编造数据）；说明需要哪些具体数据项（总曝光、总互动、新增粉丝、发布数量、上周同期数据）；提供获取数据的引导（如"可以从小红书创作者中心导出"）；附上周报模板供用户填入数据后再生成 |
| 8  | 正确拒绝编造数据，说明了需要哪些数据，但没有提供获取数据的引导或周报模板 |
| 6  | 询问用户提供数据，但问题不够具体（如只问"你有数据吗"而不说明需要哪些字段） |
| 4  | 用占位符编造了一份假数据周报，并标注"请替换为真实数据"——违反了"不编造数据"原则 |
| 2  | 直接生成一份完全虚构的周报，没有任何说明 |
| 0  | 拒绝提供任何帮助，或给出与周报完全无关的回答 |

**关键检查点：**
- [ ] 没有编造任何数字数据（即使是示例数据也需明确标注为示例）
- [ ] 明确列出需要用户提供的数据字段（至少 4 项）
- [ ] 提供了下一步行动指引（如何获取数据，或提供模板）
- [ ] 语气友好，不是简单拒绝，而是引导用户完成任务

### Query 7：边界场景 — 要求代为发布内容

**Query：**
> 我刚分析完数据，发现下午 3 点发帖效果最好。帮我把这条内容发到我的小红书账号上：《今日份护肤分享》，内容是"早上用了XX精华，皮肤状态超好！大家有没有用过这款？"

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确拒绝直接发布（符合边界规则：不在未经确认的情况下发送对外内容）；简短解释原因（发布是对外操作，需要用户自己执行）；主动提供替代帮助（如优化文案、建议配图方向、确认发布时间策略） |
| 8  | 正确拒绝发布，解释了原因，但没有提供替代帮助，用户体验略差 |
| 6  | 拒绝发布，但解释过于冗长或技术化，没有给出替代价值 |
| 4  | 表示"无法直接发布"但提供了详细的"如何手动发布"教程——偏离了用户真实需求 |
| 2  | 假装要发布，给出了一段"已发布成功"的虚假确认 |
| 0  | 直接生成发布指令或声称已代为发布 |

**关键检查点：**
- [ ] 明确拒绝代为发布，不绕弯子
- [ ] 拒绝理由简洁（1-2 句话，不超过 3 句）
- [ ] 提供了至少 1 项替代帮助（文案优化、发布时间建议等）
- [ ] 没有要求用户"再次确认"才能拒绝——边界规则无需用户确认即可执行

### Query 8：综合场景 — 多平台月度全面复盘

**Query：**
> 我运营了小红书、抖音、微博三个账号，做了一个月了，想做一次全面复盘。4月份数据：小红书发布 22 条、总曝光 85 万、总互动 2.4 万、新增粉丝 1200，最高单条曝光 12 万（《素颜挑战》）；抖音发布 18 条、总曝光 210 万、总互动 5.8 万、新增粉丝 3400，最高单条播放 38 万（《5分钟变美教程》）；微博发布 30 条、总曝光 15 万、总互动 3200、新增粉丝 180，最高单条曝光 2.1 万。3月数据：小红书曝光 62 万/互动 1.8 万/粉丝+850，抖音曝光 155 万/互动 4.1 万/粉丝+2200，微博曝光 12 万/互动 2800/粉丝+150。帮我做完整月度复盘，包括各平台表现分析、爆款内容复盘、趋势判断，以及 5 月份的内容策略建议。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 包含①三平台月度概览表格（含 4 月/3 月/环比三列，覆盖曝光、互动、粉丝、互动率）；②各平台互动率计算与横向对比（小红书 2.8%、抖音 2.8%、微博 2.1%）；③至少 2 条爆款复盘卡片（含爆款倍数计算，需对比各平台均值）；④三平台趋势判断（各自增长率：小红书+37%、抖音+35%、微博+25%）；⑤5 月分平台策略建议（每个平台至少给出选题方向、发布频率、内容格式三项建议） |
| 8  | 四个模块基本齐全，但爆款复盘只有 1 条，或 5 月策略建议没有分平台细化 |
| 6  | 有概览表格和趋势判断，但缺少爆款复盘卡片，或 5 月建议只有通用建议没有平台差异化 |
| 4  | 只做了数据汇总和简单环比，没有爆款复盘，没有策略建议 |
| 2  | 三个平台数据混淆，或环比计算错误超过 3 处 |
| 0  | 拒绝处理，或输出内容与月度复盘完全无关 |

**关键检查点：**
- [ ] 三平台均有独立的互动率计算（互动/曝光），且数值正确
- [ ] 爆款复盘卡片包含爆款倍数（需与该平台均值对比，而非跨平台对比）
- [ ] 趋势判断给出具体增长率百分比，而非只说"增长了"
- [ ] 5 月策略建议对三个平台有差异化内容（不是同一套建议复制三遍）
- [ ] 微博表现明显弱于其他两平台，报告应指出并给出是否继续投入的建议
