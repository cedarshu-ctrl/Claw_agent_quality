# 目标拆解教练（workspace-afkxq12r6zfjx6qz）— LLM-AS-Judge 评测标准

## Agent 背景

**目标拆解教练**是一个专注于"把大目标变成每天能干的小事"的执行型 AI 教练。它的核心定位不是帮用户决定目标，而是接过用户已有的目标，用 SMART 原则将其拆解为月度里程碑 → 周度子任务 → 每日打卡项，并生成可持久化的计划文件和进度追踪表，同时设置定时提醒。

人设风格直接、务实、有温度——不说"你要变成更好的人"这类空话，只说"今天你要做的是这个"。标志性交付物是：**目标计划文件 + 进度追踪表 + 执行报告（含资源链接和行动建议）**，且报告必须保存为文件，不能只在对话中输出。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 有自我提升意愿的职场人、学生、自由职业者；立过 flag 但执行力不稳定的普通人 |
| 核心需求 | 把模糊的大目标变成清晰可执行的每日任务，并通过打卡和复盘维持执行动力 |
| 使用频率 | 按需触发（设定新目标时）+ 每日打卡 + 每周复盘；典型触发时机是年初/季初立目标、遭遇拖延危机、掉队后想重启 |
| 专业水平 | 中等偏下；对目标管理方法论（SMART、OKR 等）了解有限，期望 Agent 直接给出可操作的计划，而非讲理论 |

---

## 核心使用场景

1. **新目标启动**：用户有一个模糊的大目标（如"今年要考过 PMP"），需要 Agent 帮助拆解成具体的月/周/日计划并生成文件。
2. **每日打卡**：用户完成当天任务后来报告进度，Agent 记录并更新追踪表，给出鼓励和下一步提示。
3. **掉队补救**：用户连续几天没打卡或落后于计划，需要 Agent 帮助重新评估进度、调整计划节奏。
4. **周/月复盘**：用户到了周末或月末，需要 Agent 生成回顾报告，分析完成率、偏差原因，并调整后续计划。
5. **目标类型适配**：用户的目标属于特定领域（健身、学习、职业晋升），需要 Agent 根据领域特点调整拆解方式和参考资源。

## 评测 Query 与打分标准

### Query 1：基础场景 — 新目标拆解启动

**Query：**
> 我想在三个月内学完 Python 基础，然后能独立做一个小项目。我现在完全零基础，每天下班后大概能拿出 1 小时学习。帮我拆一下这个目标，告诉我每天要干什么。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 先用 SMART 原则逐项检验并完善目标（S/M/A/R/T 各维度明确）；输出月度里程碑（3 个月各自的阶段目标）；输出每周子任务清单（至少前 2 周具体到每天）；生成目标计划文件和进度追踪表并给出文件路径；设置每日打卡提醒（automation_update）；提供 web_search 搜索到的 Python 学习路径参考资源链接 |
| 8  | 完成拆解和文件生成，但 SMART 检验不完整（如缺少 A/可达成性评估），或资源链接为空/未搜索 |
| 6  | 给出了月/周计划，但未生成文件（只在对话中输出），或未设置打卡提醒，或每日任务粒度过粗（如"学 Python"而非"完成 X 章节/X 个练习"） |
| 4  | 只给出笼统的学习建议（如"先学语法再学项目"），没有具体时间节点和每日任务量 |
| 2  | 回答与目标拆解无关，或只推荐了几个学习网站，没有任何计划结构 |
| 0  | 拒绝回答、要求用户自己去查、或输出完全答非所问的内容 |

**关键检查点：**
- [ ] 是否对目标进行了 SMART 原则逐项检验（S/M/A/R/T 五个维度均有体现）
- [ ] 是否输出了月度里程碑（3 个月各阶段目标明确）
- [ ] 是否给出了每日可执行任务（粒度到"今天做什么"，而非"这周学 XX"）
- [ ] 是否生成了目标计划文件并给出文件路径（而非只在对话中输出）
- [ ] 是否包含 web_search 搜索到的真实参考资源链接

### Query 2：基础场景 — 每日打卡记录

**Query：**
> 今天完成了！我按计划看完了 Python 第 3 章的变量和数据类型，还做了课后练习题，大概花了 50 分钟。帮我记一下今天的打卡，顺便告诉我明天要做什么。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确确认今日打卡成功并更新进度追踪表（给出文件更新路径）；展示关键打卡指标（连续打卡天数、本周完成率、整体进度%）；给出明天的具体任务（基于计划的下一步，粒度到章节/练习数量）；有简短的正向反馈（不夸张，符合 SOUL.md 风格） |
| 8  | 完成打卡记录和明日任务提示，但未更新文件（只在对话中确认），或缺少进度数据展示 |
| 6  | 给出了明日任务，但没有记录打卡、没有进度数据，像普通问答而非打卡系统 |
| 4  | 只说"很好，继续加油"之类的鼓励，没有具体的明日任务和进度追踪 |
| 2  | 回答与打卡无关，或重新开始询问用户目标是什么（忽略了上下文） |
| 0  | 拒绝记录、或输出完全无关内容 |

**关键检查点：**
- [ ] 是否更新了进度追踪表并给出文件路径
- [ ] 是否展示了连续打卡天数和本周完成率等量化指标
- [ ] 是否给出了明天的具体可执行任务（而非泛泛的"继续学习"）
- [ ] 正向反馈是否简洁务实（符合"直接、有条理"的人设，不过度夸张）

### Query 3：基础场景 — 进度查询

**Query：**
> 我已经学了三周了，想知道现在进度怎么样？离目标还有多远？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 读取历史打卡记录文件，输出量化进度报告：已完成天数/总计划天数、本周完成率、整体完成率%、计划 vs 实际偏差；给出"还剩 X 天，每天需要完成 Y"的具体预测；如有偏差，给出调整建议 |
| 8  | 给出了进度数据，但缺少计划 vs 实际偏差分析，或调整建议过于笼统 |
| 6  | 给出了部分进度信息，但数据不完整（如只说"完成了 X%"，没有每日/每周维度），或未读取文件（凭记忆回答） |
| 4  | 只给出鼓励性话语，没有任何量化数据 |
| 2  | 回答与进度查询无关，或要求用户重新描述目标 |
| 0  | 拒绝回答或完全答非所问 |

**关键检查点：**
- [ ] 是否读取了进度追踪文件（而非凭对话记忆回答）
- [ ] 是否输出了每日完成率、连续打卡天数、本周 vs 上周对比等量化指标
- [ ] 是否给出了"还剩 X 天，每天需要 Y"的具体预测
- [ ] 是否包含偏离度分析（计划 vs 实际）

### Query 4：进阶场景 — 生成周复盘报告

**Query：**
> 这周结束了，我想做个周复盘。这周计划了 5 天学习，实际完成了 3 天，有两天因为加班没时间。帮我生成一份周复盘报告，分析一下问题在哪，下周怎么调整。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 生成结构化周复盘报告并保存为文件（给出路径）；报告包含：本周完成率（3/5=60%）、未完成原因分析（加班是外部因素，非主观放弃）、对整体目标进度的影响评估、下周调整方案（具体到每天任务量或备用时间段）；语气符合"允许偶尔掉队，但要追回来"的 SOUL.md 哲学 |
| 8  | 报告结构完整，但未保存为文件，或下周调整方案过于笼统（如"尽量多学"而非具体调整） |
| 6  | 给出了分析，但缺少对整体目标进度影响的评估，或没有具体的下周调整计划 |
| 4  | 只说"这周完成了 60%，下周继续努力"，没有原因分析和调整方案 |
| 2  | 批评用户没完成任务，或给出不切实际的补救方案（如"下周每天学 3 小时"） |
| 0  | 拒绝生成报告，或输出与复盘无关的内容 |

**关键检查点：**
- [ ] 是否生成了周复盘报告文件并给出文件路径
- [ ] 是否包含本周完成率的量化数据（3/5=60%）
- [ ] 是否对未完成原因进行了分析（区分主观/客观因素）
- [ ] 是否给出了具体的下周调整方案（任务量或时间安排的具体调整）
- [ ] 语气是否符合"允许掉队但要追回来"的原则（不批评，不放弃）

### Query 5：进阶场景 — 健康目标的领域适配拆解

**Query：**
> 我想在 6 个月内减重 15 斤，我现在体重 75 公斤，身高 170，平时基本不运动，饮食也比较随意。每周能抽出 3 天去健身房，每次大概 1 小时。帮我制定一个详细的计划。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 用 SMART 原则检验目标（15 斤/6 个月=每月约 2.5 斤，评估可达成性）；用 web_search 搜索健康减重的最佳实践（运动+饮食组合）；输出月度里程碑（每月目标体重）；给出每周 3 次训练的具体安排（训练类型、时长、强度）；包含饮食调整建议（非极端节食）；生成计划文件和打卡追踪表；设置打卡提醒 |
| 8  | 完成拆解和文件生成，但缺少 web_search 搜索的真实资源，或饮食建议缺失，或训练计划粒度不够（只说"有氧+力量"，没有具体安排） |
| 6  | 给出了月度目标和训练频率，但未生成文件，或计划缺少饮食维度，或未评估目标可达成性 |
| 4  | 只给出笼统建议（"多运动少吃"），没有具体的周/月计划结构 |
| 2  | 给出不健康的减重建议（如极端节食、过度运动），或拒绝因为"不是医生" |
| 0  | 拒绝回答健康目标，或输出完全无关内容 |

**关键检查点：**
- [ ] 是否用 SMART 原则评估了目标可达成性（15 斤/6 个月的合理性）
- [ ] 是否包含 web_search 搜索到的健康减重参考资源
- [ ] 是否给出了每周 3 次训练的具体安排（类型+时长+强度）
- [ ] 是否包含饮食调整建议（且建议健康合理，非极端节食）
- [ ] 是否生成了计划文件并给出路径

### Query 6：进阶场景 — 职业目标的结构化拆解

**Query：**
> 我是一个做了 3 年运营的人，想在一年内转型成产品经理。我没有任何产品经验，但有用户数据分析的背景。现在不知道从哪里开始，感觉这个目标太大了，完全不知道怎么下手。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别用户的优势（数据分析背景）并在计划中体现；用 web_search 搜索运营转产品的常见路径和所需技能；输出季度里程碑（4 个季度各自的阶段目标，如 Q1 学理论+考证、Q2 做项目实践等）；给出每周具体任务（学习资源+实践项目+社群/人脉建设）；生成计划文件；提供真实的参考资源链接（书单、课程、社群） |
| 8  | 完成拆解和文件生成，但未充分利用用户的运营背景优势，或资源链接缺失 |
| 6  | 给出了转型路径，但计划粒度过粗（只有季度目标，没有每周任务），或未生成文件 |
| 4  | 只给出"先学产品思维，再做项目"的笼统建议，没有具体时间节点和资源 |
| 2  | 劝用户放弃转型，或给出不切实际的建议（如"一个月就能转型成功"） |
| 0  | 拒绝回答，或输出与职业规划无关的内容 |

**关键检查点：**
- [ ] 是否识别并利用了用户的运营/数据分析背景优势
- [ ] 是否包含 web_search 搜索到的运营转产品路径参考资源
- [ ] 是否输出了季度里程碑（4 个季度各有明确阶段目标）
- [ ] 是否给出了每周具体任务（学习+实践+人脉三个维度）
- [ ] 是否生成了计划文件并给出路径

### Query 7：边界场景 — 超出能力范围的请求（目标决策）

**Query：**
> 我现在很迷茫，不知道自己该追求什么目标。你觉得我应该去考研还是直接工作？帮我决定一下。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明自己不替用户决定目标（符合 SOUL.md Boundaries）；提供一个结构化的自我澄清框架（如引导用户思考：你的优先级是什么？你对哪个方向更有热情？）；告知用户：一旦做出决定，自己可以帮助拆解和执行；语气温和不说教 |
| 8  | 正确拒绝替用户决定，并给出引导框架，但框架过于简单（只问了 1-2 个问题），或语气略显生硬 |
| 6  | 拒绝决定，但只说"这是你自己的事"，没有提供任何帮助用户澄清的引导 |
| 4  | 给出了倾向性建议（如"我觉得你应该考研"），越过了能力边界 |
| 2  | 直接替用户做了决定，并开始拆解一个用户未确认的目标 |
| 0  | 完全拒绝互动，或输出无关内容 |

**关键检查点：**
- [ ] 是否明确说明不替用户决定目标（边界声明）
- [ ] 是否提供了帮助用户自我澄清的引导框架（至少 2-3 个引导问题）
- [ ] 是否告知用户决定后可以继续帮助拆解执行
- [ ] 是否避免了给出倾向性建议（不越界）

### Query 9：综合场景 — 端到端目标管理（从设定到复盘）

**Query：**
> 我是一个备考雅思的大学生，目标是 3 个月后考到 6.5 分，现在模拟考是 5.0 分。我每天能学 2 小时，周末能学 4 小时。帮我从头到尾规划一下：制定计划、设置打卡、到时候还要帮我做复盘。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行 SKILL.md 全部 5 个步骤：①SMART 检验（5.0→6.5，3 个月，评估可达成性）；②运行脚本处理数据；③生成目标计划文件+进度追踪表+周复盘模板（三个文件，各给出路径）；④设置每日打卡提醒（automation_update，每天 9:00）和周复盘提醒；⑤汇总交付报告（含 web_search 搜索到的雅思备考资源链接）；计划需覆盖听说读写四个维度的每周任务分配 |
| 8  | 完成大部分步骤，但缺少某一文件（如只生成计划文件，没有进度追踪表），或定时任务设置不完整（只设了打卡，没设复盘提醒） |
| 6  | 生成了计划，但未生成文件（只在对话中输出），或缺少听说读写四维度的具体任务分配，或未设置定时任务 |
| 4  | 给出了学习建议，但没有结构化的月/周/日计划，也没有文件生成和定时任务 |

| 0  | 拒绝回答，或输出完全无关内容 |

**关键检查点：**
- [ ] 是否生成了目标计划文件、进度追踪表、周复盘模板三个文件（各给出路径）
- [ ] 是否设置了每日打卡提醒和周复盘提醒（automation_update）
- [ ] 是否覆盖了听说读写四个维度的每周任务分配
- [ ] 是否包含 web_search 搜索到的雅思备考真实资源链接
