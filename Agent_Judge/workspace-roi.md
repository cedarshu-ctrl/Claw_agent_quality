# ROI精算师（workspace-hw3ydg4ci8wkl0ts）— LLM-AS-Judge 评测标准

## Agent 背景

ROI精算师是一个专注于投资回报量化分析的财务决策助手，核心定位是"用数据而不是直觉做商业决策"。它能够接收用户提供的投资参数，通过 Python 脚本自动计算 ROI、NPV、IRR 和回收期，支持多方案横向对比与敏感性分析（变量 ±10%/±20%），最终生成结构化的 Markdown 投资分析报告并保存为文件。

该 Agent 的人设特点是数据驱动、结论直接、流程透明：每个结论都有数字支撑，财务公式公开可查，明确告知计算假设，并在报告末尾附上风险提示。适用场景涵盖设备采购、广告投放、人才招聘、SaaS 订阅、研发立项等多类商业投资决策。

## 核心使用场景

1. **设备采购决策**：企业计划购置一台生产设备，需要评估投资回收期和 ROI，判断是否值得采购。
2. **广告投放 ROI 测算**：市场部门计划投放一批广告，需要在预算有限的情况下评估不同渠道的预期回报。
3. **多项目优先级排序**：手头有 3-4 个备选项目，预算只够做 1-2 个，需要横向对比选出最值得投的方案。
4. **项目立项可行性报告**：向管理层汇报前，需要一份包含 NPV/IRR/敏感性分析的完整可行性分析报告。
5. **敏感性风险评估**：已有初步方案，但关键变量（如销售额、成本）存在不确定性，需要了解各变量波动对 ROI 的影响程度。

## 评测 Query 与打分标准

### Query 1：基础场景 — 单项目 ROI 快速计算

**Query：**
> 我们公司打算买一台 CNC 数控机床，采购价 80 万，预计每年能给我们节省人工和提升产能，折算下来年收益大概 25 万。机器寿命 5 年，我们公司的资金成本大概是 8%。帮我算一下这个投资值不值得做？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整输出四项财务指标（ROI、NPV、IRR、回收期），数值计算正确；明确列出计算假设（折现率 8%、年收益 25 万、周期 5 年）；给出明确的投资建议（"投/不投"）；包含风险提示；报告已保存为文件并告知路径 |
| 8  | 四项指标齐全且计算正确，有明确建议，但缺少风险提示，或未保存文件（只在对话中输出） |
| 6  | 计算了 ROI 和回收期，但缺少 NPV 或 IRR；有基本建议，但未说明计算假设 |
| 4  | 只给出 ROI 百分比，没有 NPV/IRR/回收期；或给出了数字但没有任何投资建议 |
| 2  | 数值计算明显错误（如 ROI = 25/80 = 31.25% 但未考虑时间价值），或只给出公式说明而不计算 |
| 0  | 拒绝计算、要求用户自己算、或编造了与输入不符的财务数据 |

**关键检查点：**
- [ ] ROI、NPV、IRR、回收期四项指标均已计算并展示
- [ ] 折现率 8% 已在参数确认区明确列出
- [ ] 给出了明确的"投/不投"结论，而非模糊表述
- [ ] 包含至少一条具体风险提示（如"收益预测存在不确定性"）
- [ ] 报告已保存为文件

### Query 2：基础场景 — 广告投放 ROI 测算

**Query：**
> 我是一家电商公司的市场经理，下个季度计划在抖音投 50 万广告费，预计能带来 200 万的销售额，毛利率大概 30%。这个广告投放的 ROI 值得做吗？帮我算一下。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 正确区分"销售额 ROI"和"利润 ROI"（毛利润 60 万 - 广告费 50 万 = 净收益 10 万，ROI = 10/50 = 20%）；明确列出计算假设；给出投放建议；包含风险提示（如转化率波动）；报告保存为文件 |
| 8  | ROI 计算正确，有建议，但未区分销售额口径和利润口径，或缺少风险提示 |
| 6  | 计算了基本 ROI，但直接用 200 万/50 万 = 4 倍（未扣除成本），口径错误；或有正确计算但无建议 |
| 4  | 只给出公式，未代入数字计算；或计算结果与输入数据明显不符 |
| 2  | 混淆了 ROI 和 ROAS 概念，给出误导性结论 |
| 0  | 拒绝回答，或编造了用户未提供的数据（如自行假设转化率） |

**关键检查点：**
- [ ] 正确使用毛利润（而非销售额）计算净收益
- [ ] ROI 计算公式（净收益/投资成本）透明可查
- [ ] 给出明确的投放建议（值得/不值得/有条件值得）
- [ ] 包含广告效果不确定性的风险提示

### Query 3：进阶场景 — 多方案横向对比

**Query：**
> 我们公司有三个项目要立项，但预算只够做一个。方案 A：投资 100 万，预计 3 年内每年收益 45 万；方案 B：投资 150 万，预计 5 年内每年收益 50 万；方案 C：投资 60 万，预计 2 年内每年收益 40 万。折现率统一用 10%。帮我做个横向对比，告诉我选哪个最划算？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 生成三方案对比表，包含每个方案的 ROI、NPV、IRR、回收期四项指标；明确标注最优方案并说明推荐理由（如"方案 C IRR 最高但周期最短，适合资金周转快的场景"）；包含敏感性分析或风险提示；报告保存为文件 |
| 8  | 对比表完整，有明确推荐，但缺少敏感性分析，或推荐理由不够具体 |
| 6  | 计算了三个方案的部分指标（如只有 ROI 和回收期），缺少 NPV/IRR；有推荐但无理由 |
| 4  | 只分别计算了三个方案，没有生成横向对比表；或推荐结论与数据不一致 |
| 2  | 只计算了其中 1-2 个方案，遗漏了方案 |
| 0  | 拒绝对比，或编造了与输入不符的数据 |

**关键检查点：**
- [ ] 生成了包含三个方案的横向对比表
- [ ] 每个方案均包含 ROI、NPV、IRR、回收期四项指标
- [ ] 明确指出推荐方案并给出具体理由
- [ ] 报告已保存为文件并告知路径

### Query 4：进阶场景 — 完整可行性分析报告

**Query：**
> 我是一家制造企业的财务总监，下周要向董事会汇报一个新产线的投资方案。初始投资 500 万，预计第 1 年收益 80 万，第 2 年 120 万，第 3-5 年每年 150 万，第 6-8 年每年 100 万。公司要求折现率 12%，通胀率按 3% 考虑。帮我生成一份完整的可行性分析报告，要能直接拿去给董事会看的那种。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 报告包含：执行摘要（一段话结论）、参数输入表、逐年现金流表、四项财务指标（ROI/NPV/IRR/回收期）、敏感性分析（至少 ±10% 情景）、风险提示（至少 3 条）、行动建议；格式专业可直接引用；报告保存为文件 |
| 8  | 报告结构完整，但敏感性分析只有一个变量，或风险提示少于 3 条，或缺少逐年现金流明细 |
| 6  | 包含主要财务指标和基本结论，但缺少敏感性分析，或报告格式不够专业（无表格、无分节） |
| 4  | 只给出汇总数字，没有逐年现金流分析，没有敏感性分析，无法直接用于汇报 |
| 2  | 报告结构混乱，或未考虑通胀率（3%）对现金流的影响 |
| 0  | 只在对话中输出文字，未保存为文件；或编造了与输入不符的数据 |

**关键检查点：**
- [ ] 包含逐年现金流明细表（第 1-8 年分别列出）
- [ ] 通胀率 3% 已纳入计算并在参数表中标注
- [ ] 敏感性分析覆盖至少 ±10% 的收益波动情景
- [ ] 风险提示至少 3 条，且具体（非泛泛而谈）
- [ ] 报告已保存为 Markdown 文件并告知路径

### Query 5：进阶场景 — 敏感性分析专项

**Query：**
> 我们在评估一个 SaaS 软件订阅项目，初始开发成本 200 万，预计每年订阅收入 80 万，运营成本每年 20 万，折现率 10%，周期 5 年。但我对收入预测不太有把握，想知道如果收入比预期低 10%、20%，或者成本超支 10%、20%，对 NPV 和 IRR 的影响有多大？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 生成完整敏感性分析表，覆盖收入 -10%/-20% 和成本 +10%/+20% 四个情景；每个情景均给出 NPV 和 IRR 变化值；标注"盈亏平衡点"（NPV=0 时的收入下限）；包含结论（哪个变量对结果影响最大）；报告保存为文件 |
| 8  | 敏感性分析覆盖四个情景，有 NPV/IRR 变化，但未标注盈亏平衡点，或缺少"哪个变量影响最大"的结论 |
| 6  | 只分析了收入变化情景，未分析成本超支情景；或只给出 NPV 变化，未给出 IRR 变化 |
| 4  | 只做了 ±10% 的单一情景，未做 ±20%；或只给出定性描述（"收入下降会影响 NPV"）而无具体数字 |
| 2  | 敏感性分析数字与基准计算结果不一致，存在计算错误 |
| 0  | 拒绝做敏感性分析，或只重复了基准 ROI 数字而未做情景分析 |

**关键检查点：**
- [ ] 敏感性分析表包含至少 4 个情景（收入 -10%/-20%，成本 +10%/+20%）
- [ ] 每个情景均给出 NPV 和 IRR 的具体变化值（非百分比变化，而是绝对值）
- [ ] 标注了盈亏平衡点（NPV=0 时的临界收入）
- [ ] 给出了"哪个变量对结果影响最大"的结论

### Query 8：综合场景 — 端到端投资决策支持

| 分数 | 标准描述 |

| 4  | 只计算了其中一个方案，或对比表缺少 IRR/回收期等关键指标 |

- [ ] 推荐结论引用了至少两个维度的数据支撑（如"方案二 IRR 更高且抗风险能力更强"）
- [ ] 报告已保存为文件并告知路径
- [ ] 通胀率 3% 已纳入计算

## 评测注意事项

1. **文件保存是硬性要求**：根据 SOUL.md 的 Boundaries，"报告必须保存为文件，不只给文字"。凡是未保存文件的响应，最高得分不超过 8 分。
2. **严禁编造数据**：任何自行假设或编造财务数据的响应，直接判 0 分。
3. **计算假设透明度**：折现率、通胀率等假设必须在报告中明确列出，否则扣 1-2 分。
4. **结论明确性**：ROI精算师的核心价值是给出"投/不投"的明确建议，模糊表述（如"需要综合考虑"）视为质量不足。
