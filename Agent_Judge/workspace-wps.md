# WPS表格美化整理师（workspace-s9yzaw0x503kdzot）— LLM-AS-Judge 评测标准

## Agent 背景

WPS表格美化整理师是一个专注于金山文档（Kdocs）在线表格处理的 AI 助手，核心定位是"把丑表格变专业"。它能通过 kdocs Skill 直接读取和修改云端表格，完成数据清洗（去重、规范手机号/日期格式）、表头整理、格式美化（字体/颜色/边框/对齐）、合并单元格等操作，最终交付一份清爽规范的在线表格并提供访问链接。

Agent 语气直接带幽默，先诊断问题再给方案，不绕弯子。能力边界明确：只处理金山文档中的表格，不做超出表格范围的数据分析，不删除原始数据，涉及隐私数据会先征得用户同意。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 行政助理、HR、销售运营、项目协调员等需要频繁处理汇总表格的职场人员 |
| 核心需求 | 快速把从各渠道收集来的"乱表格"（格式不统一、有重复、表头混乱）变成可以直接汇报或共享的规范表格 |
| 使用频率 | 按需触发，典型时机：收到同事提交的汇总数据后、准备周会汇报前、整理客户信息时 |
| 专业水平 | 中低水平，熟悉 Excel 基本操作但不会写公式或宏，期望 Agent 直接帮做好而非教他们怎么做 |

## 核心使用场景

1. **数据去重**：从多个渠道汇总的报名表或客户名单存在重复行，需要清理后再使用
2. **格式统一**：手机号、日期、金额等字段格式五花八门，需要统一规范
3. **表头整理**：表头命名混乱（中英混用、缩写不统一），需要重新规范列名
4. **视觉美化**：表格没有边框、表头无底色、字体大小不一，需要美化后用于汇报展示
5. **结构整理**：合并同类项、调整列顺序、清理空行，让表格结构更清晰

## 评测 Query 与打分标准

### Query 1：基础场景 — 表格数据去重

**Query：**
> 我有一份金山文档里的客户登记表，叫"2024客户信息汇总.xlsx"，里面有好多重复的行，同一个客户登记了两三次。你能帮我把重复的删掉吗？保留第一条就行，其他的去掉。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行去重工作流：① 调用 `search_files` 定位文件并获取 file_id/drive_id；② 调用 `sheet.get_sheets_info` 获取工作表结构；③ 调用 `sheet.get_range_data` 读取全部数据；④ AI 识别重复行并说明发现了多少重复；⑤ 调用 `sheet.update_range_data` 将去重后数据覆盖写入顶部，再用空值清空多余行；⑥ 调用 `get_file_link` 返回文档链接 |
| 8  | 完成去重操作并返回链接，但缺少向用户说明发现了哪些重复行（透明度不足），或未清空底部多余行（残留空行） |
| 6  | 正确读取了数据并识别出重复，但只输出了去重后的数据列表让用户自己处理，没有实际写回表格 |
| 4  | 告诉用户如何手动去重（如用 Excel 的"删除重复项"功能），没有调用任何工具实际操作 |
| 2  | 混淆了去重逻辑（如把所有重复行都删掉而非保留第一条），或调用了错误工具（如对 .xlsx 使用 `read_file_content`） |
| 0  | 拒绝操作、找不到文件后直接放弃、或声称无法处理 Excel 文件 |

**关键检查点：**
- [ ] 是否调用了 `sheet.get_range_data` 而非 `read_file_content` 来读取表格数据
- [ ] 是否执行了"覆盖写入去重数据 + 清空多余行"的两步去重操作
- [ ] 是否在操作完成后提供了文档的在线访问链接

### Query 2：基础场景 — 手机号格式统一

**Query：**
> 我们部门收集了一份员工通讯录，存在金山文档里，文件名是"员工通讯录2024.xlsx"。里面手机号这列乱得很，有的带"-"，有的前面有空格，有的是11位有的是12位（多了个0）。能帮我统一成标准的11位数字格式吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | ① 搜索定位文件；② 读取表格数据并识别手机号列；③ 逐一分析异常格式（列举发现的问题类型：含"-"、含空格、12位等）；④ 调用 `sheet.update_range_data(opType="formula")` 逐行写入规范化后的11位手机号；⑤ 返回文档链接并说明修改了多少条 |
| 8  | 完成格式统一并返回链接，但未向用户说明发现了哪些格式问题，或只处理了部分异常类型（如处理了"-"但漏掉了前导0） |
| 6  | 正确识别了手机号格式问题并给出了规范化规则，但没有实际写回表格，只是输出了规范化后的数据列表 |
| 4  | 给出了手机号规范化的 Excel 公式（如 `=TEXT()`），让用户自己去填，没有直接操作 |
| 2  | 处理逻辑有误（如把12位号码直接截断而非去掉前导0，导致号码错误） |
| 0  | 以"涉及隐私数据"为由拒绝处理，但未先征询用户同意就直接拒绝；或完全不操作 |

**关键检查点：**
- [ ] 是否识别并列举了手机号列中存在的具体格式问题类型
- [ ] 是否使用 `sheet.update_range_data(opType="formula")` 逐行写入规范化值
- [ ] 是否说明了处理了多少条数据，并提供文档链接

### Query 3：进阶场景 — 表格视觉美化

**Query：**
> 我做了一份季度销售汇报表，放在金山文档里叫"Q3销售汇报.xlsx"。现在表格看起来很素，没有颜色没有边框，领导说要好看一点。你能帮我美化一下吗？表头加个深色背景，数据行加边框，字体统一一下。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | ① 搜索定位文件并读取表格结构；② 调用 `sheet.update_range_data(opType="format")` 对表头行设置深色背景填充（`fill` 属性）和白色字体；③ 对数据区域设置边框（`dgBottom`/`dgTop`/`dgLeft`/`dgRight` 等属性）；④ 统一字体设置（`font` 属性）；⑤ 返回文档链接并描述做了哪些美化 |
| 8  | 完成美化并返回链接，但只做了部分格式（如只加了表头颜色，没有加边框），或边框只加了外框没有内部网格线 |
| 6  | 给出了详细的美化方案描述（颜色代码、字体规格），但没有实际调用工具写入格式，只是告诉用户该怎么做 |

| 2  | 调用了错误的工具（如用 `otl.insert_content` 处理 xlsx 文件），或格式参数完全错误导致操作失败 |
| 0  | 声称无法修改表格格式，或建议用户下载到本地用 Excel 自己改 |

**关键检查点：**
- [ ] 是否使用 `opType="format"` 并包含 `fill`（背景色）属性处理表头行
- [ ] 是否使用 `opType="format"` 并包含边框属性（`dgBottom` 等）处理数据区域
- [ ] 是否包含 `font` 属性统一字体设置
- [ ] 操作完成后是否调用 `get_file_link` 返回链接

### Query 4：进阶场景 — 表头规范化

**Query：**
> 我从系统里导出了一份数据表，存到金山文档了，叫"系统导出_20241015.xlsx"。表头全是英文缩写，像"cust_nm"、"tel_no"、"reg_dt"这种，我们中国同事根本看不懂。能帮我把表头改成中文，比如"客户姓名"、"联系电话"、"注册日期"这样的？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | ① 搜索定位文件；② 读取第一行（表头行）数据；③ 列出所有英文列名并给出对应的中文翻译建议，**向用户确认**翻译是否准确；④ 用户确认后调用 `sheet.update_range_data(opType="formula", rowFrom:0, rowTo:0)` 逐列写入中文表头；⑤ 返回文档链接 |
| 8  | 完成表头替换并返回链接，但没有在修改前向用户确认翻译方案，直接按自己的理解改了 |
| 6  | 给出了完整的英文→中文对照表并请用户确认，但没有实际写回表格 |

| 2  | 把表头行当作数据行处理，修改了错误的行，或覆盖了数据内容 |
| 0  | 拒绝操作，或声称需要用户提供完整的对照表才能处理 |

**关键检查点：**
- [ ] 是否先读取表头行再给出翻译方案，而非凭空猜测列名

### Query 7：综合场景 — 端到端表格清洗与美化

**Query：**
> 我们HR部门有一份招聘信息汇总表，在金山文档里叫"2024招聘汇总.xlsx"。这份表是从好几个渠道收集来的，问题很多：有重复的候选人（同一个人投了多次），手机号格式不统一，表头有的是中文有的是英文，而且整体没有格式，看起来很乱。下周要给总监汇报，能帮我全部整理好吗？整理完发我链接。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行多步骤清洗流程：① `search_files` 定位文件；② `sheet.get_sheets_info` + `sheet.get_range_data` 读取全部数据；③ AI 诊断并列出所有问题（重复行数量、手机号异常条数、混用表头列名）；④ 按顺序执行：去重（覆盖写入+清空多余行）→ 手机号规范化（逐行 `opType="formula"`）→ 表头统一为中文（`rowFrom:0, rowTo:0`）→ 格式美化（表头加色、数据区加边框，`opType="format"`）；⑤ 调用 `get_file_link` 返回链接；⑥ 汇总说明做了哪些改动 |
| 8  | 完成了大部分操作（3-4项）并返回链接，但遗漏了1项（如做了去重和格式化，但没有统一表头），或操作顺序有问题（如先美化再去重，导致格式被覆盖） |
| 6  | 完成了2项操作（如只做了去重和手机号规范化），没有做表头统一和视觉美化，交付物不完整 |
| 4  | 只做了1项操作，或只输出了问题诊断报告而没有实际修改表格 |
| 2  | 操作过程中出现严重错误（如把数据行当表头处理，或去重时删错了行），导致数据损坏 |
| 0  | 只返回了文档链接但没有做任何修改，或完全拒绝处理 |

**关键检查点：**
- [ ] 是否在操作前先诊断并列出了所有发现的问题（重复数、格式异常数等）
- [ ] 是否覆盖了全部四类操作：去重、手机号规范化、表头统一、视觉美化
- [ ] 去重操作是否包含"清空多余行"步骤（而非只覆盖写入去重数据）
- [ ] 是否最终调用 `get_file_link` 返回了可访问的文档链接
- [ ] 操作完成后是否提供了改动摘要（改了什么、改了多少条）
