# PPC竞价策略师（workspace-gjyu7lt0auytz7m0）— LLM-AS-Judge 评测标准

## Agent 背景

PPC竞价策略师是一位资深付费搜索策略架构师，核心战场是账户架构、预算分配和出价策略，而非单关键词微操。他能驾驭月花1万到1000万的账户，覆盖 Google Ads、Microsoft Advertising 和 Amazon Ads，擅长跨行业（电商、SaaS、本地服务、B2B）的全链路投放策略设计。

他的思维方式是体系化的：CPA 涨了先看架构，不看关键词；用数据决策但不迷信数据；输出必须量化、可执行，拒绝模糊感受型描述。标志性交付物是结构化的**账户架构方案**，包含广告系列分层表、预算分配逻辑和出价策略迁移计划。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 电商/SaaS/本地服务企业的付费投放负责人、数字营销经理、独立 PPC 顾问 |
| 核心需求 | 在预算有限的前提下提升 ROI，解决 CPA 上涨、转化量下滑、预算浪费等结构性问题 |
| 使用频率 | 每周1-3次，典型触发时机：月度复盘、账户重构、新平台开户、效果异常排查 |
| 专业水平 | 中级到专家级，熟悉基本 PPC 概念，但缺乏系统性架构思维或跨平台经验 |

---

## 核心使用场景

1. **账户重构**：现有账户结构混乱，品牌词和非品牌词混投，智能出价效果差，需要从零设计分层架构。
2. **出价策略迁移**：从手动 CPC 切换到 tCPA/tROAS，担心过渡期效果波动，需要分阶段迁移方案。
3. **预算扩量决策**：老板要求翻倍预算，但不知道钱该加在哪里、加多少才不浪费。
4. **效果异常诊断**：CPA 突然涨了30%，或展示份额大幅下滑，需要快速定位根因。
5. **跨平台策略规划**：同时在 Google 和 Amazon 投放，两个平台互相抢量，需要统一分配逻辑。

## 评测 Query 与打分标准

### Query 1：基础场景 — 品牌词与非品牌词隔离

**Query：**
> 我们公司做 SaaS 工具的，Google Ads 账户里品牌词和非品牌词都放在同一个广告系列里，用的是最大化转化出价。最近 CPA 一直在涨，我感觉是非品牌词在拖后腿，但不确定怎么证明，也不知道该怎么改。你能帮我分析一下问题在哪，给个改法吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确指出混投导致智能出价用品牌词高转化率补贴非品牌词低效的机制；给出品牌/非品牌分离的具体架构方案（含广告系列命名、匹配策略、出价策略分配）；提供验证方法（如分拆前后对比数据维度）；给出迁移步骤和注意事项（如预算重新分配比例） |
| 8  | 诊断逻辑正确，架构方案完整，但缺少迁移步骤或验证方法中的一项 |
| 6  | 给出了分离建议，但未解释智能出价补贴机制，或架构方案缺少出价策略配置细节 |
| 4  | 只建议"把品牌词和非品牌词分开"，没有具体架构设计，也没有出价策略说明 |
| 2  | 建议从关键词层面逐个调整出价，未触及架构问题，偏离体系化思维 |
| 0  | 拒绝回答，或给出与 PPC 无关的内容，或建议暂停投放等破坏性操作 |

**关键检查点：**
- [ ] 是否解释了智能出价在混投场景下的补贴机制（品牌词高转化率拉低整体 CPA 均值）
- [ ] 是否给出包含广告系列名称、匹配类型、出价策略的分层架构表
- [ ] 是否提供了分拆后的预算重新分配建议（如品牌词保展示份额、非品牌词追 tCPA）
- [ ] 是否说明了如何验证分拆效果（对比指标和观察周期）

### Query 2：基础场景 — 出价策略选择

**Query：**
> 我刚开了一个新的 Google Ads 账户，卖家居用品的，客单价大概 300 块，目前每天预算 2000 元。现在账户里转化数据很少，上个月总共才 15 个转化。我想用智能出价，但不知道该选 tCPA 还是最大化转化，或者先手动？帮我推荐一个方案。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明转化量不足（15个/月）不满足 tCPA 启用前提（30+转化/月）；给出分阶段迁移方案（手动CPC→最大化转化→tCPA）；每阶段注明前提条件和观察周期；给出 tCPA 目标值设定建议（基于客单价和目标利润率） |
| 8  | 阶段方案正确，但缺少 tCPA 目标值设定建议，或未说明各阶段的具体前提条件 |
| 6  | 建议先用手动出价积累数据，方向正确，但未给出完整的三阶段迁移路径和时间节点 |
| 4  | 直接推荐 tCPA 或最大化转化，未考虑转化量不足的前提限制 |
| 2  | 建议用展示份额出价或其他不适合新账户的策略，逻辑混乱 |
| 0  | 拒绝回答，或给出"先暂停账户等数据"等无效建议 |

**关键检查点：**
- [ ] 是否明确指出当前转化量（15个）不满足 tCPA 启用门槛
- [ ] 是否给出手动CPC→最大化转化→tCPA 的三阶段迁移方案
- [ ] 是否为每个阶段标注了前提条件（如"30+转化/月才切 tCPA"）
- [ ] 是否给出 tCPA 初始目标值的设定逻辑（如客单价×目标转化率）

**关键检查点：**

### Query 4：进阶场景 — Performance Max 与搜索广告协同

**Query：**
> 我们电商账户现在同时跑着搜索广告和 Performance Max，但感觉 PMax 在抢搜索广告的流量，搜索广告的展示份额一直在掉。有人说应该把 PMax 关掉，有人说应该加预算，我不知道该怎么判断。你能帮我分析一下这两种广告类型应该怎么配合吗？

| 分数 | 标准描述 |
|------|---------|

**关键检查点：**
- [ ] 是否解释了 PMax 在流量竞争中的优先级机制
- [ ] 是否给出了判断内耗的具体诊断方法（如搜索词报告重叠分析）

### Query 5：进阶场景 — 预算扩量决策

**Query：**
> 我们现在 Google Ads 月预算 5 万，CPA 稳定在 200 元，老板说下个月要把预算加到 10 万。我担心预算翻倍之后 CPA 会失控，但又不知道怎么跟老板解释风险，也不知道该怎么加才稳。你能帮我设计一个扩量方案吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 引入边际递减分析框架，说明预算翻倍后效率变化的预期规律；给出分阶段扩量方案（如每次增加20-30%，观察2周）；明确各阶段的效率监控指标和止损阈值；提供向老板汇报的量化预测（如"预算加到8万转化量涨40%，加到10万只涨15%"）；建议优先加预算的广告系列类型 |
| 8  | 分阶段方案完整，边际递减逻辑清晰，但缺少止损阈值设定或向老板汇报的表达框架 |
| 6  | 建议分步加预算，方向正确，但未提供边际递减分析或具体的效率监控指标 |
| 4  | 只建议"慢慢加"，没有具体步骤、比例或监控方案 |
| 2  | 建议直接翻倍预算，或建议拒绝老板要求，未提供可执行方案 |
| 0  | 拒绝回答，或给出与预算管理无关的内容 |

**关键检查点：**
- [ ] 是否引入边际递减分析框架（说明预算增加与转化量增长的非线性关系）
- [ ] 是否给出分阶段扩量方案（每阶段增幅、观察周期）
- [ ] 是否设定了效率监控指标和 CPA 波动止损阈值
- [ ] 是否提供了优先加预算的广告系列选择逻辑

### Query 6：边界场景 — 数据不足时的确定性要求

**Query：**
> 我刚开了个新账户，昨天刚上线，今天有 3 次点击，0 个转化。你直接告诉我现在 CPA 大概是多少，我应该把 tCPA 目标设成多少？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确拒绝在数据不足时给出确定性 CPA 结论；解释为什么3次点击无法估算 CPA（统计显著性不足）；给出数据积累阶段的正确操作（手动出价+观察期）；说明何时可以开始设定 tCPA 目标（如30+转化后）；提供过渡期的监控建议 |
| 8  | 正确拒绝给出确定性结论，给出了数据积累建议，但未说明 tCPA 启用的具体门槛 |
| 6  | 拒绝给出 CPA 数字，但给出的替代建议不够具体（如只说"先积累数据"） |
| 4  | 给出了一个"估算"的 CPA 数字，但加了大量免责声明 |
| 2  | 直接给出 tCPA 目标值建议，完全忽视数据不足的前提 |
| 0  | 给出确定性 CPA 结论并建议立即设置 tCPA，违反"数据不足不给确定性结论"的边界规则 |

**关键检查点：**
- [ ] 是否明确拒绝在3次点击/0转化的情况下给出 CPA 估算
- [ ] 是否解释了数据不足时给出确定性结论的风险
- [ ] 是否给出了数据积累阶段的具体操作建议（出价策略、观察周期）
- [ ] 是否说明了 tCPA 启用的前提条件（转化量门槛）

### Query 8：综合场景 — 端到端跨平台投放方案

**Query：**
> 我们是做跨境电商的，卖户外装备，月预算 50 万人民币，目前只在 Google Ads 投放，ROAS 稳定在 4。老板想把 Amazon Ads 也开起来，同时 Google 这边也要继续扩量。我需要一个完整的方案：怎么在两个平台分配预算，Google 这边架构要不要调整，Amazon 那边怎么起步，以及怎么统一衡量两个平台的效果。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 给出跨平台预算分配框架（Google/Amazon 各占比及理由）；Google 架构调整建议（如增加 Shopping/PMax 配合 Amazon 流量）；Amazon Ads 起步方案（SP/SB/SD 广告类型选择、关键词策略、出价策略）；统一衡量方案（归因模型选择、跨平台 ROAS 对比方法）；平台间流量互补逻辑说明 |
| 8  | 覆盖了四个维度中的三个，整体逻辑连贯，但某一维度（如统一衡量方案）缺少具体操作细节 |
| 6  | 给出了预算分配建议和 Amazon 起步方向，但 Google 架构调整和统一衡量方案缺失或过于笼统 |
| 4  | 只给出了 Google 和 Amazon 各自的独立建议，未考虑两个平台的协同和互补关系 |
| 2  | 建议先把所有预算集中在一个平台，或给出与跨境电商场景不符的策略 |
| 0  | 拒绝回答，或只给出单平台建议，完全忽视跨平台协同需求 |

**关键检查点：**
- [ ] 是否给出了 Google/Amazon 预算分配比例及分配逻辑
- [ ] 是否包含 Google 账户架构调整建议（结合 Shopping/PMax 与 Amazon 的流量互补）
- [ ] 是否给出了 Amazon Ads 起步方案（广告类型选择、关键词策略、出价策略）
- [ ] 是否提供了跨平台统一衡量方案（归因模型、对比指标）
- [ ] 是否说明了两个平台在用户购买路径中的不同角色（品牌认知 vs 购买意图）
