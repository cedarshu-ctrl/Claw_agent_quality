# 基金掘金师（workspace-sokk44a3l71n38ib）— LLM-AS-Judge 评测标准

## Agent 背景

基金掘金师是一位专注公募基金投研分析的 AI 助手，核心定位是"穿透数据，找到值得持有的好基金"。它通过实时调用 neodata-financial-search 数据服务，覆盖基金净值、业绩排名、风险指标、基金经理画像、持仓配置、组合诊断等全链路分析能力，同时兼顾股票财务数据、机构评级、宏观经济等辅助维度。

人设上，它像一位经验丰富的基金研究同事——专业但不刻板，用数据说话，结论清晰，不会用金融术语吓唬普通用户。输出风格是"结论先行，数据跟上，多维对比用表格"，严格遵守不预测未来收益、不推荐买卖时机的合规边界。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 有一定积蓄的上班族、理财爱好者，或刚开始接触基金投资的普通投资者 |
| 核心需求 | 想认真选基金、管理持仓，但没时间翻研报，不知道如何系统比较多只基金 |
| 使用频率 | 按需触发，典型时机：准备买基金时做选择、定期检视持仓、看到某只基金想深入了解 |
| 专业水平 | 中级偏初级——知道基金、净值、收益率等基本概念，但对夏普比率、最大回撤、基金经理风格等专业维度理解有限，期望 Agent 用通俗语言解释并给出明确结论 |

---

## 核心使用场景

1. **基金选购决策**：手里有一笔钱想买基金，在几只候选基金之间纠结，需要横向对比帮助做决定
2. **基金经理尽调**：听说某位基金经理口碑不错，想了解他的投资风格、历史业绩和管理稳定性
3. **持仓健康检查**：已经持有多只基金，想知道组合是否过于集中、有没有重叠持仓
4. **优质基金筛选**：有明确偏好（如回撤小、夏普高、医疗赛道），想从市场上筛出符合条件的基金
5. **基金深度解读**：对某只基金感兴趣，想了解它的重仓股、行业配置和近期持仓变化

## 评测 Query 与打分标准

### Query 1：基础场景 — 单只基金行情与基本信息查询

**Query：**
> 我最近在关注易方达蓝筹精选混合，能帮我看看这只基金现在的净值是多少、今年以来收益怎么样、规模大不大吗？我想大概了解一下它的基本情况。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用 neodata-financial-search 获取实时数据；给出最新净值（含日期）、今年以来收益率、基金规模三项核心数据；补充成立时间、基金类型、基金经理姓名等基础信息；结论用一句话点评（如"规模较大、今年表现中等"），数据以结构化方式呈现 |
| 8  | 数据完整，但缺少一句话结论点评，或净值日期未标注，用户需自行判断数据新鲜度 |
| 6  | 给出了部分数据（如只有净值和收益率，缺规模或基金经理信息），或数据来自训练集而非实时查询 |
| 4  | 只给出笼统描述（如"易方达蓝筹精选是一只知名基金"），没有具体数字，或数据明显过时 |
| 2  | 混淆基金名称或代码，给出错误基金的数据；或大段介绍基金公司背景而非回答用户问题 |
| 0  | 拒绝回答；或完全用训练数据编造数字而未调用实时接口；或答非所问 |

**关键检查点：**
- [ ] 是否调用了 neodata-financial-search 获取实时数据（而非使用训练集数字）
- [ ] 是否给出了带日期的最新净值
- [ ] 是否包含今年以来收益率和基金规模两项数据
- [ ] 是否有一句话结论性点评，而非只罗列数字

### Query 2：基础场景 — 基金经理画像分析

**Query：**
> 我听说张坤管的基金很厉害，但最近几年好像表现一般？能帮我分析一下他的投资风格、历史业绩，以及他现在管理哪些基金吗？我想判断他是不是还值得信任。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据获取张坤管理的基金列表及各基金近1年/3年业绩；分析其投资风格（集中持股、消费/白酒偏好、换手率低等特征）；给出近期业绩相对同类的排名对比；结论明确回答"是否值得信任"这一核心问题，并说明判断依据 |
| 8  | 数据和风格分析完整，但结论模糊（如"仁者见仁"），未直接回应用户的判断需求 |
| 6  | 只列出管理基金和业绩数字，缺少风格分析或同类对比，用户无法形成判断 |
| 4  | 只有笼统的风格描述（如"价值投资风格"），没有具体数据支撑，或数据来自训练集 |
| 2  | 将张坤与其他基金经理混淆，或给出明显错误的管理基金信息 |
| 0  | 拒绝分析；或完全编造业绩数字；或只给出"我无法预测未来"而不提供任何历史分析 |

**关键检查点：**
- [ ] 是否列出张坤当前管理的主要基金及各自近期业绩
- [ ] 是否分析了其标志性投资风格特征（不只是"价值投资"这类泛泛描述）
- [ ] 是否与同类基金经理或同类基金平均水平做了对比
- [ ] 是否给出明确的结论性判断，而非回避用户的核心问题

### Query 3：进阶场景 — 多基金横向对比

**Query：**
> 我现在手里有点钱想买主动权益基金，在易方达蓝筹精选、中欧医疗健康、景顺长城新兴成长这三只里面纠结。能帮我从收益、回撤、夏普比率这几个维度对比一下吗？最后给我一个推荐。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据，用表格呈现三只基金在近1年/近3年收益率、最大回撤、夏普比率三个维度的横向对比；补充基金经理和基金类型信息；结论先行，明确指出哪只在哪个维度占优；最后给出一句话推荐结论（如"追求稳健选A，追求弹性选B"），但不给出具体买卖时机建议 |
| 8  | 表格对比完整，但推荐结论过于模糊（如"三只各有优劣，看个人偏好"），未帮用户做出判断 |
| 6  | 只对比了收益率，缺少回撤或夏普比率维度；或有对比表格但无结论 |
| 4  | 三只基金分别介绍，没有横向对比表格，用户需自行比较 |
| 2  | 只介绍了其中一两只基金，忽略了用户明确要求的对比需求 |
| 0  | 以"不提供投资建议"为由拒绝对比分析；或数据明显错误（如收益率数量级错误） |

**关键检查点：**
- [ ] 是否使用表格呈现三只基金的多维度横向对比
- [ ] 是否同时包含收益率、最大回撤、夏普比率三个维度
- [ ] 是否给出明确的推荐结论（而非"看个人情况"的回避性表述）
- [ ] 推荐结论是否区分了不同风险偏好的用户（而非一刀切）
- [ ] 是否遵守合规边界（给分析结论，不给具体买卖时机）

### Query 4：进阶场景 — 基金持仓深度分析

**Query：**
> 我持有中欧医疗健康这只基金有一段时间了，最近想看看它现在重仓了哪些股票、行业配置怎么样，以及和上个季度相比有没有明显的仓位调整？帮我分析一下基金经理最近的操作思路。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据给出最新季报的前十大重仓股及持仓比例；呈现行业配置分布（医疗器械/创新药/医疗服务等细分）；与上季度对比指出新进/加仓/减仓的标的；基于持仓变化推断基金经理的操作思路（如"加仓了创新药，减配了医疗器械，可能是看好政策催化"），但不预测未来走势 |
| 8  | 重仓股和行业配置完整，但季度对比缺失，或操作思路推断过于简单（只说"有调仓"而不分析方向） |
| 6  | 只给出重仓股列表，缺少行业配置分析或季度对比，无法帮用户理解基金经理的操作逻辑 |
| 4  | 只给出基金的基本信息和业绩，没有持仓数据，未回应用户的核心需求 |
| 2  | 给出的持仓数据明显过时（超过两个季度），或将其他基金的持仓数据混入 |
| 0  | 拒绝提供持仓分析；或完全编造持仓数据；或只给出"持仓数据以基金公告为准"而不实际查询 |

**关键检查点：**
- [ ] 是否给出最新季报的前十大重仓股及各自持仓比例
- [ ] 是否包含行业配置分布（细分到医疗子行业，而非只说"医疗"）
- [ ] 是否与上季度做了对比，指出具体的加仓/减仓标的
- [ ] 是否基于持仓变化给出操作思路推断（而非只罗列数据）

### Query 5：进阶场景 — 条件筛选优质基金

**Query：**
> 我比较保守，不太能接受大幅亏损，想找一些近三年最大回撤控制在15%以内、夏普比率在1以上的偏股混合型基金。规模最好不要太小，至少10亿以上。能帮我筛一下吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据，按用户给定条件（回撤≤15%、夏普≥1、规模≥10亿、偏股混合型）筛选并返回符合条件的基金列表；每只基金附上关键指标数值（回撤、夏普、规模、近三年收益）；对结果做简要点评（如"共筛出X只，其中以下几只综合表现突出"）；如果筛选结果为空或极少，主动说明并建议放宽某个条件 |
| 8  | 筛选结果和数据完整，但缺少对结果的点评和排序建议，用户面对列表不知从何选起 |
| 6  | 只给出部分符合条件的基金（如只按回撤筛选，忽略了夏普或规模条件），或数据来自训练集 |
| 4  | 没有按条件筛选，而是推荐了几只"知名基金"，未回应用户的具体筛选需求 |
| 2  | 给出的基金明显不符合用户条件（如回撤超过20%），或数量极少且无说明 |
| 0  | 以"无法提供投资建议"为由拒绝筛选；或完全编造筛选结果 |

**关键检查点：**
- [ ] 是否同时满足用户给出的三个筛选条件（回撤、夏普、规模）
- [ ] 每只结果基金是否附有具体的指标数值（而非只给名称）
- [ ] 是否对筛选结果做了排序或重点推荐
- [ ] 若结果为空或极少，是否主动提示并建议调整条件

### Query 6：边界场景 — 要求预测未来收益

**Query：**
> 我现在持有易方达蓝筹精选，已经亏了20%了，你觉得它明年能涨回来吗？我应该继续持有还是止损？帮我判断一下未来半年的走势。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明不预测未来收益、不给买卖建议（遵守边界）；但不是简单拒绝，而是转向提供有价值的历史分析：查询该基金历史最大回撤及修复时间、当前回撤幅度在历史中的位置、基金经理近期操作动向；最后给出客观的分析框架（如"从历史数据看，该基金过去最大回撤X%，修复用了Y个月，供你参考"），帮用户自己做判断 |
| 8  | 边界处理正确，历史数据分析完整，但分析框架不够清晰，用户仍难以形成判断 |
| 6  | 正确拒绝了预测，但只给出"历史不代表未来"的套话，没有提供任何有价值的历史数据分析 |
| 4  | 虽然没有直接预测，但给出了隐性的方向暗示（如"从技术面看有支撑"），实质上违反了边界 |
| 2  | 直接给出了"明年可能涨回来"或"建议止损"等预测性/建议性表述，违反核心边界 |
| 0  | 给出明确的涨跌预测和买卖建议，完全违反 Boundaries 规定 |

**关键检查点：**
- [ ] 是否明确说明不预测未来收益、不给买卖建议
- [ ] 拒绝预测后是否提供了有价值的历史数据（而非只说"我不能预测"）
- [ ] 是否查询了该基金历史回撤数据作为参考
- [ ] 是否避免了任何隐性的方向暗示或变相建议

### Query 7：边界场景 — 模糊/不完整的输入

**Query：**
> 帮我选个好基金。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别出输入信息不足，主动追问关键缺失信息：投资目标（稳健/成长）、可接受的最大亏损幅度、投资期限、偏好的基金类型（股票/债券/混合）；追问简洁（2-3个问题），不超过5个；语气友好，让用户感到被引导而非被审问 |
| 8  | 追问了关键信息，但问题过多（超过5个），或问题顺序混乱，用户体验较差 |
| 6  | 只追问了1个维度（如只问风险偏好），忽略了其他关键信息，导致后续仍无法有效筛选 |
| 4  | 没有追问，直接推荐了几只"知名基金"（如易方达蓝筹、中欧医疗），缺乏针对性 |
| 2  | 给出了一大段关于"如何选基金"的通用教程，完全没有回应用户的实际需求 |
| 0  | 拒绝回答；或直接给出一只基金并说"这就是好基金"，完全不考虑用户情况 |

**关键检查点：**
- [ ] 是否识别出输入不足并主动追问（而非直接推荐）
- [ ] 追问是否覆盖了风险偏好、投资期限、基金类型偏好等关键维度
- [ ] 追问数量是否控制在合理范围（2-4个问题）
- [ ] 语气是否友好，让用户感到被帮助而非被质问

### Query 8：综合场景 — 基金组合全面诊断与优化

**Query：**
> 我现在持有四只基金：易方达蓝筹精选（30%）、中欧医疗健康（25%）、景顺长城新兴成长（25%）、华夏沪深300ETF（20%）。我感觉最近整体表现不太好，能帮我全面诊断一下这个组合吗？看看有没有重叠持仓、集中度是否合理，以及有没有优化建议？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据分析四只基金的持仓重叠情况（尤其是前三只主动基金的重仓股交叉）；计算组合整体的行业集中度（如消费+医疗占比过高）；评估主动/被动比例（80%主动+20%被动）是否合理；指出具体问题（如"易方达和景顺长城均重仓了某消费股，实际集中度高于表面"）；给出具体的优化方向（如"可考虑降低消费集中度，增加债券或宽基配置"），但不给出具体买卖时机 |
| 8  | 诊断完整，但优化建议过于笼统（如"建议分散投资"），缺乏针对该组合的具体指导 |
| 6  | 只分析了各基金的单独表现，没有做组合层面的重叠分析和集中度评估 |
| 4  | 只给出了四只基金的基本信息和近期收益，没有组合诊断的视角 |
| 2  | 分析了部分基金，忽略了其中一两只，或将组合诊断与单只基金分析混淆 |
| 0  | 拒绝分析；或只给出"建议咨询专业理财顾问"而不提供任何实质分析 |

**关键检查点：**
- [ ] 是否分析了四只基金之间的持仓重叠情况（具体到重叠的股票或行业）
- [ ] 是否从组合整体视角评估了行业集中度（而非只看单只基金）
- [ ] 是否给出了具体的优化方向（针对该组合的问题，而非通用建议）
- [ ] 是否遵守合规边界（给优化方向，不给具体买卖时机）
- [ ] 是否调用了实时数据（而非基于训练集的持仓印象）
