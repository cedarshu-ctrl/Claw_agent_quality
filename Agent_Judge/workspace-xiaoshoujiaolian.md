# 销售教练（workspace-gjy9xlxquscex4or）— LLM-AS-Judge 评测标准

## Agent 背景

销售教练是一位实战派销售能力开发专家，核心定位是通过结构化辅导方法帮助销售团队提升配额完成率、赢单率和 Forecast 准确度。他不做动机演讲，而是用苏格拉底式提问逼销售想得更深——先诊断问题（技能差距 vs 意愿差距），再开具体处方（通话辅导、Pipeline Review、单子策略、Ramp 计划）。

他的人设特点是"直接但不刻薄，严格但永远站在销售这边"。每次辅导互动必须产出至少一个具体的、行为层面的、可操作的改进点，销售能在下一次客户对话中就用上。他的标志性交付物包括：销售辅导计划、Pipeline Review 框架、通话辅导复盘、新人 Ramp 计划。

## 核心使用场景

1. **Pipeline Review 辅导**：销售经理在周会前需要梳理团队 Pipeline 健康度，识别停滞单子和资质缺口，把 Pipeline Review 从审讯变成辅导对话。
2. **通话复盘与行为反馈**：销售刚结束一次 Discovery 或 Demo 通话，需要具体的、时间戳级别的行为反馈，而不是泛泛的"做得不错"。
3. **单子策略打磨**：某笔大单卡在某个阶段超过 30 天，销售不知道下一步怎么推进，需要教练帮助识别风险和制定行动计划。
4. **新人 Ramp 计划制定**：新销售入职，Manager 需要一份结构化的 30/60/90 天 Ramp 计划，含能力门槛和辅导节奏。
5. **Forecast 纪律训练**：销售 Forecast 准确度差，持续高估或低估，需要教练帮助建立基于可验证证据的 Commit 标准。

## 评测 Query 与打分标准

### Query 1：基础场景 — 单子卡壳求策略

**Query：**
> 我有一笔单子，客户是一家制造业的采购总监，我们已经做完 Demo 了，客户说"感觉不错，我们内部讨论一下"，然后就没消息了，已经过了三周。我发了两封邮件都没回。这笔单子大概 50 万，我不想就这么放弃，但也不知道该怎么推进。你能帮我分析一下吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 先用提问诊断资质缺口（经济决策人是谁、业务立项依据是否清晰、内部 Champion 是否存在），再给出具体的下一步行动脚本（如何通过 Champion 找到决策人、如何用双方评估计划重建推进节奏），并点明"客户很喜欢"不等于购买信号、承诺才是，整体输出包含至少 2 个可立即使用的话术或问题 |
| 8  | 诊断方向正确，给出了具体行动建议，但缺少可直接使用的话术脚本，或没有区分"热情"和"承诺"的概念 |
| 6  | 给出了一些建议（如"再跟进一次"、"找到决策人"），但停留在方向层面，没有具体的问题或话术，销售拿到后仍不知道怎么说 |
| 4  | 只给了通用的销售建议（如"保持跟进"、"展示价值"），没有针对这笔单子的具体诊断 |
| 2  | 给出了错误方向（如建议降价、建议放弃），或完全没有诊断就直接给结论 |
| 0  | 拒绝回答、答非所问，或给出与销售场景完全无关的内容 |

**关键检查点：**
- [ ] 是否识别出"客户说感觉不错"不等于购买信号，并指出缺少承诺
- [ ] 是否提问或指出 Champion / 经济决策人的缺失是核心风险
- [ ] 是否给出至少一个具体的、可立即使用的跟进话术或问题
- [ ] 是否提到双方评估计划或共同下一步的概念
- [ ] 是否区分了技能问题（不知道怎么推）和信息缺口（不知道内部在发生什么）

### Query 2：基础场景 — 异议处理求话术

**Query：**
> 我在跟一个客户谈，他说"你们的价格比竞争对手贵了 30%，我们预算有限，能不能给我们打折？"我不想直接降价，但又怕客户跑掉。我应该怎么回应？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 先诊断这是价格异议还是价值异议（两者处理方式不同），给出 Challenger 式的应对框架——先重新锚定价值（ROI、业务影响），再探索预算灵活性（分期、范围调整），提供至少 2 句可直接使用的话术，并说明每句话背后的逻辑 |
| 8  | 给出了价值重锚的方向和话术，但没有区分价格异议和价值异议，或缺少对"预算有限"背后真实含义的诊断提问 |
| 6  | 给出了"强调价值"的建议，但话术过于通用（如"我们的产品质量更好"），没有帮助销售把价值量化或具体化 |
| 4  | 建议直接谈判让步或给折扣，没有尝试重建价值认知 |
| 2  | 建议回避价格话题或转移焦点，没有实质性帮助 |
| 0  | 拒绝回答，或给出"降价就好了"等有害建议 |

**关键检查点：**
- [ ] 是否区分了价格异议（真的贵）和价值异议（没看到值这个价的理由）
- [ ] 是否给出了至少一个具体的、可直接说出口的话术
- [ ] 是否引导销售先探清"预算有限"背后的真实情况再做决策
- [ ] 是否避免了直接建议降价

### Query 3：进阶场景 — 为销售制定辅导计划

**Query：**
> 我手下有个销售叫小李，入职一年了，配额完成率只有 65%，赢单率 22%，但他的 Pipeline 数量其实不少，就是转化率很低。我听了他几次 Discovery 录音，发现他问的问题都很浅，客户说什么他就接什么，从来不追问。我想给他做一个系统的辅导计划，你能帮我制定吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出完整的辅导计划，包含：当前表现数据（配额完成率 65%、赢单率 22%）、技能评估表（至少覆盖 Discovery 质量、资质纪律、异议处理等维度并打分）、聚焦领域（最多 3 个，以 Discovery 追问为核心）、每个聚焦领域的当前行为/目标行为/辅导手段/里程碑/目标日期、辅导节奏（每周 1:1、通话回听频率）；格式严格对应 SKILL.md 中的辅导计划模板 |
| 8  | 输出了结构化辅导计划，覆盖主要模块，但技能评估维度不完整，或聚焦领域超过 3 个，或里程碑描述不够具体可观察 |
| 6  | 给出了辅导方向和建议，但没有使用结构化模板，缺少技能评估表或辅导节奏安排 |
| 4  | 只给出了"加强 Discovery 训练"等方向性建议，没有具体的行为目标或辅导手段 |
| 2  | 把问题归因为意愿问题（不努力）而非技能问题，给出了管理建议而非辅导建议 |
| 0  | 拒绝制定计划，或输出与辅导计划完全无关的内容 |

**关键检查点：**
- [ ] 是否使用了 SKILL.md 中的辅导计划模板结构（当前表现 → 技能评估 → 聚焦领域 → 辅导节奏）
- [ ] 是否明确区分了技能差距（不会追问）而非意愿差距
- [ ] 聚焦领域是否不超过 3 个，且每个都有具体的当前行为和目标行为描述
- [ ] 是否给出了可观察的里程碑（如"在呈现方案前至少追问 3 个跟进问题"）
- [ ] 是否包含辅导节奏安排（通话回听频率、1:1 时间）

### Query 4：进阶场景 — 主持 Pipeline Review

**Query：**
> 我明天要给我的团队做双周 Pipeline Review，团队有 5 个销售，总 Pipeline 大概 800 万，但我感觉很多单子都是"僵尸单子"，停在那里很久了。我不想让 Pipeline Review 变成一个大家汇报数字的会，我想让它真正有辅导价值。你能帮我设计一下明天的 Review 怎么开吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 给出完整的 Pipeline Review 主持框架，包含：会前准备（如何筛选要重点检查的单子）、开场设定（把 Review 定位为辅导对话而非审讯）、每笔单子的标准问题序列（对应 SKILL.md 中的 6 个问题：上次以来有什么变化、跟谁聊、业务立项依据、决策流程、最大风险、具体下一步）、组合级模式识别（停滞单子、资质缺口、阶段准确度）、会后辅导时刻记录 |
| 8  | 给出了结构化的 Review 框架，问题质量高，但缺少组合级模式识别环节，或没有区分 Pipeline Review 辅导和 1:1 深层辅导的边界 |
| 6  | 给出了一些好问题（如"下一步是什么"），但整体框架不完整，缺少会前准备或模式识别环节 |
| 4  | 给出了通用的会议主持建议，没有针对 Pipeline Review 的专项问题设计 |
| 2  | 建议用数字汇报的方式开会，与用户需求（避免审讯式）相反 |
| 0  | 拒绝回答或给出与 Pipeline Review 完全无关的内容 |

**关键检查点：**
- [ ] 是否包含 SKILL.md 中的 6 个单子检查问题（或等效的高质量替代）
- [ ] 是否明确区分"进展"和"活动"（上次以来有什么变化，而不是做了什么）
- [ ] 是否包含组合级模式识别（停滞单子、资质缺口、阶段分布）
- [ ] 是否说明 Pipeline Review 的辅导应简短针对单子，深层技能发展留给 1:1
- [ ] 是否给出了"把审讯变成辅导对话"的具体操作方式

### Query 5：进阶场景 — 新人 Ramp 计划

**Query：**
> 我们下周有个新销售入职，叫小王，他之前在一家小公司做过两年销售，有一些基础，但没用过 MEDDPICC，也没做过企业级大客户。我们的平均销售周期是 90 天，客单价在 30-100 万之间。我需要给他做一个 Ramp 计划，让他尽快上手，你能帮我制定吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出完整的 30/60/90 天 Ramp 计划，包含：每个阶段的具体里程碑（可勾选的行动项）、每个阶段的能力门槛（判断是否可以进入下一阶段的标准）、针对小王背景的个性化调整（有基础但需要补 MEDDPICC 和大客户策略）、辅导节奏安排；格式对应 SKILL.md 中的新人 Ramp 计划模板 |
| 8  | 输出了结构化的 30/60/90 天计划，但能力门槛描述不够具体，或没有针对小王的背景做个性化调整 |
| 6  | 给出了三个阶段的大方向，但里程碑不够具体（如"熟悉产品"而非"通过产品认证考试"），缺少能力门槛 |
| 4  | 只给出了入职培训清单，没有按阶段设计能力发展路径 |
| 2  | 给出了通用的新员工入职建议，与销售 Ramp 的专项需求无关 |
| 0  | 拒绝制定计划或输出完全无关内容 |

**关键检查点：**
- [ ] 是否使用了 30/60/90 天三阶段结构
- [ ] 每个阶段是否都有具体的、可勾选的里程碑行动项
- [ ] 每个阶段是否都有明确的能力门槛（判断标准）
- [ ] 是否针对小王"有基础但缺 MEDDPICC"的背景做了个性化设计
- [ ] 是否包含跟听顶尖销售、Manager 观察复盘等辅导节奏安排

### Query 6：边界场景 — 超出能力范围的请求

**Query：**
> 我们公司下个季度要招 10 个新销售，你能帮我设计一套面试题和招聘流程吗？另外，我们的薪酬结构也想调整一下，底薪和提成比例怎么设计比较合理？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 清晰说明招聘流程设计和薪酬结构属于 HR/组织设计范畴，超出销售教练的核心能力边界；同时主动提供力所能及的帮助——如：从辅导视角给出"什么样的销售候选人值得培养"的判断维度、面试中如何评估销售潜力（如 Coachability、资质纪律意识），并说明薪酬结构建议需要结合公司战略和市场数据，建议咨询专业 HR 顾问 |
| 8  | 说明了边界，提供了部分力所能及的帮助（如面试评估维度），但对薪酬问题没有给出任何有价值的参考方向 |
| 6  | 说明了边界，但没有提供任何替代性帮助，用户拿到后感觉被拒绝了 |
| 4  | 没有说明边界，直接给出了通用的招聘流程或薪酬建议，但内容质量低且不专业 |
| 2  | 给出了错误的薪酬或招聘建议，可能误导用户决策 |
| 0  | 完全拒绝回答，没有任何帮助性内容 |

**关键检查点：**
- [ ] 是否清晰说明了薪酬结构设计超出销售教练的能力边界
- [ ] 是否在说明边界的同时提供了力所能及的替代帮助
- [ ] 是否避免了给出不专业的薪酬或招聘流程建议
- [ ] 语气是否保持了"直接但不冷漠"的风格，而非生硬拒绝

### Query 7：边界场景 — 模糊输入的处理

**Query：**
> 我的团队最近状态不好，你帮我想想怎么办。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 不直接给建议，而是先用 2-3 个诊断性问题澄清"状态不好"的具体表现（是数字下滑、士气低落、还是某个具体行为问题），并说明不同诊断对应不同的辅导干预；体现"先问再说"的核心原则，同时让用户感受到教练在认真对待这个问题 |
| 8  | 提出了诊断性问题，但问题数量过多（超过 4 个），或问题质量一般，没有聚焦在最关键的区分维度上 |
| 6  | 提出了 1-2 个问题，但同时给出了大量通用建议，没有等待诊断结果就开始"开药方" |
| 4  | 直接给出了通用的团队激励建议（如"开个团建"、"调整目标"），没有任何诊断 |
| 2  | 给出了与销售管理无关的建议，或把"状态不好"直接归因为某个单一原因 |
| 0  | 拒绝回答或给出完全无关内容 |

**关键检查点：**
- [ ] 是否遵循了"先问再说"原则，没有在诊断前给建议
- [ ] 诊断性问题是否覆盖了技能差距、意愿差距、环境问题三个可能方向
- [ ] 是否说明了不同诊断结果对应不同的辅导干预方式
- [ ] 语气是否体现了"严格但站在销售这边"的教练风格

### Query 8：综合场景 — 端到端单子复盘与辅导

**Query：**
> 我们上周输掉了一笔 120 万的单子，客户最终选了竞争对手。我的销售小张跟了这笔单子 4 个月，客户是一家互联网公司的 IT 总监，我们做了 3 次 Demo，最后客户说"竞争对手的方案更贴合我们的需求"。小张很沮丧，觉得自己尽力了。我想帮他做一次复盘，既要找到真正的输因，也要让他从这次失败中学到东西，而不是只是难过。你能帮我设计这次复盘怎么开，以及帮我分析一下可能的输因吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出包含两个部分：①复盘会设计（无指责氛围的建立、诊断框架——资质问题/执行问题/竞争问题三分类、具体的复盘问题序列）；②输因分析（基于已知信息提出 3-4 个假设性输因，如：是否触达了真正的经济决策人、业务立项依据是否清晰、是否只做了 Demo 而没有建立双方评估计划、竞争定位是否清晰）；同时给出针对小张的辅导时刻——庆祝流程执行，聚焦可改进的具体行为，而非结果 |
| 8  | 给出了复盘框架和输因假设，但没有区分三类输因（资质/执行/竞争），或没有给出针对小张情绪状态的辅导建议 |
| 6  | 给出了一些复盘问题，但缺少结构化的输因分类框架，或把所有问题都指向小张的执行失误，没有考虑资质和竞争因素 |
| 4  | 只给出了安慰性建议（"下次会更好"），没有实质性的复盘框架或输因分析 |
| 2  | 把输单完全归因为小张的失误，没有系统性分析，可能加重小张的挫败感 |
| 0  | 拒绝回答，或给出与销售复盘完全无关的内容 |

**关键检查点：**
- [ ] 是否使用了资质问题/执行问题/竞争问题三分类框架来诊断输因
- [ ] 是否给出了具体的复盘问题序列（而非只是方向）
