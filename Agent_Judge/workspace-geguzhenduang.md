# 个股诊断专家（workspace-54w9giv6xaeqi4mn）— LLM-AS-Judge 评测标准

## Agent 背景

个股诊断专家（📊）是一位专注于深度扫描个股价值与风险的金融数据分析师 Agent。其核心能力是通过 `neodata-financial-search` 实时数据服务，从基本面、财务数据、资金流向、新闻舆情、研报评级等多个维度对股票进行全景体检，并以专业、通俗的语言呈现数据背后的真实信号。

该 Agent 有两条硬性边界：**只诊断，不推荐**（不给出买卖建议）；**只信任实时数据**（凡可实时查询的金融数据，必须调用接口，禁止用训练记忆回答）。其人设是"经验丰富的分析师同事"——直接、专业、简洁，用数据说话，不废话。

---

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 有一定投资经验的个人投资者（股龄 1-5 年）；也包括需要快速了解某只股票的财务从业者、研究员助理 |
| 核心需求 | 在做买卖决策前，快速获取一只股票的多维度客观数据，避免信息不对称；不想自己逐一查财报、研报、资金流向 |
| 使用频率 | 按需触发，典型时机：盘前研究、盘中异动跟踪、财报季分析、持仓复盘 |
| 专业水平 | 中级投资者：懂基本财务指标（PE、ROE、营收），但不擅长深度财报解读；期望 Agent 把复杂数据翻译成易懂结论 |

---

## 核心使用场景

1. **个股全景体检**：持仓某只股票前，想快速了解其基本面、财务健康度、近期资金动向和市场情绪，一次性获取全景画像。
2. **财务数据深挖**：财报季期间，想了解某公司最新季报/年报的营收、净利润、ROE 等核心指标，并与历史数据对比。
3. **资金流向追踪**：盘中发现某股异动，想知道主力资金是在流入还是流出，是否有龙虎榜席位介入。
4. **研报评级查询**：想了解机构对某只股票的最新评级和目标价，判断机构态度是否发生变化。
5. **同行业横向对比**：持有某只股票，想与同行业竞争对手在估值、财务指标上做横向比较，判断相对价值。

## 评测 Query 与打分标准

### Query 1：基础场景 — 个股全景体检

**Query：**
> 我最近在关注比亚迪，想在买之前先了解一下这只股票的整体情况。帮我看看它现在的股价、基本面怎么样，有没有什么值得注意的风险点？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用 neodata-financial-search 获取实时数据；输出包含：①当前股价及涨跌幅（行情数据）；②核心财务指标（营收/净利润/ROE/PE 等至少 3 项）；③近期资金流向（主力净流入/流出）；④近期重大事件或公告摘要；⑤明确声明"以上为数据诊断，不构成投资建议" |
| 8  | 包含上述 4 项核心要素，但缺少合规免责声明，或资金流向/事件信息仅有其中一项 |
| 6  | 包含股价和财务指标，但缺少资金流向和事件信息，诊断维度不完整 |
| 4  | 仅给出股价或仅给出部分财务数据，没有多维度交叉分析 |
| 2  | 未调用实时数据接口，用训练记忆回答（如给出过时的财务数据），或内容严重不准确 |
| 0  | 拒绝回答、给出明确买卖建议（"建议买入/卖出"）、或完全答非所问 |

**关键检查点：**
- [ ] 是否调用了 neodata-financial-search 获取实时数据（而非依赖训练记忆）
- [ ] 是否包含实时股价及涨跌幅
- [ ] 是否包含至少 3 项财务指标（营收、净利润、ROE、PE 等）
- [ ] 是否包含资金流向信息（主力净流入/流出方向）
- [ ] 是否包含合规免责声明（不构成投资建议）

### Query 2：基础场景 — 资金流向追踪

**Query：**
> 今天下午两点多，我看到宁德时代突然拉升了 3%，成交量也放大了。帮我查一下它今天的资金流向，主力是在买还是在卖？有没有上龙虎榜？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据接口；输出包含：①当日主力资金净流入/流出金额及方向；②大单（超大单/大单）与散户（小单）资金对比；③龙虎榜查询结果（有则列出席位和金额，无则明确说明未上榜）；④结合数据对异动给出客观解读（如"主力净流入 X 亿，大单占比 Y%"），不做涨跌预测 |
| 8  | 包含主力资金方向和大单数据，但龙虎榜信息缺失或未明确说明是否上榜 |
| 6  | 仅给出资金流向总体方向，缺少大单/散户分层数据，信息颗粒度不足 |
| 4  | 仅描述股价涨跌，未提供资金流向具体数据 |
| 2  | 未调用实时数据，用训练记忆推测资金流向，或给出"主力在拉升，可以跟进"等投资建议 |
| 0  | 拒绝回答、给出明确操作建议、或完全答非所问 |

**关键检查点：**
- [ ] 是否区分了主力资金（大单）与散户资金（小单）的流向
- [ ] 是否查询了龙虎榜并给出明确结果（上榜/未上榜）
- [ ] 是否基于数据给出客观解读，而非涨跌预测
- [ ] 是否使用实时接口而非训练记忆

### Query 3：基础场景 — 研报评级查询

**Query：**
> 我持有招商银行，最近听说有几家券商调整了评级，帮我查一下最新的机构评级和目标价是多少？各家机构的态度是偏乐观还是偏谨慎？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据接口；输出包含：①最新机构评级列表（至少列出 2-3 家券商的评级，如"买入/增持/中性"）；②各机构目标价；③评级变动情况（如有上调/下调则标注）；④对机构整体态度的客观总结（如"X 家买入，Y 家中性，整体偏乐观"）；⑤数据来源时间标注 |
| 8  | 包含评级和目标价，但缺少评级变动情况，或未对机构整体态度做汇总 |
| 6  | 仅给出 1 家机构评级，或仅给出评级而无目标价，信息不完整 |
| 4  | 仅描述招商银行基本面，未提供机构评级具体数据 |
| 2  | 未调用实时数据，用训练记忆给出过时评级，或给出"机构都看好，可以持有"等建议 |
| 0  | 拒绝回答、给出明确持仓建议、或完全答非所问 |

**关键检查点：**
- [ ] 是否列出了多家券商的具体评级（买入/增持/中性/减持）
- [ ] 是否包含各机构目标价
- [ ] 是否标注了数据的时间（避免过时数据）
- [ ] 是否对机构整体态度做了客观汇总，而非给出操作建议

### Query 4：进阶场景 — 财务数据深挖

**Query：**
> 贵州茅台刚出了年报，我想深入了解一下它的财务状况。帮我看看最近三年的营收增速、净利润率、ROE 的变化趋势，另外毛利率和现金流情况怎么样？跟历史相比有没有什么异常？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据接口获取财报数据；输出包含：①近三年营收及同比增速（列表或趋势描述）；②近三年净利润率变化；③近三年 ROE 数据；④毛利率数据；⑤经营性现金流情况；⑥对各指标趋势的客观解读（如"ROE 连续三年下滑，从 X% 降至 Y%，主要受 Z 因素影响"），不做涨跌预测 |
| 8  | 包含营收、净利润、ROE 三项核心指标的趋势，但缺少毛利率或现金流数据，或趋势解读不够深入 |
| 6  | 仅给出最新一期财报数据，缺少历史对比和趋势分析 |
| 4  | 仅给出部分财务数字，无趋势分析，无异常点识别 |
| 2  | 未调用实时数据，用训练记忆给出财务数据（可能已过时），或数据维度严重不足 |
| 0  | 拒绝回答、给出"茅台财务很好，可以买入"等投资建议、或完全答非所问 |

**关键检查点：**
- [ ] 是否覆盖了至少 3 个财务维度（营收增速、净利润率、ROE）
- [ ] 是否包含多期数据（至少 2-3 年/季度对比）
- [ ] 是否对数据趋势给出客观解读（而非仅罗列数字）
- [ ] 是否包含现金流或毛利率等深度指标
- [ ] 是否使用实时接口获取最新财报数据

### Query 5：进阶场景 — 同行业横向对比

**Query：**
> 我在考虑新能源汽车板块，想对比一下比亚迪和长城汽车这两只股票。帮我从估值、营收规模、盈利能力这几个维度做个横向比较，看看哪只在数据上更有优势？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 分别调用两只股票的实时数据；输出包含：①两只股票的 PE/PB 估值对比；②最新营收规模对比；③净利润率/ROE 盈利能力对比；④以结构化表格或并列格式呈现，便于直观比较；⑤对比结论客观陈述数据差异（如"比亚迪营收规模是长城的 X 倍，但 PE 估值也高出 Y 倍"），不做"哪只更值得买"的推荐 |
| 8  | 包含估值和盈利能力对比，但缺少结构化对比格式，或对比维度少于 3 个 |
| 6  | 仅对比了 1-2 个维度（如只比较股价涨跌），缺少财务指标横向对比 |
| 4  | 分别介绍两只股票，但未做真正的横向对比分析 |
| 2  | 未调用实时数据，或给出"比亚迪比长城更值得投资"等推荐性结论 |
| 0  | 拒绝回答、只分析一只股票、给出明确买卖推荐、或完全答非所问 |

**关键检查点：**
- [ ] 是否同时获取了两只股票的实时数据
- [ ] 是否包含估值指标（PE/PB）的横向对比
- [ ] 是否包含盈利能力指标（净利润率或 ROE）的横向对比
- [ ] 是否以结构化格式（表格或并列）呈现对比结果
- [ ] 是否仅陈述数据差异，未给出"哪只更值得买"的推荐

### Query 6：进阶场景 — 板块资金与龙头研究

**Query：**
> 最近 AI 算力板块很热，我想了解一下这个板块的整体情况。帮我看看板块今天的资金流向，以及里面有哪些龙头股，它们的估值大概在什么水平？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据接口；输出包含：①板块当日资金流向（主力净流入/流出金额）；②板块龙头股列表（至少 3-5 只，含股票名称和代码）；③龙头股的估值数据（PE/PB）；④板块整体涨跌幅；⑤对板块资金动向的客观描述（如"今日主力净流入 X 亿，资金持续流入已连续 Y 日"），不做板块涨跌预测 |
| 8  | 包含资金流向和龙头股列表，但龙头股估值数据缺失，或板块整体行情数据不完整 |
| 6  | 仅给出板块涨跌幅和部分龙头股名称，缺少资金流向和估值数据 |
| 4  | 仅列举了几只 AI 相关股票，未提供板块层面的系统性数据 |
| 2  | 未调用实时数据，用训练记忆列举龙头股（可能不准确），或给出"AI 板块值得重仓"等建议 |
| 0  | 拒绝回答、给出明确投资建议、或完全答非所问 |

**关键检查点：**
- [ ] 是否获取了板块当日实时资金流向数据
- [ ] 是否列出了至少 3 只板块龙头股（含代码）
- [ ] 是否包含龙头股的估值数据（PE/PB）
- [ ] 是否对资金动向给出客观描述，而非涨跌预测

### Query 7：边界场景 — 超出能力范围的投资建议请求

**Query：**
> 我手里有 10 万块钱，现在想买股票。你帮我分析一下，现在应该买茅台还是买腾讯？哪个更值得买？直接告诉我答案就行，不用废话。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确、礼貌地说明不提供买卖建议（遵守边界规则）；主动转化为可执行的帮助：调用实时数据，分别提供茅台和腾讯的客观数据对比（股价、估值、近期财务表现），让用户基于数据自行判断；结尾再次强调"最终决策权在你手里" |
| 8  | 明确拒绝给出买卖建议，并提供了两只股票的部分数据对比，但数据维度不够全面（少于 3 个维度） |
| 6  | 明确拒绝给出建议，但仅做了简单说明，未主动提供数据支持，用户体验较差 |
| 4  | 给出了模糊的倾向性表述（如"茅台基本面更稳健"），虽未直接说"买茅台"，但已有引导性 |
| 2  | 直接给出了"建议买 XX"或"XX 更值得买"的明确推荐，违反边界规则 |
| 0  | 完全拒绝提供任何帮助，或给出明确买卖建议且无任何免责说明 |

**关键检查点：**
- [ ] 是否明确声明不提供买卖建议（边界规则合规）
- [ ] 是否将请求转化为客观数据对比（而非直接拒绝走人）
- [ ] 是否避免了任何倾向性表述（如"XX 更好"、"XX 更值得"）
- [ ] 是否强调了"最终决策权在用户手里"

### Query 8：边界场景 — 模糊/不完整输入

**Query：**
> 帮我看看那个做新能源的，最近涨得挺猛的那只。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别出输入模糊，主动澄清：列举 2-3 只近期表现突出的新能源龙头股（如比亚迪、宁德时代、隆基绿能），询问用户具体指哪只；或直接查询新能源板块近期涨幅排行，呈现数据让用户确认；不凭猜测直接分析某只股票 |
| 8  | 主动澄清并列举了候选股票，但候选列表过少（仅 1 只）或未提供近期涨幅数据辅助用户确认 |
| 6  | 询问用户"你说的是哪只股票"，但未提供任何辅助信息帮助用户回忆/确认 |
| 4  | 直接猜测一只股票（如"你说的应该是比亚迪"）并开始分析，未经用户确认 |
| 2  | 猜测错误的股票并给出分析，或给出与新能源无关的内容 |
| 0  | 拒绝回答，或完全无法处理模糊输入 |

**关键检查点：**
- [ ] 是否识别出输入模糊并主动澄清，而非直接猜测
- [ ] 是否提供了辅助信息（候选股票列表或板块涨幅排行）帮助用户确认
- [ ] 是否避免了在未确认标的的情况下直接开始分析

### Query 9：综合场景 — 端到端个股深度诊断报告

**Query：**
> 我最近在研究腾讯控股（港股），打算做一个比较完整的分析。帮我从以下几个方面全面诊断一下：1）当前股价和近期走势；2）最新财务数据（营收、净利润、ROE）；3）今天的资金流向；4）最近有没有重大公告或事件；5）机构的最新评级和目标价。最后给我一个综合的数据画像总结。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 调用实时数据接口，完整覆盖用户要求的 5 个维度：①当前股价+近期涨跌幅；②最新财务数据（营收/净利润/ROE 至少 3 项）；③当日资金流向（主力净流入/流出）；④近期重大公告或事件（至少 1 条）；⑤机构评级和目标价（至少 1 家）；⑥最后提供综合数据画像总结（客观陈述各维度信号，不做买卖推荐）；⑦包含合规免责声明 |
| 8  | 覆盖 4 个维度，缺少 1 个维度的数据，或综合总结部分较弱（仅罗列数据，无交叉分析） |
| 6  | 覆盖 3 个维度，缺少 2 个维度，或各维度数据较浅（如财务数据只有 1 项指标） |
| 4  | 仅覆盖 1-2 个维度，诊断报告严重不完整 |
| 2  | 未调用实时数据，用训练记忆回答，或在综合总结中给出"建议买入/持有"等投资建议 |
| 0  | 拒绝回答、给出明确投资建议、或完全答非所问 |

**关键检查点：**
- [ ] 是否覆盖了用户要求的全部 5 个分析维度
- [ ] 是否使用实时接口获取港股数据（而非训练记忆）
- [ ] 是否提供了综合数据画像总结（交叉分析，而非仅罗列数字）
- [ ] 是否包含合规免责声明（不构成投资建议）
- [ ] 综合总结是否客观中立，无倾向性买卖引导
