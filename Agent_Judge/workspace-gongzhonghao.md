# 公众号运营专家（workspace-d6sa3w9c6xmv4jar）— LLM-AS-Judge 评测标准

## Agent 背景

本 Agent 是一名专注于微信公众号全流程运营的 AI 助手，核心差异化能力是"文风 DNA"机制——通过从作者历史文章中提炼 14 维风格画像，让 AI 生成的内容像作者本人写的，而非通用 AI 腔。Agent 覆盖从文风提取、选题规划、大纲生成、初稿撰写到标题优化的完整创作链路，严格遵循"每一步只做一件事"和"草稿自动化、发布人工确认"两条核心原则。

## 目标用户画像

| 维度 | 描述 |
|------|------|
| 职业/身份 | 微信公众号创作者，包括个人博主、企业内容运营、知识付费作者、自媒体人 |
| 核心需求 | 降低内容创作的时间成本，同时保持个人写作风格不被 AI 同质化 |
| 使用频率 | 每周 1-3 次，典型触发时机：有选题想法但不知如何展开、有素材但不知如何成文、写完初稿需要优化标题 |
| 专业水平 | 中级内容创作者，有一定写作经验和风格意识，但缺乏系统化的内容生产流程 |

## 核心使用场景

1. **文风初始化**：新用户首次使用，提供 3-10 篇历史文章，让 Agent 提炼个人文风 DNA，为后续所有创作建立风格基准。
2. **选题策划**：有一个模糊想法或一堆素材，需要 Agent 帮助梳理出 2-3 个有价值的选题角度，并推荐最值得写的方向。
3. **大纲确认**：选题确定后，需要 Agent 生成结构化大纲，明确每个章节的核心论点和证据需求，再交给写稿环节。
4. **初稿生成**：大纲确认后，基于参考资料和文风 DNA，生成一版高保真初稿，附带文风贴合说明和风险点。
5. **标题优化**：文章写完后，需要 Agent 生成 8 个标题候选，并推荐最优解及备选方案。

## 评测 Query 与打分标准

### Query 1：基础场景 — 文风 DNA 提取

**Query：**
> 我运营了一个职场成长类公众号两年了，一直觉得 AI 写出来的东西不像我写的，太正式太套路。我把我最近写的 5 篇文章发给你，你能帮我分析一下我的写作风格吗？我希望以后你帮我写稿的时候能更像我自己。
>
> [附：5 篇文章全文]

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出完整的 12 项风格画像结构（样本概况、14 维风格分析、标点符号偏好、分块习惯、段落配方、叙述方法体系、内容推进方式、Few-shot 示例段落、情绪曲线分析、起承转合模板、可复用 Prompt Block、风格画像文件路径）；每个维度包含结论+证据+可执行规则；主动请求用户校准确认，而非直接定稿 |
| 8  | 覆盖大部分维度，但缺少"内容推进方式"或"叙述方法体系"其中一项；或有维度只给结论未给证据示例；仍有校准环节 |
| 6  | 输出了基本的风格描述，但缺少标点符号偏好、分块习惯、段落配方三项中的两项以上；或跳过了用户校准直接给出"最终版" |
| 4  | 只做了词汇层面的模仿分析（高频词、口头禅），未覆盖结构、节奏、论证维度；或输出格式混乱无法直接给写稿技能调用 |
| 2  | 仅给出泛泛的风格描述（如"语气轻松、逻辑清晰"），没有任何可执行规则或示例证据 |
| 0  | 拒绝分析、答非所问、或直接开始帮用户写文章而非提取文风 |

**关键检查点：**
- [ ] 是否覆盖 14 维分析框架的四层（表层/结构/深层/独特标记）
- [ ] 是否单独输出"标点符号偏好"章节，包含各标点的使用频率和场景
- [ ] 是否单独输出"段落配方"章节，包含 1 句段/2-3 句段/长段的占比判断
- [ ] 是否单独输出"内容推进方式"章节，包含作者的典型推进路径
- [ ] 是否在初版画像后主动请求用户校准，而非直接定稿

### Query 2：基础场景 — 选题策划

**Query：**
> 我最近在研究"职场中如何向上管理"这个话题，手头有一些采访记录和自己的观察笔记。但我不知道从哪个角度切入比较好，感觉可以写的方向太多了，反而不知道怎么下手。你能帮我梳理一下吗？我的读者主要是 25-35 岁的职场人，公众号定位是帮他们在职场里少走弯路。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 输出完整的 7 项结构（Input Digest、Topic Angles x2-3、Recommended Angle、Primary Outline、Backup Outline、Evidence Checklist、Waiting For Confirmation）；2-3 个选题角度真正不同而非同义改写；每个角度附带选题评分（读者相关性/传播潜力/证据充足度/可写性/作者匹配度/业务目标匹配度）；推荐角度说明为何优于其他；停在确认环节不写正文 |
| 8  | 覆盖主要结构，但选题角度评分维度不完整（缺少 1-2 个维度）；或大纲有标题但缺少每个 section 的核心论点说明；仍停在确认环节 |
| 6  | 给出了 2-3 个选题角度，但角度之间差异不明显（实质是同一方向的改写）；或只给了主大纲没有备选大纲；或缺少 Evidence Checklist |
| 4  | 只给出了一个选题方向和简单大纲，没有角度对比和推荐理由；或直接开始写正文 |
| 2  | 给出了泛泛的建议（如"可以从沟通技巧角度写"），没有结构化的选题分析和大纲 |
| 0  | 直接写出完整文章正文，跳过选题确认环节；或拒绝回答 |

**关键检查点：**
- [ ] 是否输出了至少 2 个真正不同的选题角度（非同义改写）
- [ ] 每个选题角度是否包含"一句话选题定义+目标读者+值得写的原因+主要风险"
- [ ] 是否给出了主大纲和备选大纲两套方案
- [ ] 大纲中每个 section 是否有核心论点（不只是标题）
- [ ] 是否在最后明确停在确认环节，等待用户确认后再推进

### Query 3：基础场景 — 标题生成

**Query：**
> 我写了一篇文章，讲的是很多职场人在向上汇报时犯的一个核心错误：只汇报进展，不汇报判断。文章里有 3 个真实案例，最后给了一个"汇报公式"。读者是职场新人和中层管理者。帮我出几个标题吧，我想要那种让人看了就想点开的。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 恰好生成 8 个标题候选，覆盖反差型/情绪型/结果承诺型/认知升级型四类；所有标题无引号/冒号/破折号；多分句标题使用逗号分隔；推荐 1 个最佳标题并说明为何在点击和可信之间最平衡；同时给出更稳妥版本和更强传播版本；列出被淘汰标题及原因 |
| 8  | 生成了 8 个标题，格式约束全部满足，但推荐理由不够具体（未说明为何优于其他 7 个）；或缺少稳妥/激进备选说明 |
| 6  | 生成了 8 个标题，但有 1-2 个标题违反硬约束（含引号/冒号/破折号）；或四种类型覆盖不全（只有 2 种类型） |
| 4  | 生成了少于 8 个标题（如 5 个），或多于 8 个；或标题明显 AI 腔（空泛大词、套路化鸡汤） |
| 2  | 给出了几个标题但没有任何评分、推荐或淘汰说明；或标题与文章内容明显不符 |
| 0  | 拒绝生成标题；或生成的标题全部违反硬约束（如全部含冒号） |

**关键检查点：**
- [ ] 是否恰好生成 8 个标题（不多不少）
- [ ] 所有标题是否无引号、无冒号、无破折号
- [ ] 是否覆盖反差型、情绪型、结果承诺型、认知升级型四类
- [ ] 是否推荐 1 个最佳标题并给出具体推荐理由
- [ ] 是否同时提供更稳妥版本和更强传播版本两个备选

### Query 4：进阶场景 — 基于文风 DNA 的初稿生成

**Query：**
> 我之前已经让你帮我建立了文风 DNA 文件了。现在大纲也确认了，主题是"向上汇报的核心错误：只汇报进展，不汇报判断"，大纲分三个部分：1）大多数人汇报的方式是什么样的（现象）；2）为什么只汇报进展会让领导觉得你不够成熟（分析）；3）汇报公式：进展+判断+建议（方法）。参考资料我也准备好了，有 3 个真实案例。帮我写初稿吧。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 开写前明确声明使用的 Style DNA 文件名；输出完整的 6 项结构（Style DNA In Use、Draft v1、Style Adherence Notes、DNA Compliance Report、Weak Sections And Fix Suggestions、Optional Alternative Lead）；初稿段落长度控制在 1-3 句；无破折号；不确定信息标注 `[待补充证据]`；开头 3-5 行有明确的 hook；有具体 CTA |
| 8  | 结构完整，文风贴合说明具体，但 DNA Compliance Report 过于简略（未逐项对照 DNA 规则）；或有 1-2 处段落超过 3 句未说明原因 |
| 6  | 写出了初稿，但未声明使用的 DNA 文件；或缺少 Style Adherence Notes 和 DNA Compliance Report；或初稿中出现破折号 |
| 4  | 写出了初稿，但明显是通用 AI 腔（空泛励志语、抽象名词堆砌、重复句式开头），与文风 DNA 无关联 |
| 2  | 在大纲未确认的情况下直接写稿；或初稿内容与提供的大纲结构严重不符 |
| 0  | 拒绝写稿；或在没有文风 DNA 的情况下直接写稿而不先调用 wechat-style-profiler |

**关键检查点：**
- [ ] 是否在开写前明确声明当前使用的 Style DNA 文件或作者画像名称
- [ ] 初稿段落是否默认控制在 1-3 句（符合微信移动端阅读习惯）
- [ ] 是否输出 DNA Compliance Report（逐项对照文风规则的自检报告）
- [ ] 初稿中是否无破折号（硬约束）
- [ ] 不确定的信息是否标注了 `[待补充证据]`

### Query 5：进阶场景 — 从语音底稿到初稿

**Query：**
> 我录了一段 20 分钟的语音，讲了我对"职场中的信息差"这个话题的一些想法，已经转成文字了，大概 3000 字，但很口语化、逻辑也比较乱。我想把它整理成一篇公众号文章，大概 1500-2000 字。我的文风 DNA 文件已经有了。你能帮我先规划一下选题和大纲，确认后再写稿吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 正确识别这是"选题+大纲"阶段，先运行 wechat-topic-outline-planner；从语音底稿中区分事实、观点、情绪和未经证实的判断（Input Digest）；给出 2-3 个真正不同的选题角度；推荐主角度并说明理由；给出主大纲和备选大纲；停在确认环节，明确告知用户确认后再写稿 |
| 8  | 流程正确（先规划后写稿），但 Input Digest 对语音底稿的整理不够细致（未区分事实/观点/情绪）；或选题角度评分维度不完整 |
| 6  | 给出了选题和大纲，但两个阶段（规划和写稿）混在一起，没有明确的确认节点；或直接给出一个选题方向而非 2-3 个角度对比 |
| 4  | 跳过选题规划，直接开始整理语音底稿并写初稿；或把语音底稿原文直接改写成文章而不做结构化规划 |
| 2  | 只做了简单的语音底稿整理（去除口语化表达），没有选题策划和大纲生成 |
| 0  | 拒绝处理语音底稿；或完全忽略用户"先规划再写稿"的明确要求 |

**关键检查点：**
- [ ] 是否正确识别当前阶段为"选题+大纲"而非"写稿"
- [ ] 是否对语音底稿做了 Input Digest（区分事实/观点/情绪/待验证信息）
- [ ] 是否给出了 2-3 个真正不同的选题角度
- [ ] 是否明确停在确认环节，等待用户确认后再推进到写稿阶段
- [ ] 是否给出了主大纲和备选大纲两套方案

### Query 6：进阶场景 — 文风 DNA 校准迭代

**Query：**
> 你上次帮我生成的文风 DNA 我看了，整体方向对，但有几个地方不太准：我其实很少用感叹号，你说我"情绪表达强烈"这个判断有点偏；另外我的段落其实比你描述的要短，我基本上一个意思就换行了。其他的都还好。你能帮我修正一下这两个地方吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 定点修正用户指出的两处偏差（感叹号使用频率、段落长度），而非整体重写；修正后给出具体的修订内容（如"感叹号：从'高频使用'改为'极少使用，仅在强调核心判断时出现'"）；主动询问是否还有其他需要校准的地方；更新 DNA 文件并告知文件路径 |
| 8  | 正确定点修正两处偏差，但修正描述不够具体（如只说"已修正感叹号描述"而未给出修改前后对比）；或未主动询问是否还有其他校准需求 |
| 6  | 修正了用户指出的问题，但同时对其他维度也做了不必要的改动（违反"定点修正"原则）；或修正后未更新 DNA 文件 |
| 4  | 整体重写了文风 DNA，而非定点修正；或只修正了一处，忽略了另一处 |
| 2  | 对用户的校准反馈表示认同但没有实际修改 DNA 文件；或给出了修改建议但要求用户自己去改 |
| 0  | 拒绝修改；或声称文风 DNA 分析是正确的，不接受用户校准 |

**关键检查点：**
- [ ] 是否采用定点修正而非整体重写
- [ ] 修正内容是否给出了修改前后的具体对比
- [ ] 是否更新了 DNA 文件并告知文件路径
- [ ] 是否主动询问是否还有其他需要校准的地方
- [ ] 是否保持了其他维度不变（未做不必要的改动）

### Query 7：边界场景 — 要求直接发布

**Query：**
> 这篇文章我觉得写得挺好的，你帮我直接发布到公众号吧，今天下午 3 点发。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确说明自己不会代替用户做发布决策（"草稿自动化、发布人工确认"原则）；解释原因（发布是需要人工确认的环节）；提供实际可帮助的替代方案（如帮助检查文章、生成发布前检查清单、提醒发布时间）；语气友好不说教 |
| 8  | 正确拒绝发布操作，并给出替代方案，但解释过于简短或语气略显生硬 |
| 6  | 正确拒绝发布操作，但没有给出任何替代方案，只是说"我无法帮你发布" |
| 4  | 含糊其辞，既没有明确拒绝也没有明确说明边界，给用户造成困惑 |
| 2  | 尝试模拟发布操作（如给出"已发布"的假反馈），或声称可以帮助发布 |
| 0  | 直接声称已经帮用户发布了文章 |

**关键检查点：**
- [ ] 是否明确说明不会代替用户做发布决策
- [ ] 是否解释了"草稿自动化、发布人工确认"的原则
- [ ] 是否提供了实际可帮助的替代方案
- [ ] 语气是否友好，没有说教感

### Query 8：边界场景 — 输入不完整时的处理

**Query：**
> 帮我写一篇关于时间管理的文章。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 识别出输入不完整（缺少目标读者、文章目标、参考资料、文风 DNA 等关键信息）；先运行 wechat-topic-outline-planner 而非直接写稿；主动询问必要信息（至少：目标读者是谁、想从哪个角度切入、有没有具体素材）；不直接拒绝，而是引导用户补充信息 |
| 8  | 正确识别需要先做选题规划，主动询问了部分必要信息，但遗漏了文风 DNA 的确认（未询问是否已有 DNA 文件） |
| 6  | 给出了 2-3 个选题角度供用户选择，但没有询问目标读者和文章目标，直接基于假设推进 |
| 4  | 直接开始写文章，但在文章开头加了一段说明"由于信息不足，我做了以下假设……" |
| 2  | 直接写出了一篇通用的时间管理文章，没有任何个性化处理 |
| 0  | 拒绝回答，要求用户提供完整信息后再来 |

**关键检查点：**
- [ ] 是否识别出输入不完整，而非直接开始写稿
- [ ] 是否先触发 wechat-topic-outline-planner 流程
- [ ] 是否主动询问目标读者、文章目标等关键信息
- [ ] 是否询问用户是否已有文风 DNA 文件
- [ ] 是否以引导而非拒绝的方式处理不完整输入

### Query 9：综合场景 — 从素材到标题的完整链路

**Query：**
> 我是一个做职场成长内容的公众号作者，粉丝大概 2 万。我最近采访了 5 个在大厂工作超过 10 年的人，问他们"如果重来一次，你最希望 25 岁的自己知道什么"，整理了一份 5000 字的采访纪要。我想把这个做成一篇文章，但不知道怎么切入。我已经有文风 DNA 文件了。你能帮我从选题规划开始，一步步做到最后出标题吗？

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 严格按照"选题规划→大纲确认→初稿撰写→标题生成"的顺序推进，每个阶段完成后等待用户确认再进入下一阶段；选题阶段给出 2-3 个真正不同的角度（如"认知升级型"vs"方法论型"vs"情感共鸣型"）；大纲阶段每个 section 有核心论点；初稿阶段声明 DNA 文件并输出 DNA Compliance Report；标题阶段恰好 8 个候选，覆盖四类，推荐最优并给出稳妥/激进备选 |
| 8  | 流程顺序正确，各阶段输出质量高，但某一阶段的确认节点不够明确（如选题确认后直接进入大纲而未等待用户明确确认）；或标题阶段缺少淘汰标题的说明 |
| 6  | 流程基本正确，但有一个阶段被压缩或跳过（如大纲和初稿合并处理）；或各阶段输出结构不完整（缺少 Evidence Checklist 或 DNA Compliance Report） |
| 4  | 流程顺序混乱（如先写初稿再补大纲）；或某个阶段的输出质量明显低于其他阶段 |
| 2  | 直接输出了一篇完整文章和几个标题，跳过了所有中间确认环节 |
| 0  | 拒绝处理；或完全忽略用户"一步步来"的明确要求，一次性输出所有内容 |

**关键检查点：**
- [ ] 是否严格按照四阶段顺序推进，每阶段后等待用户确认
- [ ] 选题阶段是否给出了真正不同的 2-3 个角度，并附带选题评分
- [ ] 初稿阶段是否声明了使用的 DNA 文件并输出 DNA Compliance Report
- [ ] 标题阶段是否恰好生成 8 个候选，覆盖四类，无格式违规
- [ ] 全程是否遵循"每一步只做一件事"的原则，没有越界提前执行下一步
