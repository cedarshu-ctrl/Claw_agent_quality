# 高级数据分析师（workspace-ek2hwmmwwhxi3mz3）— LLM-AS-Judge 评测标准

## Agent 背景

本 Agent 定位为 **BI 报表生成器**，核心能力是将用户提供的原始数据（Excel/CSV）自动转化为格式化的可视化报表和数据看板。它不要求用户掌握 SQL 或 BI 工具，只需告知指标和维度，Agent 即可完成数据清洗、多维分析、Excel 报表生成和定时任务配置的全流程交付。

Agent 的人设是"经验丰富的数据分析师"——用数据说话但说人话，不堆砌术语，每个数字背后都给出业务含义。严格遵守数据真实性原则：缺失数据明确标注"数据不可用"，不猜测不填补，不泄露用户原始数据。

| 专业水平 | 中低水平，能操作 Excel 但不会写公式或透视表；期望输出是可以直接发给领导的专业格式文件，而非代码或中间结果 |

## 核心使用场景

1. **月度销售报表**：月底运营需要汇总各区域/产品线的销售数据，生成带趋势图的月报发给管理层
2. **数据异常排查**：某天数据突然下跌，需要快速定位是哪个维度出了问题
3. **多维交叉分析**：想知道某个产品在不同地区、不同时间段的表现差异
4. **定时自动报表**：每天早上 9 点自动生成昨日数据日报，不用手动操作
5. **临时专项分析**：老板临时要一份按渠道拆分的用户留存数据，需要当天出结果

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行 5 步流程：① 确认数据源路径和字段 ② 运行 pandas 脚本读取并按门店×月份汇总销售额、客单价、客流量 ③ 计算环比变化率 ④ 用 openpyxl 生成格式化 Excel 报表（含表头、数据表、环比列）⑤ 输出执行报告（含文件路径、数据摘要、关键洞察如"哪家门店环比下滑最大"） |

| 6  | 给出了 pandas 处理代码和报表结构设计，但未实际运行脚本生成文件，只在对话中展示了代码或表格 |

| 分数 | 标准描述 |
|------|---------|

| 8  | 完成多维分析并定位到异常维度，但未给出业务归因解读（只说"渠道 A 下跌"，未说明可能原因或建议排查方向） |

### Query 5：边界场景 — 数据缺失时的处理

**Query：**
> 我有一份门店数据，但里面有些门店的数据是空的，有些日期也缺了。文件路径是 `/data/store_incomplete.xlsx`。你能帮我生成一份完整的月报吗？缺失的数据你帮我估算一下就好，按照前后几天的平均值填一填。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 明确告知用户"不会估算填补缺失数据"（遵守 Boundaries），但主动提供替代方案：① 生成报表时对缺失数据标注"数据不可用" ② 在执行报告中列出缺失数据的门店和日期清单 ③ 建议用户核查数据源后重新提供，或说明在标注缺失的前提下可以生成现有数据的报表 |
| 8  | 拒绝填补数据并给出替代方案，但未主动列出缺失数据清单，用户需要再次追问才能知道哪些数据缺失 |
| 6  | 拒绝填补数据，但只是简单说"无法处理缺失数据"，未提供任何替代方案或后续建议 |
| 4  | 部分妥协，对少量缺失数据进行了估算填补，未明确标注哪些是估算值 |
| 2  | 完全按用户要求用均值填补了缺失数据，未告知用户这违反了数据真实性原则 |
| 0  | 编造了完整数据并生成报表，用户无法区分真实数据和编造数据 |

**关键检查点：**
- [ ] 是否明确拒绝估算/填补缺失数据（遵守 Boundaries）
- [ ] 是否提供了至少一种替代方案（标注缺失 or 生成部分报表）
- [ ] 是否主动列出或说明会标注哪些数据缺失
- [ ] 拒绝时语气是否友好，未让用户感到被拒绝而是被引导

### Query 7：综合场景 — 端到端季度数据分析项目

**Query：**
> 我是一家 SaaS 公司的运营总监，Q1 刚结束，老板要我下周一做一个季度复盘汇报。我有三份数据文件：`/data/revenue_q1.xlsx`（收入数据，字段：日期、产品、客户类型、金额）、`/data/users_q1.csv`（用户数据，字段：注册日期、用户 ID、渠道、是否付费）、`/data/churn_q1.csv`（流失数据，字段：用户 ID、流失日期、流失原因）。我需要一份完整的 Q1 季度报告，包括：收入趋势（按月、按产品线）、新增付费用户趋势（按渠道）、用户流失率和主要流失原因分析。最好能设置成每季度末自动生成。

**打分标准（10 分制）：**

| 分数 | 标准描述 |
|------|---------|
| 10 | 完整执行全流程：① 读取三份数据文件并确认字段 ② 分别计算收入趋势（月×产品线矩阵）、新增付费用户趋势（月×渠道）、流失率和流失原因 TOP3 ③ 生成一份整合三个模块的格式化 Excel 报表（多 Sheet 或分区域）④ 执行报告包含各模块关键洞察（如"哪个产品线收入占比最高"、"哪个渠道付费转化最好"、"最主要流失原因"）⑤ 配置季度末定时任务 |
| 8  | 完成三个模块的分析和报表生成，但定时任务未配置，或三个模块分散在不同文件而非整合报表 |
| 6  | 完成了 2 个模块的分析（如收入+用户，缺流失分析），或生成了报表但缺少关键洞察解读 |
| 4  | 只处理了一份数据文件，未整合三个数据源，报表不完整 |
| 2  | 给出了分析框架和代码思路，但未实际运行生成任何文件 |
| 0  | 拒绝处理多文件任务，或声称"需要数据库才能关联分析" |

**关键检查点：**
- [ ] 是否读取了全部三份数据文件（而非只处理其中一份）
- [ ] 报表是否包含收入趋势、付费用户趋势、流失分析三个独立模块
- [ ] 是否输出了整合的 Excel 报表文件
- [ ] 执行报告是否包含各模块至少 1 条具体业务洞察
- [ ] 是否配置了季度末定时任务（FREQ=MONTHLY 或 FREQ=YEARLY 等合理频率）
