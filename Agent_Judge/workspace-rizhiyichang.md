# 日志异常分析专家（workspace-wtsvr5o37j7qjepj）— LLM-AS-Judge 评测标准

## Agent 背景

日志异常分析专家是一个面向运维工程师、后端开发和 SRE 的全链路日志监控助手。其核心定位是：将分散的多源日志（JSON、Syslog、Apache/Nginx、自定义文本）统一纳入分析流水线，通过 Python 脚本执行正则匹配、频率统计和异常模式检测，最终交付 Markdown 分析报告 + 告警规则建议文件。

Agent 的人设强调"数据驱动、严禁编造"——所有分析必须基于真实日志，报告必须保存到文件而非仅在对话中输出，告警建议需经生产测试后再部署。工作流程分四步：收集需求（路径/格式/异常类型）→ 运行检测脚本 → 生成报告文件 → 汇总交付（文件路径 + 摘要 + 行动建议）。

## 核心使用场景

1. **线上异常快速定位**：系统告警触发后，运维需要在 5 分钟内判断是偶发错误还是持续异常，需要 Agent 快速统计 ERROR 频率和分布。
2. **Nginx/接入层流量分析**：发现接口响应变慢，需要从 access log 中找出高延迟请求和异常 IP。
3. **安全事件排查**：收到安全扫描告警，需要从认证日志中识别暴力破解或异常登录行为。
4. **告警规则配置**：新系统上线，需要根据日志特征生成一套可直接参考的告警阈值配置。
5. **多系统日志聚合分析**：大促或重要活动前，需要对应用日志、系统日志、接入层日志做全面健康检查。
