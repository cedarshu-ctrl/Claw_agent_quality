## 这个 Skill 解决什么问题？

跨境电商运营每天要盯几十个竞品的价格、Listing、评论变化，手动操作效率极低且容易遗漏。这个 Skill 让 OpenClaw 成为你的**自动化竞品雷达**：

- 竞品降价 → 5 分钟内收到通知
- 竞品 Listing 更新主图/标题 → 自动截图对比
- 竞品差评突然增多 → 自动分析原因并推送
- 新竞品上架 → 自动做初步分析

## 实操流程

### Step 1：初始竞品档案建立

```
对 OpenClaw 说：

"帮我建立以下竞品的分析档案：

竞品列表：
1. BlendJet 2 — ASIN: B0CXXXXX
2. NutriBullet Go — ASIN: B0AXXXXX  
3. Hamilton Beach — ASIN: B09XXXXX

每个竞品需要收集：
- 产品标题、价格、评分、评论数
- BSR排名、类目排名
- 五点描述和关键词
- 主图和A+页面截图
- 最近30天的评论趋势

保存为结构化文档"
```

### Step 2：设置自动监控

```
"设置以下自动监控任务：

1. 【每日监控】每天早上9点：
   - 查看以上3个竞品的价格变化
   - 如果价格变动超过5%，立即通知我
   - 生成一份每日竞品价格摘要

2. 【每周报告】每周一上午10点：
   - 生成完整的周度竞品动态报告
   - 包含：价格变化、Listing变更、新增评论分析、BSR趋势
   - 推送到飞书/企微

3. 【即时告警】以下情况立即通知：
   - 竞品价格降幅超过15%（可能在做大促）
   - 竞品差评突然增多（24h内新增>10条1星）
   - 竞品上架新变体/新产品"
```

### Step 3：竞品深度拆解

```
"对 BlendJet 2（ASIN: B0CXXXXX）做深度拆解：

1. ASIN 数据收集：
   - 提取完整的产品标题、五点描述、产品描述
   - 抓取所有变体的价格和库存状态
   - 统计评分分布（5星/4星/3星/2星/1星各占比）

2. 规格提取：
   - 技术参数（容量、电池、功率、材质）
   - 包装内容
   - 认证信息（FDA、BPA-Free等）

3. 评论质量分析：
   - Top 20 好评关键词
   - Top 20 差评关键词
   - 情感倾向分析（正面/中性/负面占比）
   - 买家最关心的3个维度

4. 竞争壁垒识别：
   - 品牌护城河（DTC品牌故事、社媒粉丝量）
   - 技术壁垒（专利、独家功能）
   - 渠道壁垒（独家合作、分销网络）
   - 可攻破的弱点和市场机会"
```

### Step 4：竞品价格策略拆解

```
"分析以下3个竞品在过去90天的价格策略：

1. 标出每次调价的日期和幅度
2. 识别促销周期（Prime Day、黑五、日常Coupon）
3. 分析 Coupon 面额和使用频率
4. 对比各平台价格差异（Amazon / 官网 / Walmart）
5. 推断其定价逻辑和利润空间
6. 给出我的应对策略建议"
```

## 自动化场景配置

### 场景 A：每日竞品价格快报

```
触发：每天 09:00 UTC+8
动作：
  1. 抓取所有监控竞品的当前价格
  2. 与昨日价格对比
  3. 生成摘要表格
  4. 推送到飞书群
  
输出格式：
┌──────────────┬────────┬────────┬────────┐
│ 竞品          │ 昨日价格│ 今日价格│ 变化    │
├──────────────┼────────┼────────┼────────┤
│ BlendJet 2   │ $44.99 │ $39.99 │ -11.1% ⚠️│
│ NutriBullet  │ $34.99 │ $34.99 │ 0%      │
│ Hamilton     │ $19.99 │ $17.99 │ -10.0% ⚠️│
└──────────────┴────────┴────────┴────────┘
```

### 场景 B：竞品 Listing 变更检测

```
触发：每天检查一次
动作：
  1. 抓取当前 Listing 内容
  2. 与上次快照 diff 对比
  3. 标出变更内容（标题/图片/五点/A+）
  4. 截图保存变更前后对比
  
输出：
"⚠️ BlendJet 2 Listing 变更检测：
 - 标题新增关键词 'USB-C Fast Charge'
 - 主图更换（新版增加了使用场景）
 - 五点描述第3条修改（强调了降噪功能）
 建议：关注其强调的新卖点，评估是否跟进"
```

### 场景 C：差评突增告警

```
触发：实时/每6小时
条件：24小时内新增1-2星评论 > 10条
动作：
  1. 抓取所有新增差评内容
  2. AI分析差评主题聚类
  3. 判断是产品质量问题还是恶意攻击
  4. 推送告警 + 建议

输出：
"🚨 竞品差评突增告警：
 BlendJet 2 过去24h新增 15 条1星评论
 
 差评聚类分析：
 - 40%：充电口松动/无法充电
 - 30%：搅拌不够细腻
 - 20%：漏水问题
 - 10%：客服响应慢
 
 判断：疑似新批次质量问题
 建议：这是我们的机会窗口！可以在Listing中强调
       我们的'Type-C稳固充电口'和'IP68防水'卖点"
```

## 注意事项

1. Amazon 数据抓取需遵守 ToS，建议使用官方 API 或合规工具
2. 监控频率不宜过高，建议每日 1-2 次检查，避免触发反爬
3. 竞品 ASIN 变更时需及时更新监控列表
4. 大促期间（Prime Day/黑五）建议加密监控频率到每 4 小时
