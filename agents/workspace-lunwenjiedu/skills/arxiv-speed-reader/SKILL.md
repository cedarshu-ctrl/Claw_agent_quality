# AI论文速读

> 论文太多读不完？AI帮你秒懂arXiv论文核心观点，自动对比多篇方法差异

## 前置依赖

```bash
pip install pandas openpyxl
```

## 核心能力

### 能力1：通过论文 ID 或关键词搜索论文（web_search/web_fetch）

用 `web_search` 搜索相关信息。

### 能力2：提取摘要、方法论、实验结果和核心贡献

用 `web_fetch` 抓取arXiv论文页面，提取标题、摘要、核心方法和实验结果。

### 能力3：用通俗语言解释关键创新点

用 `web_fetch` 抓取arXiv论文页面，提取标题、摘要、核心方法和实验结果。

### 能力4：与相关论文对比分析

运行脚本进行数据分析处理。

### 能力5：生成阅读笔记文档

用 `write_to_file` 生成文件。

## 使用流程

### 步骤 1：收集用户需求

向用户确认以下信息（如果未主动提供）：
- 关注哪个研究领域？（NLP/CV/ML/LLM/Robotics等）
- 需要阅读的论文链接/arXiv ID（或「帮我找最新的」）
- 输出偏好（极简摘要/详细解读/中英对照）

### 步骤 2：检索外部信息

执行以下搜索获取真实数据：

```
web_search("[用户主题] paper arXiv")
web_search("[用户主题] 研究综述")
```

对搜索结果中的重要链接，用 `web_fetch` 抓取页面详细内容，提取关键信息。

确保获取到以下资源：
- 论文 PDF 链接
- 相关论文推荐

### 步骤 3：运行脚本处理数据

```bash
python3 scripts/arxiv_speed_reader_tool.py run \
  --input "用户提供的输入" \
  --output "/path/to/output_file"
```

读取脚本输出的结果，确认数据处理成功。

### 步骤 4：生成最终产出

基于脚本输出和搜索到的资源，用 `write_to_file` 生成以下文件：

- **论文阅读笔记（Markdown）**
- **对比分析表**

输出格式要求：Markdown 阅读笔记 + 对比分析

### 步骤 5：汇总交付

向用户展示：
1. 生成的文件路径和内容摘要
2. 搜集到的资源链接列表
3. 关键发现和建议

## 验收标准

- ✅ 核心贡献提取准确
- ✅ 方法论解释通俗
- ✅ 对比分析有价值
- ✅ 笔记结构清晰

## 场景化适配

根据论文领域（NLP/CV/RL）调整分析深度

## 注意事项

- 所有数据必须来自 `web_search` / `web_fetch` 的真实搜索结果，**严禁编造数据**
- 数据缺失时标注"数据不可用"而非猜测
- 报告必须保存为文件（`write_to_file`），不能只在对话中输出
- 建议结合人工判断使用，AI 分析仅供参考
