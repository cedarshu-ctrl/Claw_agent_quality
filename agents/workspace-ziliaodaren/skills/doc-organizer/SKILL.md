# 把一堆资料整理成结构化文档

> 收藏夹塞满了却从没看？把资料链接发给我，帮你提炼出有用的东西

## 前置依赖

```bash
pip install pandas
```

## 核心能力

### 能力1：读取我提供的链接内容（web_fetch）

用 `web_fetch` 抓取页面内容 / 用 `read_file` 读取文件。

### 能力2：按主题自动分类归档

用 `web_fetch` 抓取用户提供的链接内容，提取正文和关键信息；用 `read_file` 读取本地文件内容。

### 能力3：提取每篇的核心观点和关键数据

用 `web_fetch` 抓取用户提供的链接内容，提取正文和关键信息；用 `read_file` 读取本地文件内容。

### 能力4：整合成一份结构化的 Markdown 文档

用 `web_fetch` 抓取用户提供的链接内容，提取正文和关键信息；用 `read_file` 读取本地文件内容。

### 能力5：生成摘要和关键词索引

用 `write_to_file` 生成文件。

## 使用流程

### 步骤 1：收集用户需求

向用户确认以下信息（如果未主动提供）：
- 要整理的资料在哪里？（提供链接列表或文件目录）
- 希望按什么分类？（主题/时间/来源）
- 输出格式偏好？（Markdown文档/PDF/Excel）

### 步骤 2：检索外部信息

执行以下搜索获取真实数据：

```
web_search("[资料主题] 知识框架 分类方法")
web_search("[资料主题] 核心概念总结")
```

对搜索结果中的重要链接，用 `web_fetch` 抓取页面详细内容，提取关键信息。

确保获取到以下资源：
- 资料原文链接
- 核心观点摘要
- 关键数据点

### 步骤 3：运行脚本处理数据

```bash
python3 scripts/doc_organizer_tool.py run \
  --input "用户提供的输入" \
  --output "/path/to/output_file"
```

读取脚本输出的结果，确认数据处理成功。

### 步骤 4：生成最终产出

基于脚本输出和搜索到的资源，用 `write_to_file` 生成以下文件：

- **结构化 Markdown 文档**
- **关键词索引文件**

输出格式要求：Markdown 结构化文档 + 关键词索引

### 步骤 5：汇总交付

向用户展示：
1. 生成的文件路径和内容摘要
2. 搜集到的资源链接列表
3. 关键发现和建议

## 验收标准

- ✅ 所有链接内容已抓取
- ✅ 分类合理
- ✅ 摘要准确
- ✅ 生成可打印文档

## 场景化适配

根据资料类型（学术/技术/新闻）调整整理结构


## 注意事项

- 所有数据必须来自 `web_search` / `web_fetch` 的真实搜索结果，**严禁编造数据**
- 数据缺失时标注"数据不可用"而非猜测
- 报告必须保存为文件（`write_to_file`），不能只在对话中输出
- 建议结合人工判断使用，AI 分析仅供参考
