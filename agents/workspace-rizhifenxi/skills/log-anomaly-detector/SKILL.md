# 日志聚合与异常识别

> 智能日志分析

## 前置依赖

```bash
pip install pandas
```

## 核心能力

### 能力1：读取日志文件（read_file）

用 `web_fetch` 抓取页面内容 / 用 `read_file` 读取文件。

### 能力2：解析日志格式（时间/级别/内容）

用 `read_file` 读取日志文件，用 `execute_command` 运行Python脚本进行正则匹配、频率统计、异常模式检测。

### 能力3：统计错误频率和分布

用 `read_file` 读取日志文件，用 `execute_command` 运行Python脚本进行正则匹配、频率统计、异常模式检测。

### 能力4：识别异常模式（execute_command + Python）

用 `read_file` 读取日志文件，用 `execute_command` 运行Python脚本进行正则匹配、频率统计、异常模式检测。

### 能力5：生成日志分析报告和告警建议

用 `write_to_file` 生成文件。

## 使用流程

### 步骤 1：收集用户需求

向用户确认以下信息（如果未主动提供）：
- 日志文件路径（单个文件或目录）
- 日志格式（JSON/纯文本/syslog/自定义）
- 关注什么类型的异常？（错误/超时/频率异常/安全事件）
- 是否需要定时监控？

### 步骤 2：运行脚本处理数据

```bash
python3 scripts/log_anomaly_detector_tool.py run \
  --input "用户提供的输入" \
  --output "/path/to/output_file"
```

读取脚本输出的结果，确认数据处理成功。

### 步骤 3：生成最终产出

基于脚本输出和搜索到的资源，用 `write_to_file` 生成以下文件：

- **日志分析报告（Markdown）**
- **告警规则建议**

输出格式要求：Markdown 分析报告 + 统计图表

### 步骤 4：汇总交付

向用户展示：
1. 生成的文件路径和内容摘要
2. 搜集到的资源链接列表
3. 关键发现和建议

## 注意事项

- 所有数据必须来自 `web_search` / `web_fetch` 的真实搜索结果，**严禁编造数据**
- 数据缺失时标注"数据不可用"而非猜测
- 报告必须保存为文件（`write_to_file`），不能只在对话中输出
- 建议结合人工判断使用，AI 分析仅供参考
