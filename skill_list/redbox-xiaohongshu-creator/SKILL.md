# RedBox — 小红书版 OpenClaw，自媒体创作者的 AI 工作台

> 一个专为小红书创作者设计的本地化桌面端 AI 创作工作台，打通「插件采集 → 知识库 → AI 创作 → 自动化发布」全链路。被称为"小红书版 OpenClaw"，将灵感-创作-发布的全流程搬到本地，AI 接管执行环节。

## 这个工具解决什么问题？

小红书内容创作者的日常困境：

| 痛点 | 具体表现 |
|:---|:---|
| **素材散落** | 爆款笔记收藏了一堆，但都在收藏夹里吃灰，用的时候找不到 |
| **选题难** | 不知道写什么，缺乏系统化的灵感来源 |
| **写作慢** | 一篇笔记从构思到出图要 2-3 小时 |
| **发布累** | 手动编辑、配图、选封面、写标题，每一步都重复劳动 |
| **矩阵号管理** | 多个账号/赛道的素材混在一起，无法隔离管理 |

**RedBox 的核心理念**：让 AI 接管"脏活累活"，创作者只负责审核和调整方向。

## 十大核心功能

| # | 功能模块 | 说明 |
|:---:|:---|:---|
| 1 | **Chrome 插件采集** | 一键采集小红书笔记/文章、YouTube 视频、网页链接、浏览器选中文字 |
| 2 | **本地知识库与检索** | 采集的素材存入本地知识库，支持高效检索，数据安全不外泄 |
| 3 | **多空间隔离** | 不同项目/赛道/账号的素材互不干扰，适合矩阵号管理 |
| 4 | **随机漫步灵感生成** | AI 随机漫步模式，从已有素材中碰撞出新选题和灵感 |
| 5 | **稿件编辑器 + AI 辅助** | 内置编辑器，AI 辅助写作、扩写、缩写、改写 |
| 6 | **智囊团与群聊协作** | 模拟多个 AI 角色进行头脑风暴，激发创意 |
| 7 | **RedClaw 自动化执行台** | 核心引擎：设定规则后 AI 自动排期、写作、发布 |
| 8 | **AI 生图与媒体库联动** | AI 生成配图，与媒体库无缝联动 |
| 9 | **封面图生成** | 模板图 + 底图 + 标题自动组合生成封面 |
| 10 | **后台定时/长周期任务** | 支持后台运行定时任务，实现真正的自动化内容产出 |

## 安装与配置

### 环境要求

- 桌面端应用（支持 Windows / macOS）
- Chrome 浏览器（安装采集插件）
- AI 模型 API Key（自行准备，支持 OpenAI / Qwen / DeepSeek 等）

### 安装步骤

```bash
# 方式一：直接下载安装包
# 前往 GitHub Releases 页面下载最新版本
https://github.com/Jamailar/RedBox/releases

# 方式二：从源码构建
git clone https://github.com/Jamailar/RedBox.git
cd RedBox
npm install
npm run build
```

### AI 模型配置

```
1. 打开 RedBox 应用
2. 进入 设置 → AI 选项卡
3. 填写：
   - Endpoint：你的 AI 模型 API 地址
     例如：https://api.openai.com/v1
     或：https://dashscope.aliyuncs.com/compatible-mode/v1
   - Key：你的 API 密钥
   - Model：模型名称（如 gpt-4o / qwen-max / deepseek-chat）
4. 点击「测试连接」确认可用
5. 保存配置
```

### Chrome 插件安装

```
1. 打开 Chrome → 设置 → 扩展程序
2. 开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 RedBox 项目目录下的 Plugin/ 文件夹
5. 插件安装完成，浏览小红书时即可一键采集
```

## 实操流程

### Step 1：素材采集

```
1. 安装 Chrome 插件后，打开小红书浏览

2. 看到优质笔记 → 点击插件按钮 → 一键采集
   采集内容包括：标题、正文、图片、标签、互动数据

3. YouTube 视频采集：
   复制视频链接 → RedBox 自动识别并下载
   （需要系统安装 yt-dlp）

4. 网页文章采集：
   选中文字 → 右键 → 发送到 RedBox 知识库
```

### Step 2：知识库管理

```
1. 创建工作空间：
   "为我的美食账号创建一个独立空间"
   "再创建一个'健身减脂'空间"
   → 不同赛道的素材完全隔离

2. 素材分类：
   采集的笔记自动按来源分类
   支持手动打标签和建文件夹

3. 素材检索：
   "搜索所有关于'蛋白质摄入'的采集内容"
   → 本地检索，速度快，不依赖网络
```

### Step 3：灵感激发 — 随机漫步

```
打开「漫步」模式：

RedBox 会从你的知识库中随机抽取素材，
通过 AI 进行关联和碰撞，产生新的选题灵感。

例如：
- 知识库中有"低卡甜品"和"健身打卡"两个素材
- AI 碰撞出选题："健身党的解馋指南：5款低卡甜品让你吃不胖"

看到喜欢的灵感 → 点击「开始创作」
→ 素材自动投喂给 RedClaw 创作引擎
```

### Step 4：AI 创作

```
方式一：编辑器 + AI 辅助
  打开稿件编辑器，手动写作
  遇到卡壳 → 选中文字 → AI 扩写/改写/续写
  
方式二：智囊团群聊
  "召集智囊团讨论：如何写一篇关于xxx的爆款笔记？"
  多个 AI 角色（策划/文案/设计/SEO）各自给出建议
  
方式三：RedClaw 全自动创作
  "基于以下素材，自动生成一篇小红书笔记：
   - 正文：1000字以内，分3-4个段落
   - 配图：生成3张相关图片
   - 封面：用模板生成带标题的封面图
   - 标题：生成5个标题候选"
```

### Step 5：自动化发布

```
RedClaw 自动化引擎配置：

1. 设定排期规则：
   "每天上午10点和晚上8点各发一篇"
   
2. 设定内容来源：
   "从知识库'美食'空间中选取素材"
   
3. 设定创作规则：
   "每篇笔记800-1200字，配3张图，
    风格参考已采集的爆款笔记结构"

4. 设定审核流程：
   "生成后先存草稿，等我确认后再发布"
   或
   "直接自动发布"（谨慎使用）

→ AI 自动排期 → 自动选题 → 自动写作 → 自动生图
→ 你只需要每天花10分钟审阅确认
```

## SEO 起号方法论（与 RedBox 高度吻合）

RedBox 的底层逻辑与小红书 SEO 起号三步法完美匹配：

| 步骤 | 方法 | RedBox 对应功能 |
|:---|:---|:---|
| **建词库** | 收集目标领域的高搜索量关键词 | Chrome 插件采集爆款笔记 → 提取高频关键词 → 存入知识库 |
| **埋词** | 在标题、正文、标签中自然嵌入关键词 | AI 创作时自动嵌入目标关键词 |
| **冲排名** | 持续高频产出优质内容 | RedClaw 自动化定时发布 + 后台长周期任务 |

## 注意事项

1. **需要自备 AI API Key**，RedBox 本身不提供免费 AI 服务。推荐 Qwen/DeepSeek（性价比高）或 GPT-4o（质量最好）
2. **目前上手门槛不低**，需要一定技术配置能力（配 API、装插件），纯新手可能需要等更易用的版本
3. **YouTube 采集功能需要安装 yt-dlp**（`brew install yt-dlp` 或 `pip install yt-dlp`）
4. **自动发布功能请谨慎使用**，建议始终保持"草稿 + 人工审核"的模式，避免低质内容影响账号权重
5. **仿写不等于抄袭**，采集爆款是为了学习结构和选题方向，输出内容必须原创
6. **本地运行保证数据安全**，所有素材和知识库数据都存在你自己的电脑上
7. **定期备份工作空间数据**，避免系统崩溃导致素材库丢失
